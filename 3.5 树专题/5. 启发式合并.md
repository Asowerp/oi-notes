#树 #启发式合并 #并查集 

启发式合并的另外一个名字是 DSU on tree。前置知识是 [[4. 树链剖分]]。

有时候把树拍到数组上然后利用 [[11. 莫队算法]] 也能达到这个效果。

回到 [[4. 树链剖分]] 中的一个案例。

>假设 A 位于一个团体。一开始团体内只有 A。一旦遇到了一个更大的团体就合并，同时 A 会大喊一声。最多能听到 A 喊几声？

当时的答案是 $O(\log n)$ 次。那如果改成集合中的所有人都喊一声呢？那么答案就是 $O(\sum n_i\log n)\sim O(n\log n)$。这就是启发式合并的原理。虽然对小团体的每个人我们都暴力操作了，但是事实上操作次数并不多。这就是启发式合并的原理。

树上启发式合并的特征是：
 - 没有修改操作；
 - 通过遍历子树、统计信息，得到所有查询的答案。

当一个问题的**统计逻辑依赖于子树信息的集合**（比如你需要知道子树里所有深度的分布、某种颜色的数量、某种数值的出现次数等），而你又无法通过简单的**树上差分**或**线段树合并**（可能因为空间开销大或逻辑太复杂）来解决时，启发式合并就是你的“头号工具”。

我们从下面这道题目入手启发式合并的原理。

>[!example] 树上数颜色
>给一棵根为 1 的树，每次询问子树颜色种类数。
>数据范围：$1\leq q,n\leq 10^5$。

>[!remark]
>很显然这个可以单次 dfs 之后在 dfn 序上跑一个 [[10. 分块]]（[[11. 莫队算法]]）解决，时间复杂度是 $O((n+q)\sqrt{n})$ 的，但是码量超级大，很难写。一个实现如下。

因此我们来思考一下怎么用树上启发式合并解决。树上启发式合并的基本流程如下：
1. 我们开始按照 `void dfs(u, keep)` 的方式递归。`u` 表示当前节点，`keep` 表示是否保留 `u` 对子树信息的贡献。
2. `dfs(轻儿子, 0)`：先遍历所有轻儿子的子树，遍历结束时，彻底消除对信息的贡献。
3. `dfs(son[u], 1)`：再遍历重儿子的子树，遍历结束时，保留信息的贡献。
4. 考察 `u` 对信息进行的贡献。
5. 再遍历轻儿子的子树，对上面的每个节点重新对信息进行贡献。
6. 得到 `u` 的答案。
7. 如果 `keep==0`，消除 `u` 的贡献，反之保留。

>[!thought] 为什么需要两次遍历轻儿子？
>因为要计算所有儿子之间的交互。假如 $u$ 的子树有 $a,b,c$，其中 $a$ 是重儿子，那么两次遍历保证了：
>1. 第一次：递归计算 $b$、$c$ 自己的答案，清除保证了不会有跨子树的影响。 
>2. 第二次：把 $b$、$c$ 的信息加到已有（$a+u$）的集合中，计算跨子树贡献。

>[!thought] 时间复杂度会不会爆炸？
>答案是否定的。
>
>显然，2、3 两步的时间复杂度是 $O(n)$。比较麻烦的在于第 5 点、第 7 点。
>
>一个节点只有在轻儿子里，并且和**重节点遇见过之后**才会重新贡献、取消贡献。然后集合大小倍增。这相当于小集合遇见大集合之后，里面所有节点操作常数次，重链剖分的性质总的时间复杂度是 $O(n\log n)$ 的。
>
>因此，上述所有操作时间复杂度最多是 $O(n\log n)$。

以这个示意图为例，我们看看启发式合并到底是怎么进行的。

我们统计 $\newcommand{\ans}{\mathrm{ans}}\newcommand{\color}{\mathrm{color}}\newcommand{\diff}{\mathrm{diff}}$$\color$ 数组表示当前每种颜色出现的次数。$\diff$ 表示当前出现的不同颜色数量。$\ans$ 表示每个子树的答案。
 - 显然，如果某种颜色`++`之后发现是 1，那么 $\diff$ 就加一；反之亦然。两个变量可以相互配合。这就是我们要维护的信息。

首先先跑一遍重链剖分。

![[Pasted image 20260204234527.png]]

然后开始执行 $dfs$。第一次是 $dfs(a,0)$（或者 $dfs(a,1)$，都可以，因为这时候我们已经获得了完整的 $\ans$ 数组了）。
- 先遍历轻儿子。比如 $g$。
	- 然后遍历 $i$。
		- 此时 $i$ 没有儿子了，所以 $\color_2$ 加一，$\diff=1$。此时得到了 $i$ 的答案，故 $\ans_i=1$。
		- `keep=0`，所以取消了自己的贡献，$\color_2=0$，$\diff=0$，但是 $\ans_i$ 不清零。
	- 回到 $g$，此时遍历 $h$ 这个重儿子。
		- 类似，$\color_2=1,\diff=1,\ans_h=1$。
		- `keep=1`，因此不清零，回到 $g$。
	- 现在考察 $g$ 自身的贡献。现在 $\color_4=1$，$\diff=2$。
		- 现在的信息是 $g+h$ 的，再纳入 $i$ 的贡献，此时 $\color_2=2$，$\diff=2$ 不变。
	- 故而 $\ans_g=2$。
	- `keep=0`，清除所有贡献（重新遍历一遍 $g$ 的子树），$\color_2=0,\color_4=0$，$\diff=0$。

以此类推。这一整个过程是是 $O(n\log n)$ 的。

```cpp
#include <bits/stdc++.h>
using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

const int MAXN = 2e6 + 5;

int n;
int sz[MAXN], son[MAXN], pa[MAXN];

int c[MAXN];
int ans[MAXN], cnt[MAXN], tot;

int head[MAXN], to[MAXN << 1], nxt[MAXN << 1];
int e_cnt;

inline void add_edge(int u, int v) {
    nxt[++e_cnt] = head[u], head[u] = e_cnt, to[e_cnt] = v;
}

inline int read() {
    char ch = getchar();
    int f = 1, ret = 0;
    while (!isdigit(ch)) {
        if (ch == '-') { f *= -1; }
        ch = getchar();
    }
    while (isdigit(ch)) {
        ret = ret * 10 + ch - '0';
        ch = getchar();
    }
    return ret * f;
}

inline void dfs1(int u, int fa) {
    pa[u] = fa, sz[u] = 1;
    for (int e = head[u]; e; e = nxt[e]) {
        int v = to[e];
        if (v == fa) continue;
        dfs1(v, u);
        sz[u] += sz[v];
        if (sz[v] > sz[son[u]]) son[u] = v;
    }
}

inline void dfs3(int u, int op) {
    if (op == 1) {
        cnt[c[u]]++;
        if (cnt[c[u]] == 1) tot++;
    } else {
        cnt[c[u]]--;
        if (cnt[c[u]] == 0) tot--;
    }
    for (int e = head[u]; e; e = nxt[e]) {
        int v = to[e];
        if (v == pa[u]) continue;
        dfs3(v, op);
    }
}

inline void dfs2(int u, bool keep) {
    for (int e = head[u]; e; e = nxt[e]) {
        int v = to[e];
        if (v == son[u] || v == pa[u]) continue;
        dfs2(v, 0);
    }
    if (son[u]) dfs2(son[u], 1);
    cnt[c[u]]++;
    if (cnt[c[u]] == 1) tot++;
    for (int e = head[u]; e; e = nxt[e]) {
        int v = to[e];
        if (v == son[u] || v == pa[u]) continue;
        dfs3(v, 1);
    }
    ans[u] = tot;
    if (!keep) { dfs3(u, -1); }
}

int main() {
    // cin.tie(nullptr)->sync_with_stdio(false);
    n = read();
    for (int i = 1; i < n; i++) {
        int u, v;
        u = read(), v = read();
        add_edge(u, v), add_edge(v, u);
    }
    for (int i = 1; i <= n; i++) c[i] = read();
    int q = read();
    dfs1(1, 0);
    dfs2(1, 0);
    while (q--) {
        int u = read();
        printf("%d\n", ans[u]);
    }
    return 0;
}
```

下面这道题是贪心转化之后的启发式合并。

>[!example] CF2188E
>
>题目链接：[CF2188E](https://codeforces.com/contest/2188/problem/E)。
>
>Jerry 和 Tom 在一个有向图 $G$ 上玩游戏，图 $G$ 有 $n$ 个顶点，编号为 $1$ 到 $n$。对每个顶点 $1 \le u < n$，都有一条从 $u$ 指向 $u+1$ 的有向边。此外，还有 $m$ 条额外的有向边。第 $i$ 条额外边从 $u_i$ 指向 $v_i$，其中 $1 \le u_i < v_i \le n$。
>
>图 $G$ 具有如下特殊性质：**不存在**两条有向边 $(u_i\to v_i)$ 和 $(u_j\to v_j)$，使得 $u_i < u_j < v_i < v_j$。
>
>游戏开始时，Jerry 和 Tom 分别站在顶点 $x$ 和 $y$ 上，且 $x \ne y$。游戏轮流进行，Jerry 先走，然后 Tom 走，每轮两位玩家按照以下规则行动：
>
>-   Jerry **必须**从他当前顶点选择一条出边，并沿该边移动到终点。如果他当前顶点没有出边，则留在原地。
>-   Tom **可以**从他当前顶点选择一条出边并沿该边移动到终点，**或者**选择不移动而留在原地。
>
>如果在任何一轮结束后，Jerry 和 Tom 位于同一个顶点（**包括顶点 $n$**），则游戏立即结束且 Tom 获胜。否则，如果 Jerry 初始时就在顶点 $n$，或在任何一轮结束后到达顶点 $n$，则 Jerry 获胜。
>
>注意，如果在一轮之后，Jerry 和 Tom 都位于顶点 $n$，那么 **Tom 获胜**。
>
>在整个游戏中，**双方都知道**对方的位置。
>
>可以证明，游戏会在有限轮内结束。
>
>对于一对整数 $1 \le x,y \le n$，$x \ne y$，定义 $f(x,y)$ 如下：
>
>-   Jerry 和 Tom 将进行一局游戏，Jerry 起始于顶点 $x$，Tom 起始于顶点 $y$。Tom 希望获胜，但同时希望最小化他实际移动的次数（即他改变顶点的轮数；**停留在原地不计为一次移动**）。假设双方都采取最优策略，若 Tom 无法获胜，则令 $f(x,y)=0$；否则，令 $f(x,y)$ 为 Tom 为了迫使获胜所需的最小移动次数。如果 Tom 在最优对局下获胜，Jerry 仍会试图最大化 Tom 必须移动的次数。
>
计算
>$$
\sum\limits_{1 \le x,y \le n, x \ne y} f(x,y).
$$

我们来思考双方的最优策略。

Tom 的最优策略肯定是不断选取更深的边去跑，这样他总能够提前拦截 Jerry。题目保证没有交叉的边，因此这样子做 Jerry 无论怎么跑，如果 Tom 能够抓住 Jerry，那么总能够提前到达两者的 LCA。然后 Jerry 无论怎么动，最终总会到达这个 LCA$\newcommand{\dep}{\mathrm{dep}}$，Tom 只用在 LCA 处等待即可。因此我们发现
$$
f(x,y)=\begin{cases}\dep_y-\dep_{\operatorname{LCA}(x,y)},&\dep_x\geq\dep_y,\\
0,&\text{otherwise.}\end{cases}
$$

问题变成了计算
$$
\sum\limits_{1 \le x,y \le n,\dep_x\geq\dep_y, x \ne y} \dep_y-\dep_{\operatorname{LCA}(x,y)}.
$$
这当然可以用启发式合并来计算，因为我们可以通过遍历 $u$ 作为 $\operatorname{LCA}(x,y)$ 的子树信息来获得答案。

考虑枚举 $u=\operatorname{LCA}(x,y)$。我们现在来计算这个子树内的值。注意到把 $x\neq y$ 删掉其实不影响结果，我们开始推式子。考虑引入顺序。
$$
\begin{aligned}
\sum\limits_{x,y\in T_u,\dep_x\geq\dep_y} \dep_y-\dep_{u}=&\sum\limits_{x<y,\dep_x\neq\dep_y} \min\{\dep_x,\dep_y\}-\dep_{u}\\&+\sum\limits_{x<y,\dep_x=\dep_y} \dep_y-\dep_{u}.
\end{aligned}
$$
固定 $y$，我们可以通过树状数组快速查询两个部分的值。

因此，整个过程时间复杂度是 $O(n\log^2n)$。

代码如下：

```cpp
#include <atcoder/fenwicktree>
#include <bits/stdc++.h>
using namespace atcoder;
using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

const int MAXN = 2e5 + 5;

int n, m;
int to[MAXN];
vi g[MAXN];
int sz[MAXN], son[MAXN];
int dep[MAXN], pa[MAXN];
ll ans;

fenwick_tree<ll> bit_sum, bit_cnt;

void calculate(int x, int u) {
    ans += bit_sum.sum(0, dep[x]) - bit_cnt.sum(0, dep[x]) * dep[u];
    ans += bit_cnt.sum(dep[x], n + 1) * (dep[x] - dep[u]);
    ans += bit_cnt.sum(dep[x], dep[x] + 1) * (dep[x] - dep[u]);
}

void update(int d, int cnt) {
    bit_cnt.add(d, cnt);
    bit_sum.add(d, cnt * d);
}

void dfs1(int u, int fa) {
    sz[u] = 1, pa[u] = fa;
    dep[u] = dep[fa] + 1;
    for (int v : g[u]) {
        if (v == fa) continue;
        dfs1(v, u);
        sz[u] += sz[v];
        if (sz[v] > sz[son[u]]) son[u] = v;
    }
}

// 这里的 skip_node 是为了防止在合并轻儿子时，重复计算已经存在的重儿子
int skip_node = 0;

void add_to_bit(int u, int op) {
    update(dep[u], op);
    for (int v : g[u]) {
        if (v == pa[u] || v == skip_node) continue;
        add_to_bit(v, op);
    }
}

void calc_subtree(int u, int lca) {
    calculate(u, lca);
    for (int v : g[u]) {
        if (v == pa[u] || v == skip_node) continue;
        calc_subtree(v, lca);
    }
}

void dfs2(int u, bool keep) {
    for (int v : g[u]) {
        if (v != pa[u] && v != son[u]) dfs2(v, false);
    }
    if (son[u]) dfs2(son[u], true);

    calculate(u, u); // 根节点自己
    update(dep[u], 1);

    skip_node = son[u];
    for (int v : g[u]) {
        if (v != pa[u] && v != son[u]) {
            // 重点：先整棵子树算贡献，再整棵子树入桶
            calc_subtree(v, u);
            add_to_bit(v, 1);
        }
    }
    skip_node = 0;

    if (!keep) add_to_bit(u, -1);
}

void solve() {
    cin >> n >> m;
    bit_cnt = fenwick_tree<ll>(n + 1);
    bit_sum = fenwick_tree<ll>(n + 1);
    ans = 0;
    fill(all(to), 0);
    for (int i = 1; i < n; i++) {
        to[i] = i + 1;
        g[i].clear();
    }
    g[n].clear();
    fill(all(sz), 0), fill(all(son), 0), fill(all(pa), 0);
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        to[u] = max(to[u], v);
    }
    for (int i = 1; i <= n; i++) { g[to[i]].emplace_back(i); }
    dfs1(n, 0);
    dfs2(n, 0);
    cout << ans << endl;
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    int t;
    cin >> t;
    while (t--) { solve(); }
    return 0;
}
```

# 关于维护数据的一点小巧思

>[!example] CF600E
>一棵树有 $n$ 个结点，每个结点都是一种颜色，每个颜色有一个编号，求树中每个子树的最多的**颜色编号**的和。

很显然我们期待维护一个 mx，但是这个 mx 是**子树相关**的，因此需要清除！

而且如果直接 `unordered_map` 然后直接遍历的话会超时，因此我们需要动态维护出现最多的颜色编号。

可以学习下面这个代码。

```cpp
#include <bits/stdc++.h>
using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;

const int MAXN = 1e5 + 5;

int n;
int head[MAXN], nxt[MAXN << 1], to[MAXN << 1];
int e_cnt;

void add_edge(int u, int v) {
    nxt[++e_cnt] = head[u], to[e_cnt] = v, head[u] = e_cnt;
}

int pa[MAXN], sz[MAXN], son[MAXN];

void dfs1(int u, int fa) {
    pa[u] = fa, sz[u] = 1;
    for (int e = head[u]; e; e = nxt[e]) {
        int v = to[e];
        if (v == fa) continue;
        dfs1(v, u);
        sz[u] += sz[v];
        if (sz[v] > sz[son[u]]) son[u] = v;
    }
}

ll ans[MAXN];
ll sum;
int c[MAXN], cnt[MAXN];
int mx;

// 动态维护最大值
void add_color(int d) {
    cnt[d]++;
    if (cnt[d] > mx) {
        sum = d, mx = cnt[d];
    } else if (cnt[d] == mx) {
        sum += d;
    }
}

void dfs3(int u, int op) {
    if (op == 1) {
        add_color(c[u]);
    } else {
        cnt[c[u]]--;
    }
    for (int e = head[u]; e; e = nxt[e]) {
        int v = to[e];
        if (v == pa[u]) continue;
        dfs3(v, op);
    }
}

void dfs2(int u, bool keep) {
    for (int e = head[u]; e; e = nxt[e]) {
        int v = to[e];
        if (v == son[u] || v == pa[u]) continue;
        dfs2(v, 0);
    }
    if (son[u]) dfs2(son[u], 1);
    add_color(c[u]);
    for (int e = head[u]; e; e = nxt[e]) {
        int v = to[e];
        if (v == son[u] || v == pa[u]) continue;
        dfs3(v, 1);
    }
    ans[u] = sum;
    if (!keep) {
        dfs3(u, -1);
        mx = 0; // 只有在轻子树的时候才清空。而且根据遍历顺序，这个不会影响最终答案。
        sum = 0;
    }
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    cin >> n;
    for (int i = 1; i <= n; i++) cin >> c[i];
    for (int i = 1; i < n; i++) {
        int u, v;
        cin >> u >> v;
        add_edge(u, v), add_edge(v, u);
    }
    dfs1(1, 0);
    dfs2(1, 0);
    for (int i = 1; i <= n; i++) { cout << ans[i] << ' '; }
    return 0;
}
```
