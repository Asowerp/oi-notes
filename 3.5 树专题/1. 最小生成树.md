#树 #最小生成树

最小生成树（MST）解决的问题是，在无向带权图中选择一些边，在连通性的情况下，权值最小。

对于图 $(V,E)$，我们下面默认 $n=|V|, m=|E|$。

打个比方，下面这是一个 $n=5$ 的无向带权图。

![[Pasted image 20260122183001.png]]

在保证连通性的情况下，最少只用 $n-1=4$ 条边，就可以保证连通性。并且最后必然是一颗树。反之，若存在一个环，那必然可以删掉一个边使得权值更小。

![[Pasted image 20260122183131.png]]

枚举不难发现上面这棵树就是我们需要的最小生成树。

并且，最小生成树不止 1 种。

可以证明，最小生成树一定是最小瓶颈树，即最大边权最小的树。
# Kruskal 算法

Kruskal 算法是最常见、最经典的求最小生成树的算法。Kruskal 算法的优点是不必建图，只用并查集就能解决问题。

Kruskal 算法的步骤如下：
1. 把权值从小到大排序。从权值小的边开始考察。
2. 如果连接当前的边不会成环，就选择当前的边；反之，丢弃该边。
重复以上步骤就得到了最终的最小生成树。

>[!hint]- 联想
>上面的做法不由得让人联想到 [关押罪犯](https://www.luogu.com.cn/problem/P1525) 这道题目，在这道题目中我们也是先根据权值排序，然后去思考每个罪犯能不能关一起（从大到小考虑，能放两边就放两边，反之必然撞）。AC 代码参看 [提交记录](https://www.luogu.com.cn/record/239858075)。

上面的做法为什么是正确的呢？可以参看下面的证明。

>[!proof]- 证明
>Kruskal 算法的思路很简单，为了造出一棵最小生成树，我们从最小边权的边开始，按边权从小到大依次加入，如果某次加边产生了环，就扔掉这条边，直到加入了 $n-1$ 条边，即形成了一棵树。
>
>我们接下来使用归纳法，证明任何时候 Kruskal 算法选择的边集都被某棵 MST 所包含。
>
>对于算法刚开始时，显然成立（最小生成树存在）。
>
>假设某时刻成立，当前边集为 $F$，令 $T$ 为这棵 MST，考虑下一条加入的边 $e$。
>
>1. $e\in T$，结论显然。
>2. $e\notin T$，则 $T+e$ 必然存在一个环。考察 $f\in T+e$ 在环上且 $f\notin F$（至少存在一条，比如 $e$）。那么 $f$ 的权值必然大于等于 $e$，否则 $f$ 先于 $e$ 被考虑；其次，$f$ 的权值必然小于等于 $e$，否则 $T+e-f$ 是一个更优的生成树。这表明 $T+e-f$ 是一颗包含 $F$ 的 MST。
>
>结论递推直到 $|F|=n-1$，因此 Kruskal 算法选择出来的一定是最小生成树。

这样就得到了一个时间复杂度是 $O(m\log m)$ 的最小生成树算法，且瓶颈在排序，搜索的过程是 $O(m\alpha(m))$ 的，其中 $\alpha(\cdot)$ 为反 Ackermann 函数。

```cpp
sort(all(edge));
dsu uf(n);
int ans = 0;
for (auto &[cost, u, v] : edge) {
	if (uf.same(u, v)) continue;
	ans += cost;
	uf.merge(u, v);
}
```

# Prim 算法

Prim 算法比 Kruskal 稍微复杂一点。在极限情况下，Prim 的理论复杂度更优，但实际上也不一定。因此基本上采用 Kruskal 算法。

Prim 算法步骤如下：
1. “解锁”的点存到集合 $S$ 里面，“解锁”的边存到小根堆 $H$ 里面。
2. 从任意点开始，开始点加入 $S$，开始点的所有边加入 $H$。
3. 弹出 $H$ 顶 $e$，去向 $x$。
	1. 如果 $x\in S$，则继续；
	2. 反之，$e$ 加入生成树，把 $x$ 的所有边加入 $H$，重复步骤 3。

这样做时间复杂度也是 $O(m\log m)$。那难道这个算法纯飞舞吗？对。

但是，可以有优化。通过优化可以做到 $O((m+n)\log n)$，如果采用 Fibonacci 堆可以进一步做到 $O(n\log n+m)$。

回顾上面的过程，我们期望堆的大小和 $n$ 相关。依旧 $S,H$。但是 $H$ 中的元素是 $(v,c)$，即(节点，到达节点花费)。按照花费排序。
1. 弹出 $H$ 堆顶 $(u,c)$。$c$ 加到总权重，考察从 $u$ 出发的边 $u\overset{w}{\to}v$。
	1. 如果 $v\in S$，忽略该边。
	2. 若 $H$ 中没有到达 $v$ 的边，往 $H$ 中添加 $(v,w)$。
	3. 若 $(v,w')\in H$，更新为 $(v,\min(w,w'))$。
2. 重复上述步骤直到 $H$ 为空。

然而注意到直接修改 $H$ 中的元素非常麻烦，我们另外维护一个 `dist` 数组，表示所有 $S$ 中元素到达 $v$ 的最小距离。这样我们就实现了优化。一个伪代码如下：

```py
S = set()  # 已选点
H = MinHeap()  # 存所有从S出发的边

dist = [INF] * n  # dist[v]: 从S到v的最小边权
parent = [-1] * n  # parent[v]: 连接到v的S中的点
in_S = [False] * n

# 选起点 s
dist[s] = 0
H.push((0, s))  # (权值, 顶点)

while not H.empty():
    _, u = H.pop_min()
    if in_S[u]: continue
    
    in_S[u] = True
    # 边 (parent[u], u) 加入生成树
    
    # 更新 u 的邻居
    for v, weight in u的所有邻居:
        if not in_S[v] and weight < dist[v]:
            dist[v] = weight
            parent[v] = u
            H.push((weight, v))  # 或 H.decrease_key(v, weight)
```

这个贪心的正确性也是类似的。

## 利用生成树解决游走问题

我们来看下面这道题。（可能需要 [[4. 线性基]] 的知识）

>[!example] ABC 451 G
>给定一个 $N$ 个顶点的连通无向图，定义两点间“行走”的权重为路径上所有边权的**异或和**（允许重复经过边）。请找出有多少对顶点 $(x, y)$（其中 $x < y$），使得它们之间所有可能行走中的**最小异或权重**不超过给定的整数 $K$。
>
>数据范围：$2 \leq N \leq 2 \times 10^5$，$N-1 \leq M \leq 2 \times 10^5$，$0 \leq K < 2^{30}$，边权 $0 \leq W_i < 2^{30}$。

这题用到了关于游走的一个比较典的性质。

>[!theorem] 性质
>在无向图中，任意两点 $x$ 和 $y$ 之间的任意一条游走路径的异或和，都可以表示为：
>$$\text{任意路径的异或和} = S_x \oplus S_y \oplus (\text{若干个环的异或和})$$
>其中，$S_i$ 是任意一个生成树中，$i$ 到根的异或和。

这是基于一个更加典的结论：我们在树上任意两个点的简单路径的异或等于两个顶点到达根的路径的异或值的异或。而更进一步，图中的每一条**非树边（返祖边）** $(u, v)$，权重为 $w$，都会与生成树上的路径构成一个环，这个环的异或和就是 $S_u \oplus S_v \oplus w$。

你可能会疑问：我这些环可能不在我的路径上啊？这是容易的，我们过去再回来就把过去的这一段路径的异或给异或掉了。

所以我们只用把所有返祖边都插入一个异或基就好。接下来，我们让每个节点的树上异或和 $S_i$，去和线性基中的元素进行异或，尽可能地把它消到最小。我们将消完后的最小值记为 $V_i$。根据线性基的性质，如果用线性基从高位到低位贪心地去最小化 $S_i$ 得到 $V_i$，那么 $V_i$ 在线性基主元（最高位为 1 的位）所在的位置必定是 0。

既然 $V_x$ 和 $V_y$ 在这些位置都是 0，那么 $V_x \oplus V_y$ 在这些位置也都是 0。这意味着，无论你再怎么加上线性基里的元素（相当于在图中绕环），都会把原本是 0 的高位变成 1，反而会使得结果变大。

因此，两点 $x$ 和 $y$ 之间游走路径的最小异或和，就绝对等于 $V_x \oplus V_y$！

现在问题就转化为，给定 $\{V_i\}$，我要怎么查询 $V_i\oplus V_j\leq K$ 的 $(i,j)$ 对数了。这时候我们不难想到 0-1 Trie。我们依次去查询和当前 $V_i$ 异或小于等于 $K$ 的个数（按照 $K$ 这一位是否是 1 分类），然后插入 0-1 Trie 即可。

总的时间复杂度就是 $O(n\log V)$。

```cpp
#include <bits/stdc++.h>

using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using i64 = long long;
using u64 = unsigned long long;
using i128 = __int128_t;
using u128 = __uint128_t;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

const int MAXN = 2e5 + 5;
vector<vector<pii>> graph;
bitset<MAXN> vis;
vi s;

class Basis {
    vector<int> p;

  public:
    Basis() : p(31, 0) {}
    void clear() { p.assign(31, 0); }
    void insert(int x) {
        for (int i = 30; i >= 0 && x; i--) {
            if ((x >> i) & 1) {
                if (!p[i]) {
                    p[i] = x;
                    break;
                } else {
                    x ^= p[i];
                }
            }
        }
    }
    int get() {
        int ret = 0;
        for (int i = 30; i >= 0; i--) {
            if (p[i]) {
                ret = max(ret, ret ^ p[i]);
            }
        }
        return ret;
    }
    int get_min(int x) {
        for (int i = 30; i >= 0; i--) {
            if (((x >> i) & 1) && p[i]) {
                x ^= p[i];
            }
        }
        return x;
    }
    bool span(int x) {
        for (int i = 30; i >= 0; i--) {
            if ((x >> i) & 1) {
                if (!p[i])
                    return false;
                x ^= p[i];
            }
        }
        return true;
    }
} bs;

void dfs(int u, int fa, int c) {
    vis[u] = true;
    s[u] = s[fa] ^ c;
    for (auto [v, w] : graph[u]) {
        if (vis[v]) {
            bs.insert(s[u] ^ s[v] ^ w);
        } else {
            dfs(v, u, w);
        }
    }
}

class Trie {
    struct Node {
        int cnt;
        int son[2];
        Node() : cnt(0) { son[0] = son[1] = -1; }
    };
    vector<Node> tree;
    int cnt = 0;

  public:
    Trie() {}
    void assign(int n) {
        tree.assign(n * 32, Node());
        cnt = 0;
    }
    int query(int x, int k) {
        int cur = 0;
        int ret = 0;
        for (int i = 30; i >= 0; i--) {
            if (cur == -1)
                break;
            if ((k >> i) & 1) {
                int nxt = ((x >> i) & 1);
                if (tree[cur].son[nxt] != -1) {
                    ret += tree[tree[cur].son[nxt]].cnt;
                }
                cur = tree[cur].son[nxt ^ 1];
            } else {
                int nxt = ((x >> i) & 1);
                cur = tree[cur].son[nxt];
            }
        }
        if (cur != -1)
            ret += tree[cur].cnt;
        return ret;
    }
    void insert(int x) {
        int cur = 0;
        for (int i = 30; i >= 0; i--) {
            int nxt = (x >> i) & 1;
            if (tree[cur].son[nxt] == -1)
                tree[cur].son[nxt] = ++cnt;
            cur = tree[cur].son[nxt];
            tree[cur].cnt++;
        }
    }
} t;

void solve() {
    int n, m, k;
    cin >> n >> m >> k;
    graph.assign(n + 1, vector<pii>());
    s.assign(n + 1, 0);
    bs.clear();
    vis.reset();
    t.assign(n);
    for (int i = 1; i <= m; i++) {
        int u, v, w;
        cin >> u >> v >> w;
        graph[u].emplace_back(v, w);
        graph[v].emplace_back(u, w);
    }
    dfs(1, 0, 0);
    i64 ans = 0;
    for (int i = 1; i <= n; i++) {
        int tmp = bs.get_min(s[i]);
        ans += t.query(tmp, k);
        t.insert(tmp);
    }
    cout << ans << '\n';
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    int t;
    cin >> t;
    while (t--) {
        solve();
    }
    return 0;
}
```
