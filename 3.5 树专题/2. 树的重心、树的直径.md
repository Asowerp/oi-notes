# 树的重心

树 $T$ 的重心 $g$ 有如下三种等价的定义：
1. 以节点 $g$ 为根，最大子树的节点数最少；
2. 以节点 $g$ 为根，每棵子树的节点数均不超过 $\left\lfloor\dfrac{n}{2}\right\rfloor$；
3. 以节点 $g$ 为根，所有节点走向 $g$ 的总边数最小。（更像性质，事实上这是性质 4 的特殊情况）

![[Pasted image 20260124173505.png]]

重心还有以下性质：
1. 一棵树 $T$ 有至多两个重心。当 $T$ 有两个重心的时候，两个重心必然相邻；
2. 树上增加 / 删除一个叶节点，转移后的重心至多移动一条边。
3. 连接两棵树得到的新树的重心在原先两棵树的重心之间的路径上。
4. 树上边权若均非负，则无论边权如何分布，所有节点走向重心的距离和最小。

以上的这些性质在后续的 [[7. 点分治、边分治]] 中依然有用。

那重心要怎么求呢？

如果按照第一种定义来求，那么有下面的 $O(n)$ 做法。

![[Pasted image 20260124182519.png]]

我们从 1 开始遍历，现在来到 $u$，想求 $u$ 底下节点个数。这个用 dfs 可以 $O(n)$ 维护一个 `size` 数组。$\newcommand{\size}{\mathrm{size}}$

比如这里，我们不妨粗略记 $\size_u[v]$ 表示以 $u$ 为根节点，$v$ 开始的子树大小，那么 $\size_u[a]=3,\size_u[b]=3,\size_u[c]=2$。在这个基础上我们还想要 $\size_u[f]$。显然 $\size_u[u]=9$，所以 $\size_u[f]=n-\size_u[u]$。因此来到了任何一个节点都可以算出来最大子树的大小，这样筛选一个最小值，就可以得到重心。

上述所有操作都可以 dfs 以 $O(n)$ 的时间复杂度完成。一个 java 示例代码如下。

```java
public static void dfs(int u, int f) {
    size[u] = 1;
    int maxsub = 0; // 以当前节点 u 为根节点，最大子树的大小 
    for (int e = head[u], v; e != 0; e = next[e]) {
        v = to[e];
        if (v != f) {
            dfs(v, u);
            size[u] += size[v];
            maxsub = Math.max(maxsub, size[v]);
        }
    }
    maxsub = Math.max(maxsub, n - size[u]);
    if (maxsub < best || (maxsub == best && u < center)) { // 题目要求的筛选 center
        best = maxsub;
        center = u;
    }
}
```

按照第二种方式，依旧可以 $O(n)$ 求。只需要把上面的 `maxsub<best` 改成 `maxsub*2<n` 即可。这是因为每棵子树的节点数均不超过 $\left\lfloor\dfrac{n}{2}\right\rfloor$ 的充要条件是最大子树节点数不超过 $\left\lfloor\dfrac{n}{2}\right\rfloor$。按这种求法，我们甚至都不需要一个 best 变量。
（或者可以存储 `maxsub[u]`，最后遍历一遍，不改变复杂度为 $O(n)$ 的事实）

上面都是没有点权的树。如果有点权，我们应该怎么求重心呢？可以猜想，如果点权均为 1，那退化成了一开始的形式，也就是求 size 的时候，和第一、二种应该是一样的。我们不妨大胆猜测，两种做法在把 `size` 换成点权（$c_i$）之和，$n$ 换成 $\sum c_i$ 之后仍然成立。事实也应该如此。带点权 $c_i$ 的节点相当于这个节点重复了 $c_i$ 次！因此感性理解上，用上述两种方法仍然可以解决这个题目。因此依旧 $O(n)$ 解决。

我们接下来再跑一次 $O(n)$ 的 dfs，收集从重心到达每个顶点的距离，这样就可以计算出来距离之和。求和又是一个 $O(n)$，因此整体复杂度是 $O(3n)\sim O(n)$。

# 树的直径

树上距离最远的两个点之间的路径就叫做树的直径。

树的直径有 2 种求法
 - 跑两次 dfs（适用边权非负）；
	 - 优点：可以得到更多信息；
	 - 缺点：要跑两次，局限性比较大。
 - 树形 dp（通用）
	 - 优点：只用跑一次。
	 - 缺点：只能得到直径长度。

如果边权都为正，那么有以下两个性质：
1. 如果有多条直径，那么这些直径一定拥有共同的部分（顶点 / 边 / 路径）；
2. 树上任意一点，相隔最远的点的集合，其中一定有一个点在直径端点中。
	- 这个性质是在说，假设直径是 $a-b$，从任意一个 $x$ 出发，可能有 $y,z,w$ 离 $x$ 最远。设这些点构成集合为 $S$，则必然有 $S\cap\{a,b\}$ 非空。

## 跑两次 dfs

![[Pasted image 20260124194831.png]]

以这棵树为例。

第一次 dfs，我们需要找到离 $a$ 最远的节点 $x$。即，维护一个 dist 数组。

第二次 dfs，我们从 $x$ 出发 dfs，找到离 $x$ 最远的点。这个路径就是树的直径。

![[Pasted image 20260125163708.png]]

然而，这种图，上面的两次 dfs 就失效了。我们跑两次 dfs 会得到 a-e 为直径（2），然而 c-d（11）显然大于 a-e。

具体实现如下。回到一开始的例子，我们需要 `dist` 和 `last` 两个数组。$\newcommand{\dist}{\operatorname{dist}}$$\newcommand{\last}{\operatorname{last}}$$\dist[i]$ 表示从当前头节点到 $i$ 的长度；$\last[i]$ 表示当前头到 $i$ 的过程中，$i$ 的父节点。
 - 在第一次 dfs 的过程中，我们更新 $\dist$ 数组，找到第二次 dfs 的头节点（直径的一个端点）；
 - 第二次 dfs 的过程中，我们更新 $\dist, \last$ 数组，找到直径的另一个端点。
这两次 dfs 可以统一用一个函数来实现。

>[!remark] 注
>事实上，两次 bfs 也可以替代上面的过程，然而实操中两次 dfs 更常见。

## 树形 dp

我们维护两个数组：`dist` 和 `ans`$\newcommand{\ans}{\operatorname{ans}}$。$\dist[u]$ 表示从 $u$ 开始必须往下走，能够走出的最大距离（可以不选择任何边）；$\ans[u]$ 表示路径一定包含 $u$，且限制在 $u$ 及其子树的时候，最大路径的长度。
 - 对每个子节点 $son$，
	 - 先更新 $\ans[u]=\max\{\ans[u],\dist[u]+\dist[son]+c\}$。
	 - 再更新 $\dist[u]=\max\{\dist[u],\dist[son]+c\}$。
为什么？$\ans$ 的更新模式表明 $u$ 出发可以有 2 个分支，并且先更新避免了同一条路径算两次的问题。$\dist$ 则如字面所示。

而这样更新，会不会出现 $u$ 往上走结果没更新到的问题呢？答案是否定的。我们不妨考虑两个端点分别是 $a,b$，则从 $c=\operatorname{LCA}(a,b)$ 往下的更新必然会将 $\ans[c]$ 更新为直径的长度。

两个的时间复杂度都是 $O(m)$。

模板题：[U81904 【模板】树的直径](https://www.luogu.com.cn/problem/U81904)。下面的代码中，`#if false` 部分是两次 dfs；`#else` 往后是树形 dp。

```cpp
#if false

#include <bits/stdc++.h>

using namespace std;

const int MAXN = 500000 + 5;

int n;
vector<pair<int, int>> e[MAXN];
int dist[MAXN], last[MAXN];
int start, end;

void dfs(int u, int fa) {
    for (auto [v, c] : e[u]) {
        if (v == fa) continue;
        dist[v] = dist[u] + c;
        last[v] = u;
        dfs(v, u);
    }
}

int main() {
    cin >> n;
    for (int i = 1; i < n; i++) {
        int a, b, c;
        cin >> a >> b >> c;
        e[a].emplace_back(b, c);
        e[b].emplace_back(a, c);
    }
    dfs(1, 0);
    for (int i = 1; i <= n; i++) {
        if (dist[i] > dist[start]) start = i;
    }
    dist[start] = 0;
    dfs(start, 0);
    for (int i = 1; i <= n; i++) {
        if (dist[i] > dist[::end]) ::end = i;
    }
    cout << dist[::end] << endl;
    return 0;
}
#else

#include <bits/stdc++.h>

using namespace std;

const int MAXN = 5e5 + 5;

int n;
vector<pair<int, int>> e[MAXN];
int dist[MAXN], ans[MAXN];

void dfs(int u, int fa) {
    dist[u] = 0;
    for (auto [v, c] : e[u]) {
        if (v == fa) continue;
        dfs(v, u);
        ans[u] = max(ans[u], dist[u] + dist[v] + c);
        dist[u] = max(dist[u], c + dist[v]);
    }
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    cin >> n;
    for (int i = 1; i < n; i++) {
        int a, b, c;
        cin >> a >> b >> c;
        e[a].emplace_back(b, c);
        e[b].emplace_back(a, c);
    }
    dfs(1, 0);
    int ret = 0;
    for (int i = 1; i <= n; i++) { ret = max(ret, ans[i]); }
    cout << ret << endl;
    return 0;
}

#endif
```
