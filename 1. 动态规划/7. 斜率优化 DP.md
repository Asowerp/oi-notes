#DP #优化DP 

我们从下面这道题目开始。

>[!example]
>给你一个整数数组 `nums` 和一个整数 `k`。
>
>你的任务是将 `nums` 分割成 恰好 `k` 个子数组，并返回所有有效分割方案中最小可能的分数。
>
>一个分割方案的分数是其所有子数组值的 总和。
>
>子数组的值定义为 sumArr * (sumArr + 1) / 2，其中 sumArr 是该子数组元素的总和。
>
>子数组 是数组中连续的非空元素序列。
>
>$n,k\leq 10^3$。

首先第一反应肯定是 dp。我们令 $dp[i][j]$ 表示进行到 $i$ 分了 $j$ 段的最小价值。于是很容易写出转移方程为
$$
dp[i][j]=\min_{k=j-1}^{i-1}\left\{dp[k][j-1]+f\left(\sum_{p=k+1}^ia_p\right)\right\}.
$$
其中 $f(x)=\dfrac{x(x+1)}{2}$。简单思考发现这应该是个 $O(n^2k)$ 的转移，显然不太能过这道题。

>[!hint]
>当 dp 转移中出现**凸函数**，且变量是前缀和的差时，很可能可以优化。

因此我们考虑优化。引入前缀和 $s_p=\sum_{k=1}^pa_k$。那么
$$
\begin{aligned}
dp[i][j]&=\min_{k=j-1}^{i-1}\left\{dp[k][j-1]+f\left(\sum_{p=k+1}^ia_p\right)\right\}\\
&=\min_{k=j-1}^{i-1}\left\{dp[k][j-1]+f\left(s_i-s_k\right)\right\}\\
&=\frac{s_i^2+s_i}{2}+\min_{k=j-1}^{i-1}\left\{dp[k][j-1]+\frac{s_k^2-s_k}{2}-s_ks_i\right\}.
\end{aligned}
$$
注意看，如果我们认为 $s_i$ 是变量的话，那么 $\min$ 里面的式子就是一次函数——能不能进行优化？

这就是我们要介绍的优化方法：斜率优化（凸包技巧）。

## 初步优化

考虑一次函数的表达式 $y=mx+b$。我们认为我们这里是关于 $i$ 的查询，因此我们把与 $i$ 有关的信息表示为 $x$，因此同时和 $i,j$ 有关的就是这里的 $mx$，仅与 $j$ 有关的就是这里的 $b$。

本题中，就是说
$$
m=-s_k,\quad b=dp[k][j-1]+\frac{s_k^2-s_k}{2}.
$$
于是我们要最小化的就是 $mx+b$ 在 $x=s_i$ 处的取值。感性理解可以发现，我们是期望所有的直线构成一个“凸壳”状的结构。

我们进行以下两个观察：
 - 斜率 $-s_k$ 是单调的；
 - 自变量 $s_i$ 是单调的。

因此，我们完全可以通过单调队列来实现一个凸壳的维护。具体地，我们分以下两个步骤维护：
1. 维护凸性。假设我们现在要加入一条新直线。因为这个新直线更陡，什么情况下我们把队尾弹出呢？（显然队列至少要有两个元素我们才讨论凸性）
	- 假设 $l_1=\texttt{dq[n-2]},l_2=\texttt{dq.back()}$，我们现在要加入 $l_3$。什么时候把 $l_2$ 踢掉呢？
	- 不难发现，在我们这样维护的情况下，只要 $l_1\cap l_2$ 的横坐标是大于等于 $l_2\cap l_3$ 的，这说明 $l_2$ 始终在 $l_1,l_3$ 的上方，进而不可能取到最小值，我们就把它踢掉。
2. 维护最小。现在我们要看 $x$ 处的取值。什么时候把队头踢掉呢？
	- 如果队头两条直线 $l_1=\texttt{dq[0]}$ 和 $l_2=\texttt{dq[1]}$ 在 `x` 处的值满足 $l_1(x)\geq l_2(x)$，这说明 $l_1$ 之后一直大于等于 $l_2$，不可能取到最小值，我们就把他踢掉。

这样我们就可以写出本题的代码了。

```cpp
class Solution {
public:
    long long minPartitionScore(vector<int>& nums, int k) {
#define int long long
        int n = nums.size();
        vector<int> sum(n + 1, 0);
        for (int i = 0; i < n; i++) {
            sum[i + 1] = sum[i] + nums[i];
        }
        vector<vector<int>> dp(n + 1, vector<int>(2, 0));

        auto get = [](int x) { return x * x + x >> 1; };
        for (int i = 1; i <= n; i++) {
            dp[i][1] = get(sum[i]);
        }
        for (int i = 2; i <= k; i++) {
            deque<pair<int, int>> dq;
            for (int j = i; j <= n; j++) {
                int m = -sum[j - 1],
                    b = dp[j - 1][i & 1 ^ 1] + get(sum[j - 1]) - sum[j - 1];
                while (dq.size() >= 2) {
                    auto l1 = dq[dq.size() - 2], l2 = dq.back();
                    if ((l2.second - l1.second) * (l2.first - m) >=
                        (l2.first - l1.first) * (l2.second - b)) {
                        dq.pop_back();
                    } else
                        break;
                }
                dq.emplace_back(m, b);
                while (dq.size() >= 2) {
                    auto l1 = dq.front(), l2 = dq[1];
                    if (l1.first * sum[j] + l1.second >=
                        l2.first * sum[j] + l2.second)
                        dq.pop_front();
                    else
                        break;
                }
                dp[j][i & 1] =
                    get(sum[j]) + dq[0].first * sum[j] + dq[0].second;
            }
        }
        return dp[n][k & 1];
#undef int
    }
};
```

## 另一种优化

这种优化也要满足两个单调性。

类似地，我们考虑 $y=mx+b$，我们希望把要最小化的信息视作 $b$，把 $j$ 相关的视作 $y$，把 $mx$ 视作同时与 $i,k$ 相关的，而且把 $m$ 归类为与 $i$ 相关，$x$ 归类为与 $k$ 相关，这样我们就可以得到 $(x_k,y_k)$ 作为一系列的点了。具体地，在
$$
dp[i][j]=\frac{s_i^2+s_i}{2}+\min_{k=j-1}^{i-1}\left\{dp[k][j-1]+\frac{s_k^2-s_k}{2}-s_ks_i\right\}
$$
情形下，我们有
$$
\begin{aligned}
b_i&=dp[i][j]-\frac{s_i^2+s_i}{2},\\
m_i&=-s_i,\\
x_k&=s_k,\\
y_k&=dp[k][j+1]+\frac{s_k^2-s_k}{2}.
\end{aligned}
$$

容易发现，可能让 $b_i$ 取到最小值的点一定在下凸壳上．因此在寻找取得最小值的点 $p$ 的时候我们不需要枚举所有 $i-1$ 个点，只需要考虑凸包上的点．因此我们可以用单调队列维护凸包．

具体地，设 $K(a,b)$ 表示过 $(x_a,y_a)$ 和 $(x_b,y_b)$ 的直线的斜率．考虑队列 $q_l,q_{l+1},\dots,q_r$，维护的是下凸壳上的点．也就是说，对于 $l < i < r$，始终有 $K(q_{i-1},q_i) < K(q_i,q_{i+1})$ 成立．

我们维护一个指针 $e$ 来计算 $b_i$ 最小值．我们需要找到一个 $K(q_{e-1},q_e) \le m_i < K(q_e,q_{e+1})$ 的 $e$（特别地，当 $e = l$ 或者 $e = r$ 时要特别判断。这里相当于用 $f_i$ 去切这个凸包），这时就有 $p = q_e$，即 $q_e$ 是 $i$ 的最优决策点．由于 $m_i$ 是单调递增的，因此 $e$ 的移动次数是均摊 $O(1)$ 的．

在插入一个点 $(x_i,y_i)$ 时，我们要判断是否 $K(q_{r-1},q_r) < K(q_r,i)$，如果不等式不成立就将 $q_r$ 弹出，直到等式满足．然后将 $i$ 插入到 $q$ 队尾．

类似也可以写出对应的代码。

两种方法实质上是等价的。第一种比较好写，第二种比较好扩展。

我们总结一下上面的操作，就是：
1. 将初始状态入队．
2. 每次使用一条和 $i$ 相关的直线 $f(i)$ 去切维护的凸包，找到最优决策，更新 $dp_i$．
3. 加入状态 $dp_i$．如果一个状态（即凸包上的一个点）在 $dp_i$ 加入后不再是凸包上的点，需要在 $dp_i$ 加入前将其剔除．

利用斜率优化 dp 可以将前面的 $O(n^2)$ 的操作通过单调队列优化为 $O(n)$。因为我们在维护一个凸包，这个优化有时也叫做凸包技巧（CHT, Convex Hull Trick）。

## 二分 / CDQ 优化 dp

对于有些问题，斜率并不是单调的．这时我们需要维护凸包上的每一个节点，然后每次用当前的直线 $f_i$ 去切这个凸包．这个过程可以使用二分解决，因为凸包上相邻两个点的斜率是有单调性的．这就是所谓的二分优化 dp。

但是，如果加入的点的坐标也不是单调的，加入决策点的时候，我们有两种方法。
 - 用平衡树维护。那么寻找决策点的二分操作就转化为在平衡树上二分，插入决策点就转化为在平衡树上插入一个结点，并删除若干个被踢出凸壳的点。这个可以通过 `set` 或者 `map` 实现。
	 - 更具体地，首先通过 $x$ 坐标找到 $P$ 应该在的位置，并找到它的前驱 $P_{prev}$ 和后继 $P_{next}$。
	 - 如果 $P$ 落在直线 $P_{prev}P_{next}$ 的上方，说明 $P$ 在凸壳内部，直接丢弃，不执行插入。
	 - 如果 $P$ 确定要插入，它可能会使两侧原本在凸壳上的点变得不再“凸”。
		 - **向右看：** 检查 $P, P_{next}, P_{next\_next}$。如果 $P_{next}$ 导致斜率不再递增（即 $P_{next}$ 变成了“凹”点），就把它从平衡树中删掉。重复此过程，直到右侧恢复凸性。
		- **向左看：** 同理，检查 $P, P_{prev}, P_{prev\_prev}$，删掉左侧所有变“凹”的点。

一个实现案例是
```cpp
#include <set>
#include <limits>
#include <algorithm>

using namespace std;

struct Line {
    mutable long long m, b, p; // p 是这条直线与后继直线的交点横坐标
    bool operator<(const Line& o) const { return m < o.m; } // 用于 set 内部排序（按斜率）
    bool operator<(long long x) const { return p < x; }    // 用于查询时的二分定位
};

struct LineContainer : std::set<Line, less<>> {
    // 定义无穷大，注意处理溢出
    static const long long inf = LLONG_MAX;

    // 计算两条直线的交点，并更新 p
    long long div(long long a, long long b) { 
        return a / b - ((a ^ b) < 0 && a % b); // 整数除法向下取整，处理负数
    }

    bool isect(iterator x, iterator y) {
        if (y == end()) { x->p = inf; return false; }
        if (x->m == y->m) x->p = x->b > y->b ? inf : -inf;
        else x->p = div(y->b - x->b, x->m - y->m);
        return x->p >= y->p; // 如果当前交点位置超过了下一个交点，说明 y 是冗余的
    }

    void add(long long m, long long b) {
        auto z = insert({m, b, 0}), y = z++, x = y;
        while (isect(y, z)) z = erase(z); // 向后剔除冗余线
        if (x != begin() && isect(--x, y)) isect(x, y = erase(y)); // 向前剔除冗余线
        while ((y = x) != begin() && (--x)->p >= y->p) isect(x, erase(y)); // 继续向前追溯
    }

    long long query(long long x) {
        auto l = *lower_bound(x); // 利用重载的 < 运算符进行二分查找
        return l.m * x + l.b;
    }
};
```

# 总结

>斜率优化 dp 需要灵活运用，其宗旨是将最优化问题转化为二维平面上与凸包有关的截距最值问题．遇到性质不太好的方程，有时需要辅以数据结构来加以解决，届时还请就题而论．
