# 0-1 背包

0-1 背包的基本流程是：
 - 确定**容量**；
 - 确定**物品**；
 - 外层枚举物品，内层倒序枚举容量（防止重复更新）。

反过来就是完全背包了。请务必注意。

>[!example]
>给你两个 **正** 整数 `n` 和 `x` 。
>
>请你返回将 `n` 表示成一些 **互不相同** 正整数的 `x` 次幂之和的方案数。换句话说，你需要返回互不相同整数 `[n1, n2, ..., nk]` 的集合数目，满足 `n = n1^x + n2^x + ... + nk^x` 。
>
>由于答案可能非常大，请你将它对 `10^9 + 7` 取余后返回。
>
>比方说，`n = 160` 且 `x = 3` ，一个表示 `n` 的方法是 `n = 2^3 + 3^3 + 5^3` 。

容量——`n`，物品——`ni^x`。因此要这么写：
```cpp
p = pow(n * 1.0, 1.0 / x);
for (int j = 1; j <= p + 1; j++) {
    for (int i = n; i >= pow[j]; i--) {
	    dp[i] += dp[i - pow[j]];
	    dp[i] %= mod;
	}
}
```

# 多重背包

现在有 $n$ 个物品，每个物品有 $c_i$ 个，背包容量为 $V$，每个物品的体积为 $v_i$，价值为 $w_i$。很容易写出一个 $O(V\sum c_i)$ 的程序。但很多情况下这个会超时，所以我们引入下面的优化。
## 二进制优化

我们考虑把每个物品分割成 1、2、4、... 个物品，最后剩下 $r$ 个。然后跑 0-1 背包。这样复杂度就达到了 $O(V\sum \log c_i)$。这很牛了。

>[!example]
>玛莎和比尔拥有一套弹珠收藏。他们希望将收藏品分给两人，使各自获得的弹珠总价值相等。如果所有弹珠价值相同，事情就很简单——直接平分即可。但遗憾的是，有些弹珠更大或更美观。于是，玛莎和比尔先为每颗弹珠标定了 1 到 6 之间的自然数作为价值。现在他们希望分配弹珠，使两人获得的总价值相同。但可能无法实现这样的公平分配（即使所有弹珠总价值为偶数）。例如：若有一颗价值 1、一颗价值 3 和两颗价值 4 的弹珠，则无法分成总价值相等的两份。因此，他们请你编写程序，判断是否存在公平的分配方案。

```cpp
#include <bits/stdc++.h>
using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;

const int MAXN = 20000;
int f[MAXN * 6 + 5];

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    int n[7], t = 0;
    while (true) {
        bool iscase = false;
        int tot = 0;
        for (int i = 1; i <= 6; i++)
            cin >> n[i], tot += i * n[i], iscase = iscase || (n[i]);
        if (!iscase) break;
        t++;
        if (tot % 2) {
            cout << "Collection #" << t << ":" << endl;
            cout << "Can't be divided." << endl;
            cout << endl;
            continue;
        }
        fill(all(f), 0);
        f[0] = 1;
        vector<int> p;
        for (int i = 1; i <= 6; i++) {
            for (int j = 0; 1 << j <= n[i]; j++) {
                p.emplace_back((1 << j) * i);
                n[i] -= 1 << j;
            }
            if (n[i]) p.emplace_back(n[i] * i);
        }
        for (int i = 0; i < p.size(); i++) {
            for (int w = tot; w >= p[i]; w--) { f[w] += f[w - p[i]]; }
        }
        if (f[tot >> 1]) {
            cout << "Collection #" << t << ":" << endl;
            cout << "Can be divided." << endl;
            cout << endl;
        } else {
            cout << "Collection #" << t << ":" << endl;
            cout << "Can't be divided." << endl;
            cout << endl;
        }
    }
    return 0;
}
```

当然，本题作为可行性问题，甚至可以做到 $O(V)$ 的做法。

## 单调队列优化

**问题**：对于每种物品 i（价值为 i，数量为 $n[i]$），我们要判断能否用这些物品凑出目标值 target。
**关键观察**：对于每种物品 i，所有重量 w 可以按 **模 i 的余数** 分成 i 组：
- 余数为 0: w = 0, i, 2i, 3i, ...
- 余数为 1: w = 1, 1+i, 1+2i, 1+3i, ...
- ...
- 余数为 i-1: w = i-1, i-1+i, i-1+2i, ...

**为什么这样分组**？  
因为如果我们用 k 个物品 i，那么：
- 从重量 w 转移到 w + k×i
- 这两个重量对 i 取模的余数相同

## 单调队列优化实现


```cpp

for (int i = 1; i <= 6; i++) {
    int cnt = n[i];  // 物品 i 的数量
    // 按余数分组处理
    for (int mod = 0; mod < i; mod++) {
        deque<int> dq;  // 单调队列，存储的是重量下标
        
        // 遍历该余数分组中的所有可能重量
        for (int j = 0; j * i + mod <= target; j++) {
            int cur_weight = j * i + mod;  // 当前重量
            
            // 维护队列：队首元素使用物品数量超过 cnt 则弹出
            while (!dq.empty() && 
                   j - dq.front() > cnt) {  // j - dq.front() 表示使用的物品数量
                dq.pop_front();
            }
            
            // 如果当前重量可达，加入队列，因为可以作为状态的起点。
            if (f[cur_weight]) {
                dq.push_back(j);
            }
            
            // 如果队列非空，说明可以通过使用物品 i 从某个状态转移到当前状态
            if (!dq.empty()) {
                f[cur_weight] = true;
            }
        }
    }
}
```

**每个状态只进出队列一次**，所以复杂度是线性的。
