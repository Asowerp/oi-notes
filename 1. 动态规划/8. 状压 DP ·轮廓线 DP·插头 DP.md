## 一些基本的操作

 - 获得 `s` 第 `j` 位的状态：`(s>>j)&1`（返回0/1）或者 `s&(1<<j)`（返回 `1<<j` 或 0）；
 - 将 `s` 的第 `j` 位设置成 1：`s|=(1<<j)`；
 - 将 `s` 的第 `j` 位设置成 0：`s&=(~(1<<j))`。

## 状压 dp

状压 DP 是动态规划的一种，通过将状态集合转化为整数记录在 DP 状态中来实现状态转移的目的．

为了达到更低的时间复杂度，通常需要寻找更低状态数的状态．大部分题目中会利用二元状态，用 $n$ 位二进制数表示 $n$ 个独立二元状态的情况．

例题：[P2704 [NOI2001] 炮兵阵地](https://www.luogu.com.cn/problem/P2704)。

**请注意转移的时候有可能可以先预处理所有可能的情况再转移**！！！！

例如：[1793. 矩形覆盖](http://noi.openjudge.cn/ch0405/1793/)。

```cpp
#include <bits/stdc++.h>
using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;

int f[1 << 20];

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    int n;
    while (cin >> n) {
        if (!n) break;
        vector<pii> p;
        vector<pii> r;
        for (int i = 1; i <= n; i++) {
            int x, y;
            cin >> x >> y;
            p.emplace_back(x, y);
        }
        sort(all(p));
        fill(all(f), INT_MAX >> 1);
        f[0] = 0;
        for (int i = 0; i < p.size(); i++) {
            for (int j = i + 1; j < p.size(); j++) {
                int lx, rx, ly, ry;
                tie(lx, rx) = minmax(p[i].first, p[j].first);
                tie(ly, ry) = minmax(p[i].second, p[j].second);
                if (lx == rx) { lx--; }
                if (ly == ry) { ly--; }
                int t = 0;
                for (int k = 0; k < p.size(); k++) {
                    if (p[k].first >= lx && p[k].first <= rx &&
                        p[k].second >= ly && p[k].second <= ry) {
                        t += 1 << k;
                    }
                }
                r.emplace_back(t, (rx - lx) * (ry - ly));
            }
        }
        for (auto [i, a] : r) {
            for (int j = 0; j < (1 << n); j++) {
                if ((i & j) == i) continue;
                f[i | j] = min(f[i | j], f[j] + a);
            }
        }
        cout << f[(1 << n) - 1] << endl;
    }
    return 0;
}
```

### SOS DP（子集和 DP·高维前缀和 DP）

状压 dp 的一个重要维护手段就是 SOS dp（Sum Over Subsets）。

SOS dp 可以快速维护 $f[S]=\sum_{x\subset S}a[x]$。

>[!example] P15261
>给定一行 $n\ (n\leq 20)$ 个格子，你可以在每个格子上填上 `M` 或 `O`，使做完给定的 $k\ (k\leq 2\times 10^5)$ 次操作的得分总和最高。 
>
>操作会给定 3 个数 $x,y,z$，如果第 $x$，第 $y$，第 $z$ 个格子分别填了 `M`，`O`，`O`，则获得一分。

很显然我们自然会想到 dp。那么这种情况我们要维护整个棋盘，而且 $n$ 很小，很自然就是状压 dp 了。

接下来思考的就是如何设计状态转移。我们想容斥。记 1 表示 `M`，0 表示 `O`，那么考虑对每个操作：
 - 如果 $x,y,z$ 对应的恰好是 1，0，0，那么就加一；
 - 反之，如果是 $1,1,0$ 或 $1,0,1$，说明不满足情况，就要减一。
 - 如果是 $1,1,1$，那就重复了，要加一。

最后总体统计一下就好了。时间复杂度是 $O(n\cdot 2^n+k)$ 的。

```cpp
#include <bits/stdc++.h>

using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

inline int read() {
    int x = 0, f = 1;
    char ch = getchar();
    while (ch < '0' || ch > '9') {
        if (ch == '-') f = -1;
        ch = getchar();
    }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + (ch - '0');
        ch = getchar();
    }
    return x * f;
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    int n = read(), k = read();
    n = 1 << n;
    vector<int> f(n);
    while (k--) {
        int x = read(), y = read(), z = read();
        x = 1 << x - 1, y = 1 << y - 1, z = 1 << z - 1;
        f[x]++, f[x + y]--, f[x + z]--, f[x + y + z]++;
    }
    for (int i = 1; i < n; i *= 2) {
        for (int j = n - 1; j >= 1; j--) {
            if (i & j) j -= i;
            f[i + j] += f[j];
        }
    }
    int ans = *max_element(all(f));
    int cnt = ranges::count(f | views::drop(1), ans);
    cout << ans << ' ' << cnt;
    return 0;
}
```

>[!example] LC 3757
>给你一个整数数组 $a$，长度为 $n$，满足 $n\leq 10^5, \max \{a\}\leq 10^6$。
>
>数组的 **强度** 定义为数组中所有元素的 **按位或 (Bitwise OR)**  。
>
>如果移除某个 **子序列** 会使剩余数组的 **强度严格减少** ，那么该子序列被称为 **有效子序列** 。
>
>返回数组中 **有效子序列** 的数量。由于答案可能很大，请返回结果对 $10^9+7$ 取模后的值。
>
>**子序列** 是一个 **非空** 数组，它是由另一个数组删除一些（或不删除任何）元素，并且不改变剩余元素的相对顺序得到的。
>
>空数组的按位或为 0。

我们进行一些思考。在本题中我们记 $a\vee b=a\mid b$，也就是按位与。

很显然我们首先会得到一个 $\bigvee_{i=1}^n a_i=:t$。很显然直接统计似乎比较麻烦，我们反面考虑只需要统计所有满足 $\Lambda \subset\{1,\cdots ,n\}$ 并且 $\bigvee_{\lambda \in \Lambda} a_\lambda=t$ 的子序列个数。问题转化为怎么去求这个数量。

我们通过 sos dp 能够快速维护 $f[S]=\sum_{x\subset S}1$。那一个非常重要的观察就是，我们求 $\vee$ 之后等于 $t$，相当于先求包含于 $t$ 的子集个数，然后剪掉只包含 $t$ 删掉一位的，加上删掉两位的，...。这样通过容斥原理我们就得到了上面所求的个数了。

>[!hint]
>在组合数学中，至多型问题（或者至少型问题）往往比恰好型问题更好算。
>
>上面的这个技巧叫做**子集反演**。

```cpp
static constexpr int MOD = 1'000'000'007;
static constexpr int MAX_N = 100'001;
int pow2[MAX_N];

int init = [] {
    // 预处理 2 的幂
    pow2[0] = 1;
    for (int i = 1; i < MAX_N; i++) {
        pow2[i] = pow2[i - 1] * 2 % MOD;
    }
    return 0;
}();

class Solution {
public:
    int countEffective(vector<int>& nums) {
        // 优化：如果 nums 只有一种数字，那么非空子序列的按位或都是 or_，只有空子序列的按位或比 or_ 小
        if (ranges::all_of(nums, [&](int x) { return x == nums[0]; })) {
            return 1;
        }

        int or_ = reduce(nums.begin(), nums.end(), 0, bit_or<>());
        int w = bit_width((uint32_t) or_);

        vector<int> f(1 << w);
        for (int x : nums) {
            f[x]++;
        }
        for (int i = 0; i < w; i++) {
            if ((or_ >> i & 1) == 0) { // 优化：or_ 中是 0 但 s 中是 1 的 f[s] 后面容斥用不到，无需计算
                continue;
            }
            for (int s = 0; s < (1 << w); s++) {
                s |= 1 << i;
                f[s] += f[s ^ (1 << i)];
            }
        }
        // 计算完毕后，f[s] 表示 nums 中的是 s 的子集的元素个数

        long long ans = pow2[nums.size()]; // 所有子序列的个数
        // 枚举 or 的所有子集（包括空集）
        int sub = or_;
        do {
            int sign = popcount((uint32_t) or_ ^ sub) % 2 ? -1 : 1;
            ans -= sign * pow2[f[sub]];
            sub = (sub - 1) & or_;
        } while (sub != or_);
        return (ans % MOD + MOD) % MOD; // 保证结果非负
    }
};
```
## 轮廓线 dp

接下来看下面这道题。

>[!example] P1879 [USACO06NOV] Corn Fields G
>农场主 $\rm John$ 新买了一块长方形的新牧场，这块牧场被划分成 $M$ 行 $N$ 列 $(1 \le M \le 12, 1 \le  N \le 12)$，每一格都是一块正方形的土地。 $\rm John$ 打算在牧场上的某几格里种上美味的草，供他的奶牛们享用。
>
>遗憾的是，有些土地相当贫瘠，不能用来种草。并且，奶牛们喜欢独占一块草地的感觉，于是 $\rm John$ 不会选择两块相邻的土地，也就是说，没有哪两块草地有公共边。
>
>$\rm John$ 想知道，如果不考虑草地的总块数，那么，一共有多少种种植方案可供他选择？（当然，把新牧场完全荒废也是一种方案）

很显然，这个用朴素的状压 dp 可以做到 $O(m\cdot 2^{2n})$，瓶颈在于每次状态转移都要贡献一个 $2^n$ 的复杂度。

```cpp
#include <bits/stdc++.h>

using namespace std;

int dp[20][5000], a[20][20];
int cnt[5000];
int n, m;
const int mod = 1e8;

int main() {
    cin >> n >> m;
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= m; j++) cin >> a[i][j];
    }
    dp[0][0] = 1;
    for (int i = 1; i <= n; i++) {
        for (int j = 0; j < (1 << m); j++) {
            if (j & (j >> 1) || j & (j << 1)) continue;
            bool ok = true;
            for (int k = 0; k < m; k++) {
                if (!a[i][k + 1] && (1 << k) & j) {
                    ok = false;
                    break;
                }
            }
            if (!ok) continue;
            for (int k = 0; k < (1 << m); k++) {
                if (k & j || k & (k >> 1) || k & (k << 1)) continue;
                dp[i][j] = (dp[i][j] + dp[i - 1][k]) % mod;
            }
        }
    }
    int ans = 0;
    for (int i = 0; i < (1 << m); i++) {
        ans = (ans + dp[n][i]) % mod;
    }
    cout << (ans %= mod) << endl;
    return 0;
}
```

能不能更快？或者说，当 $n=10^4$，有没有更好的做法？这就是下面介绍的轮廓线 dp 的应用。

我们先定义轮廓线。我们用 $f_{i,j,s}$ 表示第 $i$ 行第 $j$ 列轮廓线上状态为 $s$ 的情况下，能够填满的方法数。（请注意，列是 0-indexed 的）

所谓第 $i$ 行第 $j$ 列确定的轮廓线，就如下图所示：

![[da7e085c3fc0b4f8bb1fe6d4d1d52667.jpg]]

换言之，$s[0..j-1]$ 表示第 $i$ 行的状态，$s[j..m-1]$ 表示第 $i-1$ 行的状态。其中 1 表示已占用，0 表示未占用。

我们来看这样的状态设计如何服务于本题。

我们当前进行到 $(i,j)$ 了。我们要看这里怎么放。

![[e822c8b460dc55569cff5aed8712171f.jpg]]

如果 left, up 有一个是 1，那么这里就不能放。因此这里之后的轮廓线必然是 0。我们就更新下一个轮廓线是
$$
f_{i,j+1,\texttt{s}\otimes\texttt{(1<<j)}}\leftarrow f_{i,j+1,\texttt{s}\otimes\texttt{(1<<j)}}+f_{i,j,\texttt{s}}.
$$
这里 $a\otimes b$ 表示 $a$ 清空 $b$ 中为 1 的位上的数。

以此类推，我们发现 $i$ 其实根本没有用，所以完全可以扔掉这一维。我们最后实际上要的就是 $f_{n,m,2^m-1}$。

可以看出，我们的精髓就是考虑当前这一格的分类讨论对下面一步的操作会有什么影响。**但是请注意每一步操作前要先把下一步的清零**，否则影响叠加就不是我们想要的了。

一个实现如下（包含空间压缩）：

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MOD = 1e8;

int main() {
    int n, m;
    cin >> n >> m;
    
    vector<vector<int>> grid(n, vector<int>(m));
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            cin >> grid[i][j];
        }
    }
    
    // 使用滚动数组优化空间
    vector<vector<int>> dp(2, vector<int>(1 << m, 0));
    dp[0][0] = 1;
    
    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            int cur = (i * m + j) & 1;
            int nxt = cur ^ 1;
            
            fill(dp[nxt].begin(), dp[nxt].end(), 0);
            
            for (int mask = 0; mask < (1 << m); mask++) {
                if (dp[cur][mask] == 0) continue;
                
                // 不能种草的情况
                if (grid[i][j] == 0 || (mask & (1 << j))) {
                    // 当前位置为 0
                    int new_mask = mask & ~(1 << j);
                    dp[nxt][new_mask] = (dp[nxt][new_mask] + dp[cur][mask]) % MOD;
                    continue;
                }
                
                // 检查左边是否有草
                if (j > 0 && (mask & (1 << (j-1)))) {
                    // 左边有草，不能种草
                    int new_mask = mask & ~(1 << j);
                    dp[nxt][new_mask] = (dp[nxt][new_mask] + dp[cur][mask]) % MOD;
                } else {
                    // 可以种草
                    int new_mask = mask | (1 << j);
                    dp[nxt][new_mask] = (dp[nxt][new_mask] + dp[cur][mask]) % MOD;
                    
                    // 也可以不种草
                    new_mask = mask & ~(1 << j);
                    dp[nxt][new_mask] = (dp[nxt][new_mask] + dp[cur][mask]) % MOD;
                }
            }
        }
    }
    
    // 计算最终答案
    int ans = 0;
    int cur = (n * m) & 1;
    for (int mask = 0; mask < (1 << m); mask++) {
        ans = (ans + dp[cur][mask]) % MOD;
    }
    
    cout << ans << endl;
    return 0;
}
```

>[!example] Mondriaan's Dream
>输出给定大长方形（$n\times m$，$n,m\leq 11$）可以用 $2\times 1$ 的小长方形填充的不同方式的数量。

一个比较朴素的想法是，我们直接去考虑当前这一行是怎么的状态。但是很显然，这样我们就需要至少三个状态，时间复杂度是不可接受的 $O(n\cdot 3^{2n})$，这是因为状态转移也是 $O(3^n)$ 的。

利用前面介绍的轮廓线 dp 就可以做到 $O(nm\cdot 2^{\min\{n,m\}})$ 解决。

具体来讲，我们依然考虑 left（$2^{j-1}$） 和 up（$2^j$）。

接下来分类讨论。
 - 如果 up 为空，那这里一定要有一个竖着放的，因此 $f_{j+1,s\oplus 2^j}\leftarrow f_{j+1,s\oplus 2^j}+f_{j,s}$ ；
 - 如果 up 不为空，
	 - 无论 left 是否为空，这一格总可以留空不填，因此依然有 $f_{j+1,s\oplus 2^j}\leftarrow f_{j+1,s\oplus 2^j}+f_{j,s}$；
	 - 如果 left 为空，那这里可以横着放一个，因此 $f_{j+1,s\oplus 2^{j-1}}\leftarrow f_{j+1,s\oplus 2^{j-1}}+f_{j,s}$。

```cpp
#include <bits/stdc++.h>
using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;

int n, m;
ll f[2][4000];

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    while (cin >> n >> m) {
        if (!n && !m) break;
        if (n * m & 1) {
            cout << 0 << endl;
            continue;
        }
        if (n < m) swap(n, m);
        int cur = 0;
        memset(f, 0, sizeof(f));
        f[0][(1 << m) - 1] = 1;
        for (int i = 1; i <= n; i++) {
            for (int j = 0; j < m; j++) {
                int nxt = cur ^ 1;
                fill(all(f[nxt]), 0);
                for (int s = 0; s < (1 << m); s++) {
                    int left = 0, up = (s >> j) & 1;
                    if (j) left = (s >> j - 1) & 1;
                    if (j && up && !left)
                        f[nxt][s ^ (1 << j - 1)] += f[cur][s];
                    f[nxt][s ^ (1 << j)] += f[cur][s];
                }
                cur = nxt;
            }
        }
        cout << f[cur][(1 << m) - 1] << endl;
    }
    return 0;
}
```

## 插头 dp

轮廓线更进一步就是所谓的插头 dp。

>[!example] CF845F
>Polycarp 在 Berland 首都拥有一家店铺。最近首都的犯罪活动增加了，因此 Polycarp 正在考虑为他的仓库建立更好的安全措施。
>
>这个仓库可以用一个 $n$ 行 $m$ 列的矩阵来表示。矩阵中的每个元素要么是 $.$（空地），要么是 $x$（墙）。
>
>Polycarp 想雇佣一些警卫（可以为零个），守卫仓库。每个警卫将站在矩阵的某个格子里，能够守卫从他所在格子起，向右直到最近的墙、向下直到最近的墙的所有格子。更形式化地说，如果警卫站在格子 $(x_0,y_0)$，他能守卫格子 $(x_1,y_1)$，当且仅当以下条件全部满足：
>
>- $(x_1,y_1)$ 是一个空地；
>- 要么 $x_0 = x_1$ 且 $y_0 \leq y_1$，要么 $x_0 \leq x_1$ 且 $y_0 = y_1$；
>- 在 $(x_0,y_0)$ 与 $(x_1,y_1)$ 之间没有墙，可以有其他警卫经过，警卫可以“看穿”其他警卫。
>
>警卫只能被放在空地上（且只能守卫空地）。警卫布置方案是指出警卫站在哪些格子的方案（如果有至少一个格子只包含在第一个方案而不在第二个方案，或者反之，则这两个方案不同）。Polycarp 认为一个方案是合适的，如果至多只有一个空地没有被守卫。
>
>Polycarp 想知道有多少种合适的方案。由于答案可能很大，你需要输出对 $10^9+7$ 取模的结果。

本质上，依然属于轮廓线 dp 的思考方式。我们看下面这个示意图。

![[0faffafb5abe412edb63d856238a650d.jpg]]

我们警卫的视线会和轮廓线产生交点。这种东西我们就形象地叫作“插头”。于是，我们就可以开始用轮廓线的想法了。只不过这里状态是 $m+1$ 位的，因为要额外来一个向上的一步给的一个向右的插头。

我们用 $s$ 来表示这一个位置是否有插头经过。不难发现，如果 $s$ 的第 $j$ 位表明是转折的那一格。就是当前的上方，实际上是 $s$ 的第 $j+1$ 位。

> 轮廓虽然是一行的信息，但我们是一格一格 dp 的，即每右移一格到达第 `j` 个格子时，状态空间的所有轮廓，`0~j-1`编号表达的是左边 `j`个格子朝下是否有插头的轮廓，这些信息对这一行的格子更新已经用不到了，它们会在DP到下一行时被用到。即我们是带着整行的轮廓信息，但一格一格地去 DP 和更新轮廓状态空间。

这里有一个细节是，当我们现在最右格子处理完，要转头处理下一行最左格子的时候，会发现一个情况：

![[50e4b37059f7294d32aa033a9fd8b3b9.jpg]]

这时候需要做一个 Line Shift（插头平移）操作：
- 把上一步状态空间的所有状态（即所有压缩表示的轮廓线）的二进制位左移一位
- 每个状态的最高位丢弃（最高位表达的是最右边格子朝右的插头，这个信息也用不着）
- 每个状态的最低为设为 `0` （最左边格子不可能有左边伸过来的插头）
这样我们就可以开始下一行的转移了。

令 $f_{0/1,i,j,s}$ 表示是否用掉空一格，当前位于 $(i,j)$，轮廓线状态为 $s$ 的方法数。我们现在关注的就是当前这一格放 / 不放警卫对后续的状态有什么影响。

如果是空地的话，
 - 如果当前这一格放了警卫，那么显然有转移 $$f_{0/1,i,j+1,s|(2^j+2^{j+1})}\leftarrow f_{0/1,i,j+1,s|(2^j+2^{j+1})}+f_{0/1,i,j+1,s}$$
 - 如果当前这一格不放警卫，那我们考虑 left 和 up。
	 - 如果 left 和 up 有一个是 true，那这一格已经被管住了。
		 - 如果 left 是 true，这个可以接着转下去。
			 - 但是如果当前的 up 不是 true，那么 $$f_{0/1,i,j+1,(s|(2^{j+1}))\otimes (2^j)}\leftarrow f_{0/1,i,j+1,(s|(2^{j+1}))\otimes (2^j)}+f_{0/1,i,j+1,s}$$
			 - 如果 up 是 true，$$f_{0/1,i,j+1,(s|(2^{j+1}+2^j))}\leftarrow f_{0/1,i,j+1,(s|(2^{j+1}+2^j))}+f_{0/1,i,j+1,s}$$
		 - 如果 left 不是 true，那么这时候一定 up 是 true，那么 $$f_{0/1,i,j+1,(s|(2^j))}\leftarrow f_{0/1,i,j+1,(s|(2^j))}+f_{0/1,i,j+1,s}$$
	- 如果都不是 true，那只能用掉 $0/1$ 的机会了，因此 $$f_{1,i,j+1,(s\otimes(2^j+2^{j+1}))}\leftarrow f_{1,i,j+1,(s\otimes(2^j+2^{j+1}))}+f_{0,i,j+1,s}$$

如果是墙，那类似，只不过把所有的插头都挡住了，我们直接清空 $j,j+1$ 位即可。

这样我们就实现了 $O(nm2^{\min\{n,m\}})$ 的解法。本题中，$\min\{n,m\}\leq\sqrt{250}<15$，因此能够通过。

当然，也可以选择不用 Line Shift 的技巧，额外开一维表示是否左侧被监视（左侧有没有插头过来）。

```cpp
#include <algorithm>
#include <iostream>
#include <string>
#include <vector>

using namespace std;

const int MOD = 1e9 + 7;

int n, m;
vector<string> grid;
// dp[k][mask][left_cover]
int dp[2][1 << 15][2];
int ndp[2][1 << 15][2];

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    int r, c;
    cin >> r >> c;
    grid.resize(r);
    for (int i = 0; i < r; ++i) cin >> grid[i];

    // 确保列数较小，优化状态空间
    if (r < c) {
        vector<string> new_grid(c, string(r, ' '));
        for (int i = 0; i < r; ++i)
            for (int j = 0; j < c; ++j) new_grid[j][i] = grid[i][j];
        grid = new_grid;
        swap(r, c);
    }
    n = r, m = c;

    dp[0][0][0] = 1;

    for (int i = 0; i < n; ++i) {
        for (int j = 0; j < m; ++j) {
            // 清空下一格的临时 DP 数组
            for (int k = 0; k < 2; ++k)
                for (int mask = 0; mask < (1 << m); ++mask)
                    ndp[k][mask][0] = ndp[k][mask][1] = 0;

            for (int k = 0; k < 2; ++k) {
                for (int mask = 0; mask < (1 << m); ++mask) {
                    for (int left = 0; left < 2; ++left) {
                        if (!dp[k][mask][left]) continue;

                        long long val = dp[k][mask][left];
                        bool up = (mask >> j) & 1;

                        if (grid[i][j] == 'x') {
                            // 墙：清除横向和纵向插头
                            int nmask = mask & ~(1 << j);
                            int nleft = 0;
                            ndp[k][nmask][nleft] =
                                (ndp[k][nmask][nleft] + val) % MOD;
                        } else {
                            // 1. 放警卫
                            int nmask_guard = mask | (1 << j);
                            int nleft_guard = 1;
                            ndp[k][nmask_guard][nleft_guard] =
                                (ndp[k][nmask_guard][nleft_guard] +
                                 val) %
                                MOD;

                            // 2. 不放警卫
                            if (up || left) {
                                // 已被覆盖，直接转移
                                ndp[k][mask][left] =
                                    (ndp[k][mask][left] + val) % MOD;
                            } else if (k == 0) {
                                // 未被覆盖且有跳过机会
                                ndp[1][mask][left] =
                                    (ndp[1][mask][left] + val) % MOD;
                            }
                        }
                    }
                }
            }

            // 滚动数组
            for (int k = 0; k < 2; ++k)
                for (int mask = 0; mask < (1 << m); ++mask)
                    for (int left = 0; left < 2; ++left)
                        dp[k][mask][left] = ndp[k][mask][left];
        }

        // 换行处理：每一行结束时，横向插头必须重置为 0
        for (int k = 0; k < 2; ++k) {
            for (int mask = 0; mask < (1 << m); ++mask) {
                if (dp[k][mask][1]) {
                    dp[k][mask][0] =
                        (dp[k][mask][0] + dp[k][mask][1]) % MOD;
                    dp[k][mask][1] = 0;
                }
            }
        }
    }

    long long ans = 0;
    for (int k = 0; k < 2; ++k)
        for (int mask = 0; mask < (1 << m); ++mask)
            ans = (ans + dp[k][mask][0]) % MOD;

    cout << ans << endl;

    return 0;
}
```

如果使用插头平移的话，就是如下的代码：

```cpp
#include <iostream>
#include <vector>
#include <string>
#include <algorithm>

using namespace std;

const int MOD = 1e9 + 7;

int f[2][1 << 16][2]; // f[k][mask][is_skipped]

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int n, m;
    cin >> n >> m;
    vector<string> s(n);
    for (int i = 0; i < n; i++) cin >> s[i];

    // 确保 m 是较小的那一个，状态数 2^(m+1)
    if (n < m) {
        vector<string> tmp(m, string(n, ' '));
        for (int i = 0; i < n; i++)
            for (int j = 0; j < m; j++)
                tmp[j][i] = s[i][j];
        s = tmp;
        swap(n, m);
    }

    int cur = 0;
    f[cur][0][0] = 1;

    for (int i = 0; i < n; i++) {
        for (int j = 0; j < m; j++) {
            int nxt = cur ^ 1;
            // 清空下一格状态
            for (int mask = 0; mask < (1 << (m + 1)); mask++) {
                f[nxt][mask][0] = f[nxt][mask][1] = 0;
            }

            for (int mask = 0; mask < (1 << (m + 1)); mask++) {
                for (int k = 0; k < 2; k++) {
                    if (!f[cur][mask][k]) continue;
                    int val = f[cur][mask][k];

                    int up = (mask >> j) & 1;      // 上方插头
                    int left = (mask >> m) & 1;    // 左侧插头

                    if (s[i][j] == 'x') {
                        // 遇到墙：向下和向右的插头全部断掉
                        int nmask = mask & ~(1 << j);  // 清除 up 位
                        nmask &= ~(1 << m);            // 清除 left 位
                        f[nxt][nmask][k] = (f[nxt][nmask][k] + val) % MOD;
                    } else {
                        // 1. 在空地放警卫：产生向下和向右的插头
                        int nmask = mask | (1 << j) | (1 << m);
                        f[nxt][nmask][k] = (f[nxt][nmask][k] + val) % MOD;

                        // 2. 不放警卫
                        if (up || left) {
                            // 已经被覆盖，插头状态继承，但不产生新的“源”
                            // 注意：up 还是那个 up，left 还是那个 left
                            f[nxt][mask][k] = (f[nxt][mask][k] + val) % MOD;
                        } else if (k == 0) {
                            // 没被覆盖且还有一次跳过机会
                            f[nxt][mask][1] = (f[nxt][mask][1] + val) % MOD;
                        }
                    }
                }
            }
            cur = nxt;
        }

        // --- Line Shift / 换行处理 ---
        // 每一行结束后，第 m 位的 left 插头必须清零，因为它不能跨行
        int nxt = cur ^ 1;
        for (int mask = 0; mask < (1 << (m + 1)); mask++) 
            f[nxt][mask][0] = f[nxt][mask][1] = 0;

        for (int mask = 0; mask < (1 << (m + 1)); mask++) {
            for (int k = 0; k < 2; k++) {
                if (!f[cur][mask][k]) continue;
                // 将 mask 的第 m 位清零，存入新状态
                int nmask = mask & ~(1 << m);
                f[nxt][nmask][k] = (f[nxt][nmask][k] + f[cur][mask][k]) % MOD;
            }
        }
        cur = nxt;
    }

    int ans = 0;
    for (int mask = 0; mask < (1 << (m + 1)); mask++) {
        ans = (ans + f[cur][mask][0]) % MOD;
        ans = (ans + f[cur][mask][1]) % MOD;
    }

    cout << ans << endl;

    return 0;
}
```

当然，上面的思考还是局限于轮廓线的看法了，实际上，我们还有一类问题必须用插头 dp 解决。

>[!example]
>一个网格图上，部分格子标记为不能通过，在网格图上走一些回路保证每个可行格子经过且只经过一次，有多少种方案？

简单思考发现，我们依然可以用插头的思想。一个合理的回路，这个格子的左边与上边放的拼图无非有 4 种可能性：
1. 左边伸过来，上边没伸
2. 左边没伸，上边伸过来了
3. 左边伸过来，上边也伸过来
4. 左边和上边都没有

把这当作一个拼图游戏，我们需要往这个格子放一个合理的拼图块。对于前 3 种，没得选，左边来上边没来，那肯定这一格要继续往右边走。左边没有上边有，就往下走。左边上边都有，那一定得填一个拐角把它们接起来。

如果左边上边都没有，仿佛是有得选了。但考虑一下这道题的要求：“回路”，“所有格子经过且只经过一次“，如果这一格放了个横向或竖向的拼图，那么这个线的另一头就”悬空“了，因为我们是从上往下、从左往右地放，它左边和上边的格子是已经定的方案，不会再有格子接下这个接头，最后拼完拼图，那个悬空的头显然没有构成回路。同理，放朝上或朝左的拐角也不行。但放一个朝右朝下的拐角是可以的，因为它右边和下边的格子还没放拼图，总有机会把他们接起来。

好，现在我们来聊聊轮廓线与插头。

在考虑一个特定格子时，需要相邻的上边格子与左边格子的信息，而这不是那个递推问题，上边和左边都会有很多种可能性。动态规划我们很熟悉了，会有个状态空间，我们是从一个状态空间转移到下一个状态空间。如果用前文的滚动数组（`dp[2][8]`） ，存这个状态空间就会很棘手：

- 状态转移时，又要用上面格子，又要用左边格子，属于两行，如果用滚动数组，枚举状态空间很不方便
- 一行格子的很多种可能性，要开很多维的数组？显然这里我们用状态压缩

我们发现，决策一个格子填哪个拼图的关键，并不是上边和左边格子填的是哪个拼图这件事，而是上边和左边的拼图有没有伸过来的路径，或者说现在这个格子要不要对接一个伸过来的路径 —— 这像个插头啊。

所以我们不需要用滚动数组表达两行的内容，只需要记录一条轮廓线上是否有插头就好了。

>路径模型与染色模型是轮廓线 dp 最为典型的两种问题模型。
>
>路径模型指在网格上走出符合要求的路径，通常对每个格子处理左侧和上方的插头，通常轮廓线编号 `j` 表示左侧插头，`j + 1` 表示上方插头。
>
>染色模型指在限制条件下对网格染色，通常对每个格子处理左侧、左上、上方三个格子的颜色，通常轮廓线编号 `j` 表示左侧方格，`j + 1` 表示左上方格，`j + 2` 表示上方方格。
>
>一些题目会把路径放在网格的边上，这类我们可以把格点当成网格，把网格当成格点来做。 一行 8 个格子有 9 个格点。
>
>一些题目会需要计算路径时考虑染色，大胆一点，路径+染色模型，不就多存几个二进制的状态么🤣。

## k 进制状压 dp
