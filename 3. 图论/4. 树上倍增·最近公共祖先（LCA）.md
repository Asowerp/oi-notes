#LCA

最近公共祖先简称 LCA（Lowest Common Ancestor）。两个节点的最近公共祖先，就是这两个点的公共祖先里面，离根最远的那个。$\newcommand{\LCA}{\operatorname{LCA}}$$\newcommand{\dep}{\operatorname{dep}}$$\newcommand{\dist}{\operatorname{dist}}$
## 倍增算法

求 LCA 最朴素的思想就是一步一步回跳。在树上，这两个点最后一定会相遇，最初相遇的位置就是想要求的 LCA。

我们定义 $f_{x,i}$ 为 $x$ 的第 $2^i$ 个祖先。显然 $f_{x,0}$ 表示 $x$ 的父亲。于是不难发现 $f_{i,j}=f_{f_{i,j-1},j-1}$。

于是深度和 $f$ 数组可以像下面这样初始化。

```cpp
inline void dfs (int u, int fa) {
	f[u][0] = fa;//记录父亲节点
	dep[u] = dep[fa] + 1;//记录深度
	for (auto v : e[u])
		if (v != fa) dfs (v, u);
}
inline void init () {
	for (int j = 1; (1 << j) <= n; ++j)//边界范围
		for (int i = 1; i <= n; ++i)
			f[i][j] = f[f[i][j - 1]][j - 1];
}
```

那如果要求解 $\LCA(u,v)$，先将 $u,v$ 跳到同一深度，随后从大到小枚举 $f_{u,i}, f_{v,i}$。一旦不同，立即赋值 $u\leftarrow f_{u,i}, v\leftarrow f_{v,i}$。这样最终相等的时候就是两者的最小公共祖先。

代码如下。

```cpp
inline int lca (int u, int v) {
	if (dep[u] < dep[v]) swap (u, v);
	for (int i = 22; i >= 0; i--) {
		if (dep[f[u][i]] >= dep[v]) u = f[u][i];
	}//跳至同一深度
	if (u == v) return u;//此时节点为 LCA
	for (int i = 22; i >= 0; i--) {
		if (f[u][i] != f[v][i]) {
			u = f[u][i];
			v = f[v][i];
		}
	}//一起往上跳
	return f[u][0];
}
```

## Tarjan 算法

Tarjan 算法是一种 **离线算法**，需要使用并查集记录某个结点的祖先结点。做法如下：

1. 首先接受输入边（邻接链表）、查询边（存储在另一个邻接链表内）。查询边其实是虚拟加上去的边，为了方便，每次输入查询边的时候，将这个边及其反向边都加入到 `queryEdge` 数组里。
2. 然后对其进行一次 DFS 遍历，同时使用 `visited` 数组进行记录某个结点是否被访问过、`parent` 记录当前结点的父亲结点。
3. 其中涉及到了 **回溯思想**，我们每次遍历到某个结点的时候，认为这个结点的根结点就是它本身。让以这个结点为根节点的 DFS 全部遍历完毕了以后，再将这个结点的根节点设置为这个结点的父一级结点。
4. 回溯的时候，如果以该节点为起点，`queryEdge` 查询边的另一个结点也恰好访问过了，则直接更新查询边的 LCA 结果。
5. 最后输出结果。

我们接下来详细讲解一下整个流程。

第一步是构建一个查询列表。如下图所示。

![[Pasted image 20260122002604.png]]


我这里有 6 个查询。第一个查询是 $(f,e)$ 的查询，因此 f 的列表中有一个 e，e 的列表中有一个 f，并且两个的 id 都是 1。以此类推。

第二步，遍历。来到当前节点 cur，标记 `vis[cur]=true`，表示已经访问。我们遍历 cur 的所有子树，每棵子树遍历完都和 cur 合并，并设置 cur 为头节点。

打个比方，我们从 a 开始，先遍历了 b，e，f，这样我们就有了一个 {b,e,f,a} 的集合，其头节点为 a。（这里我们需要用到并查集）这样之后我们再去遍历 c。

遍历完 cur 的所有子树之后，我们处理关于 cur 的询问 $(\text{cur}, x)$。
 - 如果 `vis[x]=true`，说明在之前已经访问过 `x` 了。也就是说，当前的 cur 的头节点一定位于 x 的头节点的下面 / cur 本身就是 x 的头节点（这是 dfs 的遍历顺序决定的，我们首先遍历了 cur，说明要么：1) x 在 cur 下面，此时 cur 的 fa 已经是 x 了；2) x 在另外一个分支，此时 cur 的 fa 还没更新，但是必然是 x 的 fa 的子节点）。因此，x 的头节点就是 cur 和 x 的最近公共祖先。
 - 反之，则轮到处理 $x$ 的时候再处理 cur 和 $x$ 的 LCA。

这样 dfs 的复杂度是 $O(n)$，遍历问题的复杂度是 $O(2q)=O(q)$，因为每个问题至多查询 2 次。因此总复杂度就是 $O((n+q)\alpha(n+q))\sim O(n+q)$。

```cpp
#include <bits/stdc++.h>

using namespace std;

const int MAXN = 5e5 + 5;

int n, m, s;
int pa[MAXN];
bool vis[MAXN];
vector<int> e[MAXN];
vector<pair<int, int>> q[MAXN];
int ans[MAXN];

class dsu {
    vector<int> fa;

  public:
    dsu() {}
    dsu(int _n) : fa(_n + 1) { iota(fa.begin() + 1, fa.end(), 1); }
    int find(int x) { return fa[x] == x ? x : fa[x] = find(fa[x]); }
    void merge(int x, int y) { fa[x] = y; }
};

dsu uf;

void tarjan(int u, int fa) {
    vis[u] = true;
    for (auto v : e[u]) {
        if (v == fa) continue;
        tarjan(v, u);
        uf.merge(v, u);
    }
    for (auto [v, id] : q[u]) {
        if (vis[v]) { ans[id] = uf.find(v); }
    }
}

int main() {
    cin >> n >> m >> s;
    uf = dsu(n);
    for (int i = 1; i < n; i++) {
        int u, v;
        cin >> u >> v;
        e[u].emplace_back(v), e[v].emplace_back(u);
    }
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        q[u].emplace_back(v, i);
        q[v].emplace_back(u, i);
    }
    tarjan(s, 0);
    for (int i = 1; i <= m; i++) cout << ans[i] << endl;
    return 0;
}
```

## 最短路径问题

在树上，如果 $\LCA(a,b)=c$，那么 $$\dist(a,b)=\dist(a,c)+\dist(b,c)=\dep(a)+\dep(b)-\dep(c).$$接下来几道题目基本上都依赖这个结论。以及 MST 一定是 MBST（见 [[1. 最小生成树]]）。

 - [P1967](https://www.luogu.com.cn/problem/P1967)：最大生成树 + LCA。——汽车能走大边一定走大边了，因此我们想要一个 MBST，也就是我们想构建一个 MST。时间复杂度 $O(m\log m+q\log n)$。
 - [ABC438F](https://atcoder.jp/contests/abc438/tasks/abc438_f)：树上倍增的应用。trick 有计算 $f(x)$ 表示$\newcommand{\mex}{\operatorname{mex}}$ $\mex=x$ 的路径个数，那么答案就等于 $∑f(x)\times x$。发现不好算，因为要计算 $f(x)$ 需要算出包含 $0\sim x−1$ 而**不**包含 $x$ 的路径个数。重新刻画一下，定义 $g(x)$ 表示 $\mex≥x$ 的路径个数，答案就等于 $∑g(x)$。然后递推求解，假设我们已经维护了经过 0∼x−1 的路径的左右端点，考虑加上 x：
	 - 如果 x 已经在路径上，那么 g(x)=g(x−1)。
	 - 否则我们就暴力往上跳直到跳到路径上某个点。
		 - 如果跳到的是路径上的端点，就将该端点到 x 间的点都加进维护路径里，然后把端点设为 x。
		 - 如果跳到的是非端点，显然不存在这样的路径，那就直接结束。
	然后得到两个端点后我们乘法原理子树大小相乘便是 g(x)，注意一个端点是 0 的情况要特判。
