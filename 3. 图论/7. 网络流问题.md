#网络流 

# 概念定义

以下，令 $n:=|V|, m:=|E|$。

网络是指一个特殊的有向图 $G=(V,E)$。它在这个网络上还有所谓“容量”与“源点”的概念。
 - $E$ 中的每个边都有“容量”的权值 $c(u,v)$，并且当 $(u,v)\notin E$ 时，规定 $c(u,v)=0$。
 - $V$ 中存在两个特殊的点：源点 $s$ 与汇点 $t$。顾名思义。

称函数 $f:E\to\mathbb R$ 是 $G=(V,E)$ 上的**流**，如果其满足：
1. 容量限制，$0\leq f(u,v)\leq c(u,v)$；
2. 流守恒性：流入某个节点的量等于流出某个节点的量。
同时，我们规定 $|f(s)|$ 为 $f$ 的**流量**。对整个图运用流守恒性，我们不难发现 $f(s)+f(t)=0$。

对于网络 $G$，如果 $\{S,T\}$ 是 $V$ 的分划，并且 $s\in S,t\in T$，则 $\{S,T\}$ 是 $G$ 的一个 $s-t$ 割。其容量定义为 $\left||S,T\right||:=\sum_{u\in S}\sum_{t\in T} c(u,v)$。

# 最大流问题

最大流问题是十分形象的。我们期望最大化从 $s$ 流向 $t$ 的总流量。解决这个问题我们有 Edmonds-Karp 算法、Dinic 算法与 ISAP 算法。其中 Edmonds-Karp 算法比较简单，但效率不高。ISAP 算法比 Dinic 更快，代码长度相仿，因此是优选方法。上面三种算法都可以归结为 Ford–Fulkerson 增广。该方法运用贪心的思想，通过寻找增广路来更新并求解最大流。

## Ford–Fulkerson 算法思路

对于边 $(u,v)$，我们将其容量与流量之差称为**剩余容量** $c_f(u,v):=c(u,v)-f(u,v)$。所有节点和 $c_f>0$ 的边构成一个 $G$ 的子图，称作**残量网络** $G_f$。

在 $G_f$ 上，任意一条从 $s$ 到 $t$ 的路径（以下简记为 $s\to t$，注意它是集合，即 $s\to t\subset G_f$）称为**增广路**。对于一条增广路，我们给每一条边 $(u,v)\in s\to t$ 都加上等量的流量，以令整个网络的流量增加，这一过程被称为**增广**。由此，最大流的求解可以被视为若干次增广分别得到的流的叠加。

在建图的过程中，我们规定流 $f$ 需要满足反对称性，即 $f(u,v)=-f(v,u)$。这一性质可以通过在每次增广时引入退流操作，即 $f(u,v)$ 增加时令 $f(v,u)$ 减少相同的量，来保证。

Ford-Fulkerson 算法可以归结为以下三步：
1. 初始时，设置 $f\equiv 0$；
2. 找到 $s\to t\in G_f$，执行增广操作，得到一个新的流。
	- 这实际上相当于找到了一个新流 $g_i$，并更新 $f\leftarrow f+g_i$。
3. 重复第二步，直到没有边可以更新。

>[!tip]+
>在最大流算法的代码实现中，我们往往需要支持快速访问反向边的操作．在邻接矩阵中，这一操作是 trivial 的．但主流的实现是更加优秀的链式前向星．其中，一个常用的技巧是，我们令边从偶数（通常为 0）开始编号，并在加边时总是紧接着加入其反向边使得它们的编号相邻．由此，我们可以令编号为 $i$ 的边和编号为 $i\oplus 1$ 的边始终保持互为反向边的关系．

初次接触这一方法的读者可能察觉到一个违反直觉的情形——反向边的流量 $f(v,u)$ 可能是一个负值。实际上我们可以注意到，在 Ford–Fulkerson 增广的过程中，真正有意义的是剩余容量 $c_f$，这表明给定的边 $e$ 还能够支撑多少流量，而 $f(v,u)$ 的绝对值是无关紧要的。我们可以将反向边流量的减少视为反向边剩余容量 $c_f(v,u)$ 的增加——这也与退流的意义相吻合——反向边剩余容量的增加意味着我们接下来可能通过走反向边来和原先正向的增广抵消，代表一种「反悔」的操作。

我们来看一下反悔操作是怎么进行的。

![[Pasted image 20260131182301.png]]

1. **第一次增广**  
   $G$ 上有多条增广路，其中，我们选择进行一次先后经过 $u,v$ 的增广（如左图所示），流量增加 $1$。
2. **发现问题**  
   我们注意到，如果进行中图上的增广，这个局部的最大流量会变成 $2$。但由于指向 $u$ 的边和从 $v$ 出发的边在第一次增广中耗尽了容量，此时我们无法进行中图上的增广。这意味着我们当前的流是不够优的，但局部可能已经没有其他（只经过原图中的边而不经过反向边的）增广路了。
3. **退流操作**  
   第一次增广后，退流意味着 $c_f(v,u)$ 增加了 $1$ 剩余容量，即相当于新增 $(v,u)$ 这条边，因此我们可以再进行一次先后经过 $p,v,u,q$ 的增广（如右图橙色路径所示）。无向边 $(u,v)$ 上的流量在两次增广中抵消，我们惊奇地发现两次增广叠加得到的结果实际上和中图是等价的。

这样的反悔保证了我们只要选尽了所有的增广路，无论按照何种次序选择，这种影响会被抵消掉。

于是，只要 $G_f$ 上存在增广路，只要进行增广操作，那流量不减，因此只要反复进行第二步操作，直到没有增广路，那我们就能够得到最大流。

因此，请记住，**以下所有 bfs / dfs 都是在 $G_f$ 上（或者由 $G_f$ 衍生的 $G_L$）进行的！**

为什么 Ford-Fulkerson 增广算法是正确的呢？这一正确性和最大流最小割定理等价。即 $\max |f|=\min||S,T||$。

直观上，正确性是显然的，因为最小割实际上就是网络中的**最细瓶颈**。最大流无法超过这个瓶颈，而增广路算法本质上就是在不断填满这个瓶颈，直到无法通过为止。

>[!proof]- 证明
>为了证明最大流最小割定理，我们先从一个引理出发：对于网络 $G = (V, E)$，任取一个流 $f$ 和一个割 $\{S, T\}$，总是有 $|f| \leq ||S, T||$，其中等号成立当且仅当 $\{(u, v) \mid u \in S, v \in T\}$ 的所有边均满流，且 $\{(u, v) \mid u \in T, v \in S\}$ 的所有边均空流．
>
>$$
\begin{aligned}
|f| & = f(s) \\
& = \sum_{u \in S} f(u) \\
& = \sum_{u \in S} \left( \sum_{v \in V} f(u, v) - \sum_{v \in V} f(v, u) \right) \\
& = \sum_{u \in S} \left( \sum_{v \in T} f(u, v) + \sum_{v \in S} f(u, v) - \sum_{v \in T} f(v, u) - \sum_{v \in S} f(v, u) \right) \\
& = \sum_{u \in S} \left( \sum_{v \in T} f(u, v) - \sum_{v \in T} f(v, u) \right) + \sum_{u \in S} \sum_{v \in S} f(u, v) - \sum_{u \in S} \sum_{v \in S} f(v, u) \\
& = \sum_{u \in S} \left( \sum_{v \in T} f(u, v) - \sum_{v \in T} f(v, u) \right) \\
& \leq \sum_{u \in S} \sum_{v \in T} f(u, v) \\
& \leq \sum_{u \in S} \sum_{v \in T} c(u, v) \\
& = ||S, T||
\end{aligned}
>$$
>
>为了取等，第一个不等号需要 $\{(u, v) \mid u \in T, v \in S\}$ 的所有边均空流，第二个不等号需要 $\{(u, v) \mid u \in S, v \in T\}$ 的所有边均满流．原引理得证．
>
>那么，对于任意网络，以上取等条件是否总是能被满足呢？如果答案是肯定的，则最大流最小割定理得证．以下我们尝试证明．
>
>假设某一轮增广后，我们得到流 $f$ 使得 $G_f$ 上不存在增广路，即 $G_f$ 上不存在 $s$ 到 $t$ 的路径．此时我们记从 $s$ 出发可以到达的结点组成的点集为 $S$，并记 $T = V \setminus S$．
>
>显然，$\{S, T\}$ 是 $G_f$ 的一个割，且 $||S, T|| = \sum_{u \in S} \sum_{v \in T} c_f(u, v) = 0$．由于剩余容量是非负的，这也意味着对于任意 $u \in S, v \in T, (u, v) \in E_f$，均有 $c_f(u, v) = 0$．以下我们将这些边分为存在于原图中的边和反向边两种情况讨论：
>
>1. $(u, v) \in E$：此时，$c_f(u, v) = c(u, v) - f(u, v) = 0$，因此有 $c(u, v) = f(u, v)$，即 $\{(u, v) \mid u \in S, v \in T\}$ 的所有边均满流；
>2. $(v, u) \in E$：此时，$c_f(u, v) = c(u, v) - f(u, v) = 0 - f(u, v) = f(v, u) = 0$，即 $\{(v, u) \mid u \in S, v \in T\}$ 的所有边均空流．
>
>因此，增广停止后，上述流 $f$ 满足取等条件．根据引理指出的大小关系，自然地，$f$ 是 $G$ 的一个最大流，$\{S, T\}$ 是 $G$ 的一个最小割．

平凡地，如果增广过程中，每次增广的流量都是整数，那么复杂度的上界是 $O(|E||f|)$，这是因为每次增广 $O(m)$，增广导致总流量至少上升 1，因此至多 $O(m|f|)$ 即可。

我们在下面这个图看一下 Ford-Fulkerson 方法实际的过程。

![[Pasted image 20260131183350.png]]

 - 第一次迭代，我们找到了 $s\to a\to t$ 这个增广路径，$f\leftarrow f+2$。
 - 第二次迭代，我们找到了 $s\to b\to t$ 这个增广路径，$f\leftarrow f+3$。
然后 $G_f$ 不存在 $s\to t$，因此 $|f|=5$。

![[Pasted image 20260131183645.png]]

 - 第一次迭代，我们找到了 $s\to a\to b\to t$，$f\leftarrow f+1$，同时我们更新 $c_f$，相当于添加了一条 $b\to a$ 的 $c_f=1$ 的边。
 - 第二次迭代，我们找到了 $s\to b\to a\to t$，$f\leftarrow f+1$，此时 $c_f(b,a)\leftarrow c_f(b,a)-1=0$。
 - $G_f$ 上不再存在 $s\to t$，结束，$|f|=2$。

不难发现，最终反向路径的 $c_f$ 就是实际流量。

## 实现 1：Edmonds-Karp

在实现上，选择增广路径时，我们可以选择 bfs 和 dfs 两种方法。但是，dfs 容易陷入长时间的迭代，更有可能达到 $O(|E||f|)$ 的上界。

>[!hint]- 为什么？
>![[Pasted image 20260131184124.png]]
>在 (b) (c) 中，dfs 会反复选择 $s\to b\to a\to t$ 和 $s\to a\to b\to t$ 这样的路径，重复 200 次。
>但如果采用 bfs，几次就够了。

因此，通常选择 bfs 实现。这时候，我们就得到了 **Edmonds-Karp 算法**。流程如下：
 1. 如果 $G_f$ 上可以 $s\overset{\rm bfs}{\longrightarrow}t$，我们就找到了一条增广路。
 2. 对 $p:s\to t$，计算 $\delta=\min_{(u,v)\in p} c_f(u,v)$，给每条边 $e\in p$ 都加上 $\delta$ 流量，即 $f(e)\leftarrow f(e)+\delta$；每条反向边 $\bar e$  都退掉 $\delta$ 流量，即 $f(\bar e)\leftarrow f(\bar e)-\delta$。这样，最大流 $f(s)$ 增加了 $\delta$。
 3. 这样我们自然得到了新的 $G_f$（不必可以更新，在 bfs 的时候选取 $c_f>0$ 的即可）。重复上述过程直到不存在 $s\to t$。

单轮 bfs 时间复杂度 $O(m)$，我们可以证明这样反复 bfs 最短路增广轮数的上界为 $O(nm)$。因此总时间复杂度为 $O(nm^2)$。在邻接矩阵存图的情况下，甚至会退化到 $O(n^3m)$。下面是一个可能实现。

```cpp
#include <bits/stdc++.h>

using namespace std;

#define int long long

const int MAXN = 205;

int n, m;
int flow[MAXN], pre[MAXN];
int graph[MAXN][MAXN];

int bfs(int s, int t) {
    memset(pre, -1, sizeof(pre));
    flow[s] = LLONG_MAX, pre[s] = 0;
    queue<int> q;
    q.push(s);
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        if (u == t) break;
        for (int i = 1; i <= n; i++) {
            if (i != s && graph[u][i] > 0 && pre[i] == -1) {
                q.push(i);
                pre[i] = u;
                flow[i] = min(flow[u], graph[u][i]);
            }
        }
    }
    if (pre[t] == -1) return -1;
    return flow[t];
}

int maxflow(int s, int t) {
    int ret = 0;
    while (true) {
        int f = bfs(s, t);
        if (f == -1) break;
        ret += f;
        int cur = t;
        while (cur != s) {
            graph[pre[cur]][cur] -= f, graph[cur][pre[cur]] += f;
            cur = pre[cur];
        }
    }
    return ret;
}

signed main() {
    int s, t;
    cin >> n >> m >> s >> t;
    for (int i = 1; i <= m; i++) {
        int u, v, w;
        cin >> u >> v >> w;
        graph[u][v] += w;
    }
    cout << maxflow(s, t) << endl;
    return 0;
}
```
## 实现 2：Dinic

回顾 Edmonds-Karp 算法中，我们试图通过反复 bfs 来不断寻找最短路。这个能否优化？我们期待一次找到多条增广路径。这就是所谓的**多路增广**。

通过前面 Edmonds-Karp 中我们发现，上界为 $O(nm)$ 的一个条件是**每次都找 $G_f$ 中的最短路**。因此，我们可以进行所谓的 bfs 分层。

什么意思呢？
 - 在 $G_f$ 中，我们可以求 $d(u)$ 为 $u$ 到 $s$ 的最短距离；
 - 所有的最短增广路都满足沿着路径，每一步都走到下一层。

因此，我们删除掉所有不满足 $d(v)=d(u)+1$ 的边 $(u,v)$。这样的图叫做分层图 $G_L$。

在构建分层图之后，我们下一步是期待在 $G_L$ 上找到一个极大的增广流 $f_b$，使得 $f_b$ 是 $G_L$ 的一个最大流。我们把这样的 $f_b$ 叫做**阻塞流**。

在如上定义的情况下，Dinic 算法由以下四步构成：
1. 在 $G_f$ 上通过 bfs 构建 $G_L$。
2. 在 $G_L$ 上通过 dfs 获得 $f_b$。
3. $f\leftarrow f+f_b$。
4. 重复以上过程，直到不存在 $s\to t$。

我们先来分析时间复杂度。bfs 构建 $G_L$ 是 $O(m)$，可以证明在接下来讲解的优化下，单次增广 dfs 复杂度为 $O(nm)$，因此增广过程复杂度为 $O(nm)$。可以证明，增广轮数是 $O(n)$ 的。因此总的时间复杂度是 $O(n^2m)$，优于 Edmonds-Karp 算法。

>[!remark] 注
>篇幅原因，这里的证明详见 [OI Wiki](https://oi-wiki.org/graph/flow/max-flow/) 的讲解。

在 dfs 中，我们需要进行所谓的**当前弧优化**，这是保证 dfs 时间复杂度是 $O(nm)$ 的必要优化。

注意到，在 $G_L$ 上 dfs 的过程中，考察一个节点 $u$，如果对每个入边都遍历一遍出边的话，那在这里局部是 $O(m^2)$ 的。那怎么办呢？如果某个时刻 $(u,v)$ 已经增广到极限了，即 $c_f(u,v)=0$ 或者 $v$ 的后侧已经是阻塞了，那我们没有必要再尝试 $(u,v)$ 了。

因此，实现上，我们去维护出边中第一条有必要尝试的边 `cur[u]`，习惯上称之为**当前弧**。实现代码如下。

当前弧优化的保障是，只要我每次跑的都是最短路，并且当前这条边后面的都取完了，以后就不可能会有别的情况在跑过来抢当前这条路。

```cpp
#include <bits/stdc++.h>
using namespace std;

#define rep(i, a, b) for (int i = a; i < (b); ++i)
#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;

const int MAXN = 205, MAXM = 5005;

int n, m, s, t;
int head[MAXN], cur[MAXN], nxt[MAXM << 1], to[MAXM << 1],
    w[MAXM << 1], ecnt;
int dep[MAXN];
ll mf;

void add_edge(int u, int v, int c) {
    nxt[ecnt] = head[u], head[u] = ecnt, to[ecnt] = v, w[ecnt] = c;
    ecnt++;
}

bool bfs() {
    queue<int> q;
    q.push(s);
    memset(dep, -1, sizeof(dep));
    dep[s] = 0;
    cur[s] = head[s];
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        for (int e = head[u]; ~e; e = nxt[e]) {
            int v = to[e];
            if (dep[v] == -1 && w[e] > 0) {
                dep[v] = dep[u] + 1;
                cur[v] = head[v];
                q.push(v);
            }
        }
    }
    return dep[t] != -1;
}

ll dfs(int u, int flow) {
    if (u == t || (!flow)) return flow;
    int ret = 0;
    for (int &e = cur[u]; ~e; e = nxt[e]) {
        // 这里直接引用，思考：如果下面的循环能够跑完
        // 也就是 ret 没有够到 flow，这说明 dfs(v,min(..))
        // 把剩下的压榨干净了，也不足够提供 ret 到 flow
        // 的。而后续操作中，既然这里已经是 f_b 了
        // 就不用再访问了，因此 e=nxt[e]，相应的 cur[u] 也变成了
        // nxt[e]
        int v = to[e], tmp;
        if (dep[v] == dep[u] + 1 &&
            (tmp = dfs(v, min(flow - ret, w[e])))) {
            ret += tmp;
            w[e] -= tmp, w[e ^ 1] += tmp;
            if (ret == flow) return flow;
        }
    }
    return ret;
}

void maxflow() {
    while (bfs()) { mf += dfs(s, INT_MAX >> 1); }
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    cin >> n >> m >> s >> t;
    memset(head, -1, sizeof(head));
    rep(i, 0, m) {
        int u, v, c;
        cin >> u >> v >> c;
        add_edge(u, v, c), add_edge(v, u, 0);
    }
    maxflow();
    cout << mf << endl;
    return 0;
}
```

诚然，一般情况下，Dinic 算法的时间复杂度是 $O(n^2m)$。但是，在一些性质良好的图上，Dinic 算法的时间复杂度更好。

比如，单位图（也就是 $c(u,v)\in\{0,1\}$ 恒成立）的图上，Dinic 算法单轮增广的时间复杂度变成了 $O(m)$，这是因为，每次增广都会导致增广路上的所有边均饱和并消失，故单轮增广中每条边只能被增广一次。而增广次数变成了惊人的 $O(\min\{n^{2/3},m^{1/2}\})$，总的时间复杂度变成 $O(m\min\{n^{2/3},m^{1/2}\})$，十分优秀。这也就说明，对于二分图最大匹配问题，尽管我们常使用 Hopcroft–Karp 算法解决，但是这一算法实际上是 Dinic 算法在满足上述度数限制的单位容量网络上的特例。

当然，反向边也可以通过邻接表存储。可以看下面的 Rust 实现。

```rust
#[derive(Clone)]
struct Edge {
    to: usize,
    cap: i32,
    rev: usize,
}

struct Dinic {
    graph: Vec<Vec<Edge>>,
    level: Vec<i32>,
    iter: Vec<usize>,
}

impl Dinic {
    fn new(n: usize) -> Self {
        Self {
            graph: vec![Vec::new(); n + 1],
            level: vec![-1; n + 1],
            iter: vec![0; n + 1],
        }
    }

    fn add_edge(&mut self, u: usize, v: usize, cap: i32) {
        let u_len = self.graph[u].len();
        let v_len = self.graph[v].len();
        self.graph[u].push(Edge {
            to: v,
            cap,
            rev: v_len,
        });
        self.graph[v].push(Edge {
            to: u,
            cap: 0,
            rev: u_len,
        });
    }

    fn bfs(&mut self, s: usize, t: usize) -> bool {
        self.level.fill(-1);
        self.level[s] = 0;
        let mut queue = VecDeque::new();
        queue.push_back(s);

        while let Some(u) = queue.pop_front() {
            for edge in &self.graph[u] {
                if edge.cap > 0 && self.level[edge.to] < 0 {
                    self.level[edge.to] = self.level[u] + 1;
                    queue.push_back(edge.to);
                }
            }
        }
        self.level[t] != -1
    }

    fn dfs(&mut self, u: usize, t: usize, flow: i32) -> i32 {
        if u == t || flow == 0 {
            return flow;
        }

        for i in self.iter[u]..self.graph[u].len() {
            self.iter[u] = i; // 更新当前弧
            let edge_to;
            let edge_cap;
            let edge_rev;

            {
                let e = &self.graph[u][i];
                edge_to = e.to;
                edge_cap = e.cap;
                edge_rev = e.rev;
            }

            if self.level[u] < self.level[edge_to] && edge_cap > 0 {
                let d = self.dfs(edge_to, t, flow.min(edge_cap));
                if d > 0 {
                    self.graph[u][i].cap -= d;
                    self.graph[edge_to][edge_rev].cap += d;
                    return d;
                }
            }
        }
        0
    }

    fn max_flow(&mut self, s: usize, t: usize) -> i64 {
        let mut flow: i64 = 0;
        while self.bfs(s, t) {
            self.iter.fill(0);
            loop {
                let f = self.dfs(s, t, i32::MAX);
                if f == 0 {
                    break;
                }
                flow += f as i64;
            }
        }
        flow
    }
}
```

## 实现 3：ISAP 算法

Edmonds-Karp 和 Dinic 都采用了最短增广路径（Shortest Augmenting Path）的思路，因此也称作 SAP 算法。下面介绍的 ISAP（Improved SAP）算法是对 Dinic 的进一步改进。

在 Dinic 中，我们反复跑 bfs，每次进行一次分层。ISAP 中，我们仅做一次 bfs 分层，并且重复寻找增广路径。但是在寻找一次增广路径后，$G_f$ 必然发生更改，因此需要更新层次。由于每次仅仅更新路径上的点，因此 ISAP 效率优于 Dinic 算法。

![[Pasted image 20260201023422.png]]

ISAP 的算法步骤如下：
1. 在反图上（即从 $t$ 开始反向）跑 bfs 进行分层，存在 `dep` 数组中，规定 `dep[t]=1`；
2. 从源点 $s$ 开始，按照高度递减的顺序，找 $s\to t$ 的增广路径。我们只选择比当前点层数少 1 的点来增广。
3. 在增广的过程中，我们同步完成重分层的过程。具体来讲，在结束 $i$ 号点的增广过程中，遍历 $i$ 在 $G_f$ 上的所有出边，找到 $dep$ 最小的 $j$，然后 $deg_i\leftarrow dep_j+1$。特别地，如果 $i$ 在 $G_f$ 上无出边，则 $dep_i\leftarrow n$。然后，重置**当前弧**。
	- 这是因为，有可能有一些 $dep$ 偏大的在当前弧的前面，而之前的已经跳过了这些了。
4. 重复直径上述操作，直到出现断层，或者说 $d_s\geq n$。
	- 显然后者是前者的充分条件，而前者出现断层说明 $G_f$ 上不存在最短路了，这是因为后续过程中如果从 $s$ 出发，无法越过这个断层。

和 Dinic 类似，ISAP 中也存在当前弧优化。ISAP 的另外一个优化就是所谓的 **GAP 优化**。

```cpp
#include <bits/stdc++.h>
using namespace std;

#define rep(i, a, b) for (int i = a; i < (b); ++i)
#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;

const int MAXN = 205, MAXM = 5005;

int n, m, s, t;
int head[MAXN], cur[MAXN], nxt[MAXM << 1], to[MAXM << 1],
    from[MAXM << 1], w[MAXM << 1], ecnt;
int dep[MAXN], gap[MAXM], pre[MAXM << 1];

void add_edge(int u, int v, int c) {
    nxt[ecnt] = head[u], head[u] = ecnt, from[ecnt] = u, to[ecnt] = v,
    w[ecnt] = c;
    ecnt++;
}

void bfs() {
    queue<int> q;
    q.push(t);
    memset(dep, 0, sizeof(dep));
    dep[t] = 1;
    while (!q.empty()) {
        int u = q.front();
        q.pop();
        gap[dep[u]]++;
        for (int e = head[u]; ~e; e = nxt[e]) {
            int v = to[e];
            if (dep[v] == -1 && w[e ^ 1]) {
                dep[v] = dep[u] + 1;
                q.push(v);
            }
        }
    }
}

ll augment() {
    int u = t;
    ll ret = LLONG_MAX;
    while (u != s) {
        int e = pre[u];
        ret = min(ret, ll(w[e]));
        u = from[e];
    }
    u = t;
    while (u != s) {
        int e = pre[u];
        w[e] -= ret, w[e ^ 1] += ret;
        u = from[e];
    }
    return ret;
}

void ISAP() {
    bfs();
    int u = s;
    ll flow = 0;
    memcpy(cur, head, sizeof(head));
    while (dep[s] <= n) {
        if (u == t) {
            flow += augment();
            u = s;
        }
        bool ok = false;
        for (int &e = cur[u]; ~e; e = nxt[e]) {
            int v = to[e];
            if (dep[v] == dep[u] - 1 && w[e] > 0) {
                ok = true;
                pre[v] = e;
                u = v;
                break;
            }
        }
        if (!ok) {
            if (!--gap[dep[u]]) break;
            int mn = n + 10;
            for (int e = head[u]; ~e; e = nxt[e]) {
                int v = to[e];
                if (w[e]) mn = min(dep[v], mn);
            }
            cur[u] = head[u];
            dep[u] = mn + 1;
            gap[dep[u]]++;
            if (u != s) u = from[pre[u]];
        }
    }
    cout << flow << endl;
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    cin >> n >> m >> s >> t;
    memset(head, -1, sizeof(head));
    rep(i, 0, m) {
        int u, v, c;
        cin >> u >> v >> c;
        add_edge(u, v, c), add_edge(v, u, 0);
    }
    ISAP();
    return 0;
}
```

## 预流推进（Push-Relabel）算法思路

这个算法和前面的 Ford-Fulkerson 算法大相径庭。

>如果说 ISAP/Dinic 是“找路送水”（基于路径），那么 Push-Relabel 就是“在坡道上推水”（基于局部点）。

什么意思呢？我们拿水塔打比方，在 Push-Relabel 中：
- 我们把源点 $s$ 看作一个**无限大的水塔**，它一开始就把所有相连的管道全部灌满（初始化）。
- 水流进中间节点后，这些点就会“溢出”。
- 溢出的点会想办法把手里的水推给**比自己低**（高度 $h(u) = h(v) + 1$）的邻居。
- 如果周围都比自己高，水送不出去，该点就会“长高”（Relabel），直到能把水送出去为止。
- **最终结果：** 送到汇点 $t$ 的水就是最大流；送不到 $t$ 的水最终会因为高度不断增加，被“反向退回”给源点 $s$。

体感上和 ISAP 比较类似。感性理解之后我们来看一下实际写法。

预流推进算法通过对单个结点的更新操作，直到没有结点需要更新来求解最大流。算法过程维护的流函数不一定保持流守恒性，对于一个结点，我们允许进入结点的流超过流出结点的流，同时定义所谓的 **超额流** $e(u)$：
$$
e(u)=\sum_{(x,u)\in E} f(x,u)-\sum_{(u,y)\in E}f(u,y).
$$
如果 $e(u)>0$，则称 $u$ **溢出**。请注意这里关于超额流和溢出的讨论不包括 $s,t$。

Push-Relabel 主要维护的函数是所谓的 **高度函数** $h(u)$。我们规定超额流超出的部分只能向高度更小的节点扔。我们形式化地定义 $h:V\to\mathbb N$。

称 $h:V\to\mathbb N$  为 **高度函数**，如果 $h(s)=n, h(t)=0$，并且对 $(u,v)\in E_f$，总有 $h(u)\leq h(v)+1$。也就是说，如果水从 $u$ 流向 $v$，当且仅当 $h(u)-h(v)=1$。

Push-Relabel 主要由以下三步构成：
1. 初始化。起始规定 $f(u,v)=\delta_{us} \cdot c(u,v)$，$h(u)=\delta_{us}\cdot n$，$e(u)$ 按定义计算，这里 $\delta$ 表示 Kronecker 记号。
	- 实际上，上述操作仅仅进行了将 $(s,v)\in E$ 充满，并且将 $h(s)$ 抬高。
2. Push：对 $e(u)>0$ 的节点，如果存在 $v$ 满足 $c_f(u,v)>0$，且 $h(u)=h(v)+1$，则执行 push 操作，把 $\min\{e(u), c_f(u,v)\}$ 的超额通过 $(u,v)$ 传递到 $v$，并更新 $G_f$。
3. Relabel：对 $e(u)>0$ 的节点，如果每个邻居都比 $u$ 高，就更新 $h(u)\leftarrow\min_{(u,v)\in E_f}\{h(v)\}+1$。
4. 重复 Push-Relabel，直到无法操作为止。

此时，$e(t)$ 就是最大流。不难发现，这也等价于 $-e(s)$。如果朴素实现，即每次扫描整个顶点集来执行 Push-Relabel，那么时间复杂度是 $O(n^2m)$。这是因为 Relabel 的上界是 $O(n^2)$，Push 的上界是 $O(n^2m)$。一个实现如下。

```cpp
void init() {  // 初始化
  for (int i = h[s]; i; i = e[i].nex) {
    const int &v = e[i].t;
    ex[v] = e[i].v, ex[s] -= ex[v], e[i ^ 1].v = e[i].v, e[i].v = 0;
  }
  ht[s] = n;
}

bool push(int ed) {
  const int &u = e[ed ^ 1].t, &v = e[ed].t;
  int flow = min(ex[u], e[ed].v);
  ex[u] -= flow, ex[v] += flow, e[ed].v -= flow, e[ed ^ 1].v += flow;
  return ex[u];  // 如果 u 仍溢出，返回 1
}

void relabel(int u) {
  ht[u] = INF;
  for (int i = h[u]; i; i = e[i].nex)
    if (e[i].v) ht[u] = min(ht[u], ht[e[i].t]);
  ++ht[u];
}
```
### 实现：HLPP 算法

HLPP 是 Push-Relabel 的一个高效实现，我们可以在 $O(n^2\sqrt m)$ 的时间内获得最大流。

具体来讲，HLPP 的算法过程如下：
1. 初始化，与 Push-Relabel 相同；
2. 选择高度最高的节点 $u$，push 一次；
3. 如果 $e(u)>0$，对它 relabel，重复操作 2；
4. 如果没有溢出的节点，算法结束。

但是，这里初始化的时候我们可以进行优化。我们从 $t$ 跑一遍反图的 bfs，规定 $h(u)$ 为 $u$ 到 $t$ 的最短距离；但是特别地，$h(s)=n$。做了 bfs 后，每个点起手就知道自己离 $t$ 有多远。省去了中间无数次无效的 relabel。

>[!hint]+ 为什么要规定 $h(s)=n$？
>在残量网络中，水流有两个去处：
>1. 流向汇点 **$t$**（我们的目标）。
>2. 流回源点 **$s$**（表示这些水流不过去，得还回去）。
>
>但是，请注意一件事情，如果满足高度函数的定义，那么**残量网络中一定不存在从 $s$ 到 $t$ 的增广路。
>**
>假设存在一条从 $s$ 到 $t$ 的增广路：$s \to v_1 \to v_2 \to \dots \to t$。这条路径上的点数最多为 $n$ 个，那么路径上的边数最多为 $n-1$ 条。根据定义，每条边 $(u, v)$ 都满足 $h(u) - h(v) \leq 1$。我们将整条路径的不等式累加：$(h(s) - h(v_1)) + (h(v_1) - h(v_2)) + \dots + (h(v_{k}) - h(t)) \leq \text{边数}$，抵消中间项后得到：$h(s) - h(t) \leq n - 1$。但定义规定 $h(s) = n, h(t) = 0$，所以 $h(s) - h(t) = n$。矛盾！
>
>特别地，我们发现上面把 $s$ 换成任意一个节点 $u\in V$ 都可行，因此一旦 $h(u) \ge n$，$u$ 到汇点 $t$ 就已经完全不连通了。这时候相当于我们在“退货”，把水倒灌给 $s$。
>
>水可以从 $s$ 流向邻居，但邻居想要把水“退货”给 $s$，高度必须涨到 $n+1$。这就迫使溢出的水在尝试退回之前，先在图里尽可能地向 $t$ 推进。

我们同样可以实现当前弧优化。但是鉴于立刻就要 relabel 了，relabel 之后就要清空，似乎意义不大。

类似，也有 **GAP 优化**。一旦某个时刻，$h(u)=k$ 的节点个数为 0，那么考虑 $h(u)>k$ 的节点（必然有一个是这个时刻刚刚从 $k$ 跳上来变成 $k+1$ 的），因为我们必然是从最高的结点开始，此时都无法推送到 $h(u)<k$ 的部分。此时我们把这些部分直接让他们的高度变成至少 $n+1$，以便快速回流到 $s$。

![[Pasted image 20260201035401.png]]

一个实现方式是 KACTL 库中的实现。如下。

```cpp
struct PushRelabel {
    struct Edge {
        int dest, back; // 目的地、反向边编号
        ll f, c;        // 流量、容量
    };
    vector<vector<Edge>> g;
    vector<ll> ec; // 前面定义的 e[u]
    vector<Edge *> cur; // 当前弧
    vector<vi> hs; // 桶。hs[i] 存储了所有 h=i 的下标
    vi H;
    PushRelabel(int n) : g(n), ec(n), cur(n), hs(2 * n), H(n) {}

    void addEdge(int s, int t, ll cap, ll rcap = 0) {
        if (s == t) return;
        g[s].push_back({t, sz(g[t]), 0, cap});
        g[t].push_back({s, sz(g[s]) - 1, 0, rcap});
    }

    void addFlow(Edge &e, ll f) {
        Edge &back = g[e.dest][e.back];
        if (!ec[e.dest] && f) hs[H[e.dest]].push_back(e.dest);
        e.f += f;
        e.c -= f;
        ec[e.dest] += f;
        back.f -= f;
        back.c += f;
        ec[back.dest] -= f;
    }
    
    ll calc(int s, int t) {
        int v = sz(g);
        H[s] = v;
        ec[t] = 1;
        vi co(2 * v);
        co[0] = v - 1; // 高度计数，等价于 ISAP 中的 gap 数组
        rep(i, 0, v) cur[i] = g[i].data(); // 相当于获得 head，
					        // 这样即使是在邻接表我们也可以快速通过 cur 遍历
        for (Edge &e : g[s]) addFlow(e, e.c); // 初始化填满

        for (int hi = 0;;) { // 从当前最高开始。这个代码没有实现 bfs 优化
            while (hs[hi].empty())
                if (!hi--) return -ec[s]; // 如果存在 gap，直接返回最大流
            int u = hs[hi].back();
            hs[hi].pop_back();
            while (ec[u] > 0) // e[u]>0，进行 push-relabel 操作
                if (cur[u] == g[u].data() + sz(g[u])) {
	                // 当前弧遍历到末尾了，说明邻居 H 都比 u 大，进行 relabel
                    H[u] = 1e9; // 设定极大值
                    for (Edge &e : g[u])
                        if (e.c && H[u] > H[e.dest] + 1)
                            H[u] = H[e.dest] + 1, cur[u] = &e;
								// 下次遍历直接从第一个最小的开始
                    if (++co[H[u]], !--co[hi] && hi < v)
							// 逗号运算符返回后面一个值
							// 这里相当于 gap 优化，如果断层了
							// 那就全都设置成 v+1，也就是都达不到
                        rep(i, 0, v)
							if (hi < H[i] && H[i] < v) --co[H[i]], 
													   H[i] = v + 1;
                    hi = H[u];
                } else if (cur[u]->c && H[u] == H[cur[u]->dest] + 1)
                    addFlow(*cur[u], min(ec[u], cur[u]->c));
                    // 否则执行增广操作，推流。
                else
                    ++cur[u]; // 遍历下一个边
        }
    }
    bool leftOfMinCut(int a) { return H[a] >= sz(g); } 
    // 在最小割中是否和 s 同侧。如果 H[a]>=v，说明这个点不能到t，因此肯定流满了
    // 进而和 s 同侧。反之和 t 同侧。
};
```

# 最小割

## 全局最小割问题

显然，如果枚举两点跑 Dinic，那复杂度将会是不可接受的 $O(n^3m)$。

但是在**无向图**上，Stoer-Wagner 算法用惊人的 $O(n^3)$ 复杂度解决了这个问题。它**完全抛弃了“流”的概念**，只通过简单的**贪心加和**就能找到一个割。

Stoer-Wagner 算法流程如下：
- **初始化**：令当前最小割结果 $min\_cut = \infty$。
- **寻找 $s-t$ 最小割（Minimum Cut Phase）**：
    - 在当前图中，找出一对点 $s$ 和 $t$，以及它们之间的最小割。
    - 这个过程使用一种类似 Prim 算法的贪心策略（称为 **Maximum Adjacency Search**，MAS）。
	    - 维护一个集合 $A$，初始包含任意一个点。
		- 每次从 $V - A$ 中选择一个点加入 $A$，选择的标准是：该点与 $A$ 集合中所有点之间的**边权之和最大**。
		- **最后进入 $A$ 的两个点就是 $s$ 和 $t$。**
    - 在该阶段结束时，最后一个加入集合的点就是 $t$，倒数第二个是 $s$。
    - 此时，$t$ 到图其余部分的边权之和即为该阶段的 $s-t$ 最小割。
- **更新全局最优解**：如果当前阶段得到的 $s-t$ 最小割比 $min\_cut$ 小，则更新。
- **合并节点**：将 $s$ 和 $t$ 合并为一个点（合并边权），减少图中点的数量。
- **重复**：重复上述过程直到图中只剩下一个点。

样例实现如下。瓶颈位于 MAS，复杂度为 $O(n^2)$，如果用 `priority_queue` 可以优化到 $O(nm\log m)$，但是稠密图上不如 $O(n^2)$。加上 $n$ 次遍历，因此总复杂度为 $O(\min\{n^3,n^2m\log m\})$。

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

const int MAXN = 505;
const int INF = 1e9;

int edge[MAXN][MAXN]; // 邻接矩阵
int dist[MAXN];       // 存储点到集合 A 的权重之和
bool vis[MAXN];       // 标记点是否在集合 A 中
bool merged[MAXN];    // 标记点是否已被合并（删除）
int n, m;

int stoer_wagner(int n) {
    int min_cut = INF;
    
    // 剩下的节点数，直到只剩 1 个点为止
    for (int i = 0; i < n - 1; ++i) {
        int s = -1, t = -1;
        int max_weight = -1;
        
        // 初始化搜索状态
        for (int j = 1; j <= n; ++j) {
            vis[j] = false;
            dist[j] = 0;
        }

        // Maximum Adjacency Search (MAS)
        for (int j = 0; j < n - i; ++j) {
            int next_node = -1;
            int current_max = -1;

            for (int k = 1; k <= n; ++k) {
                if (!merged[k] && !vis[k]) {
                    if (dist[k] > current_max) {
                        current_max = dist[k];
                        next_node = k;
                    }
                }
            }

            if (next_node == -1) break;

            vis[next_node] = true;
            s = t;         // s 记录倒数第二个点
            t = next_node; // t 记录最后一个点
            max_weight = current_max;

            // 更新其余点到集合 A 的权重
            for (int k = 1; k <= n; ++k) {
                if (!merged[k] && !vis[k]) {
                    dist[k] += edge[next_node][k];
                }
            }
        }

        // 更新全局最小割（此时的 max_weight 即为 s-t 最小割）
        min_cut = min(min_cut, max_weight);

        // 合并点 t 到 s 中
        if (s != -1 && t != -1) {
            merged[t] = true;
            for (int k = 1; k <= n; ++k) {
                if (!merged[k]) {
                    edge[s][k] += edge[t][k];
                    edge[k][s] += edge[t][k];
                }
            }
        }
    }
    return min_cut;
}

int main() {
    // 示例输入：n 个点，m 条无向边
    if (!(cin >> n >> m)) return 0;

    for (int i = 0; i < m; ++i) {
        int u, v, w;
        cin >> u >> v >> w;
        edge[u][v] += w; // 处理重边
        edge[v][u] += w;
    }

    if (n < 2) cout << 0 << endl;
    else cout << "Global Minimum Cut: " << stoer_wagner(n) << endl;

    return 0;
}
```

## 用网络流解决混合图欧拉回路问题

# 上下界网络流
# 最小费用最大流

解决费用流，我们回顾 Ford-Fulkerson 思想，我们期待每次跑最短路，然后进行增广（当然，增广的依据就变为下一个和这个之间是否相差 cost）。因此，我们就可以将 EK / Dinic 与 SPFA 算法（当然，通过引入势[^1]，可以把 Dijkstra 算法纳入）进行结合。容易看出，就算每次查询流量加一，那复杂度也是 $O(nm+Fm\log m)$ 或者 $O(Fnm)$ 的。值得注意的是，这是一个关于值域的多项式，所以是伪多项式时间的。

请注意，如果有负环，那需要先消掉这个负环。

像下面这个代码（解决 [P1251](https://www.luogu.com.cn/problem/P1251)），就是 $O(nmF)$ 的实现。

```cpp
#include <bits/stdc++.h>

using std::max, std::min;
using std::vector, std::pair, std::string, std::cin, std::cout;

#define all(x) std::begin(x), std::end(x)
#define sz(x) (int) (x).size()

using i64 = long long;
using u64 = unsigned long long;
using i128 = __int128_t;
using u128 = __uint128_t;
using pii = pair<int, int>;
using vi = vector<int>;

template <typename T> T read(T) {
    T x = 0, f = 1;
    char ch = getchar_unlocked();
    while (!isdigit(ch)) {
        if (ch == '-')
            f *= -1;
        ch = getchar_unlocked();
    }
    while (isdigit(ch)) {
        x = x * 10 + ch - '0';
        ch = getchar_unlocked();
    }
    return x * f;
}

int N;
vi r;
int p, f, m, n, s;

const int MAXN = 4e3 + 5;
class MinCostMaxFlow {
    int head[MAXN], to[MAXN << 4], nxt[MAXN << 4];
    i64 cap[MAXN << 4], cst[MAXN << 4];

    i64 dis[MAXN], flow[MAXN];
    int pre[MAXN], ppre[MAXN];
    const i64 INF = LLONG_MAX >> 1;
    int s, t;

  public:
    MinCostMaxFlow(int s, int t) : s(s), t(t) { memset(head, -1, sizeof(head)); }
    void add_edge(int u, int v, i64 c, i64 w) {
        static int cnt = -1;
        nxt[++cnt] = head[u];
        head[u] = cnt;
        to[cnt] = v;
        cap[cnt] = c;
        cst[cnt] = w;
        nxt[++cnt] = head[v];
        head[v] = cnt;
        to[cnt] = u;
        cap[cnt] = 0;
        cst[cnt] = -w;
    }

    bool spfa() {
        std::fill(all(dis), INF);
        std::fill(all(pre), -1);
        std::fill(all(flow), INF);
        std::queue<int> q;
        q.push(s);
        dis[s] = 0;
        while (!q.empty()) {
            int u = q.front();
            q.pop();
            for (int e = head[u]; ~e; e = nxt[e]) {
                int v = to[e];
                i64 c = cap[e];
                if (c != 0 && dis[v] > dis[u] + cst[e]) {
                    dis[v] = dis[u] + cst[e];
                    flow[v] = min(flow[u], i64(c));
                    pre[v] = u;
                    ppre[v] = e;
                    q.push(v);
                }
            }
        }
        return dis[t] < INF;
    }

    i64 maxflow() {
        i64 ans = 0;
        while (spfa()) {
            ans += dis[t] * flow[t];
            for (int cur = t; cur != s; cur = pre[cur]) {
                int e = ppre[cur];
                cap[e] -= flow[t];
                cap[e ^ 1] += flow[t];
            }
        }
        return ans;
    }
};

const i64 INF = LLONG_MAX >> 1;

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    N = read(0);
    r.assign(N + 1, 0);
    for (int i = 1; i <= N; i++) {
        r[i] = read(0);
    }
    p = read(0), m = read(0), f = read(0), n = read(0), s = read(0);
    int t = N << 1 | 1;
    MinCostMaxFlow mcmf(0, t);
    for (int i = 1; i <= N; i++) {
        mcmf.add_edge(0, i + N, r[i], 0);
        mcmf.add_edge(i, t, r[i], 0);
        mcmf.add_edge(0, i, INF, p);
        if (i + m <= N)
            mcmf.add_edge(i + N, i + m, INF, f);
        if (i + n <= N)
            mcmf.add_edge(i + N, i + n, INF, s);
        if (i + 1 <= N)
            mcmf.add_edge(i + N, i + N + 1, INF, 0);
    }
    cout << mcmf.maxflow() << '\n';
    return 0;
}
```

```cpp
class MinCostMaxFlow {
    // ... 其他成员
    i64 h[MAXN];  // 势能数组
    bool vis[MAXN];
    
    bool dijkstra() {
        std::fill(all(dis), INF);
        std::fill(all(pre), -1);
        std::fill(all(flow), INF);
        std::fill(all(vis), false);
        
        using pli = std::pair<i64, int>;
        std::priority_queue<pli, std::vector<pli>, std::greater<pli>> pq;
        
        dis[s] = 0;
        pq.push({0, s});
        
        while (!pq.empty()) {
            auto [d, u] = pq.top();
            pq.pop();
            if (vis[u]) continue;
            vis[u] = true;
            
            for (int e = head[u]; ~e; e = nxt[e]) {
                int v = to[e];
                i64 c = cap[e];
                // 约化费用
                i64 reduced_cost = cst[e] + h[u] - h[v];
                if (c > 0 && dis[v] > dis[u] + reduced_cost) {
                    dis[v] = dis[u] + reduced_cost;
                    flow[v] = min(flow[u], c);
                    pre[v] = u;
                    ppre[v] = e;
                    pq.push({dis[v], v});
                }
            }
        }
        
        // 更新势能
        if (dis[t] < INF) {
            for (int i = 0; i <= n; i++) {
                if (dis[i] < INF) h[i] += dis[i];
            }
            return true;
        }
        return false;
    }
    
    i64 maxflow() {
        i64 ans = 0;
        // 初始用 SPFA 计算势能（处理初始的负边）
        std::fill(all(h), INF);
        h[s] = 0;
        std::queue<int> q;
        q.push(s);
        bool inq[MAXN] = {false};
        inq[s] = true;
        
        while (!q.empty()) {
            int u = q.front();
            q.pop();
            inq[u] = false;
            for (int e = head[u]; ~e; e = nxt[e]) {
                if (cap[e] > 0 && h[to[e]] > h[u] + cst[e]) {
                    h[to[e]] = h[u] + cst[e];
                    if (!inq[to[e]]) {
                        q.push(to[e]);
                        inq[to[e]] = true;
                    }
                }
            }
        }
        
        // 之后用 Dijkstra
        while (dijkstra()) {
            ans += (dis[t] + h[t] - h[s]) * flow[t];  // 真实费用
            for (int cur = t; cur != s; cur = pre[cur]) {
                int e = ppre[cur];
                cap[e] -= flow[t];
                cap[e ^ 1] += flow[t];
            }
        }
        return ans;
    }
};
```
# 与图匹配的联系

图匹配在 [[8. 图匹配问题]] 中会专门讲述。我们在此简述联系。

二分图 $(S,T)$ 匹配可以通过以下的方式转换为一个网络流问题：
 - 虚拟 $s,t$ 两个节点；
 - 连接 $(s,S)$，$(T,t)$，容量为 1；
 - 连接 $(S,T)$，容量为 1；
 - 跑最大流。
因为这是一个单位图，Dinic 的时间复杂度是 $O(m\min\{n^{2/3},m^{1/2}\})$，这也就是所谓的 Hopcroft–Karp 算法。

这背后是有**整数流**定理背书的。

[^1]: 具体来讲，我们先引入一个虚拟节点 0，然后从 0 开始向各个点跑一个最短路（用 spfa）获得路径长度 $h_i$，那从 $u$ 到 $v$ 权值为 $w$ 的边现在的新权值就是 $w+h_u-h_v$。可以类比势能的概念。认为 $h_i$ 就是 $i$ 的势能。具体细节参看 [OI Wiki](https://oi-wiki.org/graph/shortest-path)。
