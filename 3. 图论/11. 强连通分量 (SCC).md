#SCC 

## Tarjan 算法

如图所示，在图中可以相互到达的点就构成所谓的「强连通分量」（Strongly Connected Components）。

如果把所有的 SCC 都缩成一个点，我们就能够把一个图变成 DAG，从而简化问题。$\newcommand{\dfn}{\mathrm{dfn}}$

我们下面着重介绍 Tarjan 算法。Tarjan 算法把边分为 **树边、回边、弃边** 三种类型。
 - 树边：往下的点还没有分配 dfn 序号。即，这个节点第一次访问是被这条边贡献的。
 - 回边：往下的点已经分配了 dfn 序号，并且尚未分配 SCC。
 - 弃边：往下走的点已经分配了 dfn 序号，并且已经位于某个 SCC 了。我们直接忽略掉所有的弃边，因为弃边不会贡献新的 SCC。

本质上，我们是在图上的生成树（dfs 生成树）上进行递归地分类 SCC。在递归的过程中，我们要维护所谓的 $\newcommand{\low}{\mathrm{low}}\low$ 数组与 $\newcommand{\scc}{\mathrm{scc}}\scc$ 数组。
 - $\low_u$ 的定义是，$T_u\cup\{u\}$ 上的点，最多走一条回边，能够到达的树的最上方的点的 dfn，而且还得在栈中。
 - $\scc_u$ 的定义是，$u$ 分配的 SCC 序号。

Tarjan 算法的本质就是「扎口袋」，我们通过回边把「口袋」补充完整。如果一个节点能够被自己「扎起口袋」，感性理解，我们就得到了一个 SCC。我们下面具体阐述过程。
1. 初次来到 $u$ 时，$\dfn[u] = \low[u] =$ 分配的 dfn 序号，然后节点 $u$ 进栈  
2. 遍历 $u$ 的每个儿子 $v$，讨论 $u$ 和 $v$ 之间的边是什么类型：
	1. 如果是弃边，什么也不做  
	2. 如果是树边，先递归计算 $\low[v]$，然后 $\low[u] = \min(\low[u], \low[v])$  
	3. 如果是回边，直接 $\low[u] = \min(\low[u], \dfn[v])$。  
3. 遍历结束后，判断 $\dfn[u]$ 是否等于 $\low[u]$：
	1. 如果不等，代表节点 $u$ 不能「扎起口袋」。  
	2. 如果相等，代表节点 $u$ 能「扎起口袋」，栈弹出节点，直到 $u$ 出现时停止，这批节点属于一个强连通分量，分配对应的 $\scc$。
4. 考察图中每个节点，一旦发现 $\dfn[i] = 0$，就从 $i$ 节点开始，执行一遍 Tarjan 算法

可以看出，Tarjan 算法的时间复杂度就是对图跑一次 dfs 的时间复杂度，也就是 $O(n+m)$。

```cpp
#include <bits/stdc++.h>

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using namespace std;

using i64 = long long;
using u64 = unsigned long long;
using i128 = __int128_t;
using u128 = __uint128_t;
using pii = std::pair<int, int>;
using vi = vector<int>;

const int MAXN = 1e4 + 5, MAXM = 1e5 + 5;

int head[MAXN], nxt[MAXM << 1], to[MAXM << 1];
set<pii> st;

void add_edge(int u, int v) {
    static int cnt = 0;
    nxt[++cnt] = head[u];
    head[u] = cnt;
    to[cnt] = v;
}

int n, m;
int dfn[MAXN], scc[MAXN], low[MAXN];
int d, s;
stack<int> sta;
vector<vi> ans;

void tarjan(int u) {
    low[u] = dfn[u] = ++d;
    sta.push(u);
    for (int e = head[u]; e; e = nxt[e]) {
        int v = to[e];
        if (!dfn[v]) {
            tarjan(v);
            low[u] = min(low[u], low[v]);
        } else if (!scc[v]) {
            low[u] = min(low[u], dfn[v]);
        }
    }
    if (dfn[u] == low[u]) {
        s++;
        int v;
        do {
            v = sta.top();
            sta.pop();
            scc[v] = s;
        } while (v != u);
    }
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    cin >> n >> m;
    for (int i = 1; i <= m; i++) {
        int u, v;
        cin >> u >> v;
        add_edge(u, v);
    }
    for (int i = 1; i <= n; i++) {
        if (!dfn[i])
            tarjan(i);
    }
    cout << s << '\n';
    ans.assign(s + 1, vi());
    for (int i = 1; i <= n; i++) {
        ans[scc[i]].push_back(i);
    }
    sort(ans.begin() + 1, ans.end());
    for (int i = 1; i <= s; i++) {
        for (int j : ans[i])
            cout << j << ' ';
        cout << '\n';
    }
    return 0;
}
```

请注意，Tarjan 算法天然支持重边与自环。

>[!hint] 疑问
>在回边的时候，我们改为 $\low[u]=\min(\low[v],\low[u])$ 是否可行？
>
>答案是肯定的。但是此时 $\low$ 的含义发生了改变。我们原先的含义是「至多使用一条回边」，但是现在就变成了「可以使用多条回边」，但是我们没有穷尽所有的可能性，这是因为我们有一些支路在 `tarjan` 并没有访问到。这是更加模糊的，所以不推荐这么写。


>[!info]
> Tarjan 算法在处理过程中，实际上是按照某种 **逆拓扑序** 来发现强连通分量的，这是因为算法在深度优先搜索的过程中会先访问完那些没有出边的节点，而这与拓扑排序的过程是相反的．
>
> 如果我们将图中的所有强连通分量缩成单个节点，那么在这些缩点后的节点形成的 DAG 中进行拓扑排序，得到的顺序将与 Tarjan 算法给出的强连通分量的标号顺序相反．
>
> 因此，可以说，在缩点后的 DAG 中，**强连通分量（缩点后）的标号顺序是其拓扑序的逆序**．但要注意的是，这种说法仅在考虑了强连通分量之间的依赖关系（即从一个强连通分量到另一个强连通分量的有向边）时才成立．单个强连通分量内部的节点由于存在环，所以内部并不满足拓扑序的定义．
## 缩点

在跑完 SCC 之后，我们就可以把 SCC 缩成一个点去考虑。

我们跑一遍 SCC，然后在 SCC 之间连边考虑新的图。这就是所谓的「缩点」过程。

>[!example] P2341 [USACO03FALL / HAOI2006] 受欢迎的牛
> 每头奶牛都梦想成为牛棚里的明星。被所有奶牛喜欢的奶牛就是一头明星奶牛。所有奶牛都是自恋狂，每头奶牛总是喜欢自己的。奶牛之间的「喜欢」是可以传递的——如果 $A$ 喜欢 $B$，$B$ 喜欢 $C$，那么 $A$ 也喜欢 $C$。牛栏里共有 $N$ 头奶牛，给定一些奶牛之间的爱慕关系，请你算出有多少头奶牛可以当明星。
> 
> 对于 $100\%$ 的数据，$1\le N\le10^4$，$1\le M\le5\times 10^4$。

缩点之后考虑出度为 0 的点是否唯一。类似还有 [这道题](https://www.luogu.com.cn/problem/P2002)，我们判断缩点之后入度为 0 的点的个数即可。
