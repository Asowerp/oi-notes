#欧拉路径

欧拉路径本质上就是一笔画。我们想要找一个每个边都走一遍的路径。

## Hierholzer 算法

>[!definition] 基本定义
>- **欧拉路径**：每条边必须经过一次，且必须经过所有边所形成的路径。
>- **欧拉回路**：起点和终点是同一个点的欧拉路径。
>- **欧拉图**：存在欧拉回路的图。
>- **半欧拉图**：存在欧拉路径但不是回路的图。

有向图存在欧拉回路的条件是：
1. 忽略入度和出度都为 0 的点后，剩下的每个点满足：**入度 = 出度**。
2. 剩余图是**强连通**的（任意两点可以互相到达）。

>[!proof]- 证明
>1. 从任意点 $s$ 出发，遇到边就选一条继续走，直到无法再走，最终一定会回到 $s$，形成一条**主线**回路。
>2. 如果还有剩余的边没走到，那么这些边必然可以被主线上的某个点 $x$ 到达，因为图是强连通的。
>3. 从点 $x$ 出发，继续走未走过的边，按照步骤 1 的方式，最终一定会回到 $x$，形成一条**支线**回路。
>4. 将支线插入到主线中（在点 $x$ 处插入），拼接成一个更大的回路。
>5. 重复上述过程，主线不断插入支线，最终得到整个欧拉回路。

类似地，有向图存在欧拉路径的条件是：
1. 忽略入度和出度都为 0 的点以及起点终点后，剩下的每个点满足：**入度 = 出度**。
2. 起点满足出度=入度+1，终点满足入度=出度+1。
3. 从终点向起点连一条有向边，判断是否存在欧拉回路

对于**无向图**的欧拉路径与欧拉回路，同理：
1. 忽略度为 0 的点（孤立点），剩余部分图必须连通。
2. 统计图中度数为奇数的顶点个数，记为 odd。
3. 判断：  
   - 如果 odd = 0：存在 欧拉回路（可以回到起点）。  
   - 如果 odd = 2：存在 欧拉路径（不能回到起点）。  
   - 否则（odd 为其他值，如 4, 6, …）：既不存在欧拉回路，也不存在欧拉路径。

然而，实际操作中，根本不需要这么复杂。我们直接通过 Hierholzer 算法跑一遍就好。类似 [[7. 网络流问题]]，我们在 Hierholzer 算法中也存在当前弧优化，这是因为每一次我们向下跑，肯定是跑光了再回来的，所以一样可以使用当前弧优化。

1. 从顶点 $v$ 出发，在剩余的边中寻找一个回路（环），也就是在 `head[v]` 中往下调用。
2. 如果找不到回路（即 `circle` 为空）：
    - 直接返回当前顶点 $v$。
3. 否则：
    - 将找到的回路中的边从边集 $e$ 中移除。（相当于 `cur` 向后移动）
    - 对于回路 `circle` 中的每一个顶点 $v$：
        - 递归调用 `Hierholzer(v)`，并将返回的路径拼接在当前顶点之后。
    - 最后返回拼接好的完整路径（从当前回路开始）。

因此只要我们从正确的起点开始跑，一定就是能够得到一个欧拉路径（回路）。这个代码的时间复杂度就是 $O(|E|)$。
## 一些题目

>[!example] LC 2097
>给你一个下标从 **0** 开始的二维整数数组 `pairs` ，其中 `pairs[i] = [starti, endi]` 。如果 `pairs` 的一个重新排列，满足对每一个下标 `i` （ `1 <= i < pairs.length` ）都有 `endi-1 == starti` ，那么我们就认为这个重新排列是 `pairs` 的一个 **合法重新排列** 。
>
>请你返回 **任意一个** `pairs` 的合法重新排列。数据保证至少存在一个 `pairs` 的合法重新排列。

```cpp
const int MAXN = 2e5 + 5;

class Solution {
public:
    int head[MAXN], nxt[MAXN], to[MAXN], cur[MAXN], ecnt = 0;
    int ind[MAXN], outd[MAXN];
    vector<int> path;
    void add_edge(int u, int v) {
        nxt[++ecnt] = head[u], head[u] = ecnt, to[ecnt] = v;
    }
    void dfs(int u) {
        for (int& e = cur[u]; e;) {
            int v = to[e];
            e = nxt[e];
            dfs(v);
        }
        path.push_back(u);
    }
    vector<vector<int>> validArrangement(vector<vector<int>>& pairs) {
        vector<int> nums;
        vector<vector<int>> ans;
        for (auto& p : pairs) {
            nums.push_back(p[0]);
            nums.push_back(p[1]);
        }
        sort(nums.begin(), nums.end());
        nums.erase(unique(nums.begin(), nums.end()), nums.end());
        int n = nums.size();
        map<pair<int, int>, vector<int>> mp;
        for (auto& p : pairs) {
            int u = p[0], v = p[1];
            int nu =
                    lower_bound(nums.begin(), nums.end(), u) - nums.begin() + 1,
                nv =
                    lower_bound(nums.begin(), nums.end(), v) - nums.begin() + 1;
            mp[{nu, nv}] = {u, v};
            ind[nv]++, outd[nu]++;
            add_edge(nu, nv);
        }
        int s = 1;
        for (int i = 1; i <= n; i++) {
            cur[i] = head[i];
            if (outd[i] - ind[i] == 1)
                s = i;
        }
        dfs(s);
        reverse(path.begin(), path.end());
        for (int i = 0; i < path.size() - 1; i++) {
            ans.push_back(mp[{path[i], path[i + 1]}]);
        }
        return ans;
    }
};
```

>-   Choose an index $i$ ($1 \le i \le n$) and swap $a_i$ with $b_i$.
>
>You can perform the operation any number of times (possibly zero), but each index can be chosen by **at most one** operation. Your task is to make $a$ a rearrangement of $b$ after all operations, or state that it is impossible. You do not have to minimize the number of operations.

>
>为了创作音乐，机器人拥有一种特殊的乐器，能够产生 $n$ 种不同的声音。每种声音由其音量和音高来表征。一系列声音被称为音乐。如果任意两个连续的声音仅在音量或仅在音高上有所不同，则该音乐被认为是优美的。如果存在三个连续的声音在音量或音高上相同，则该音乐被认为是单调的。
>
>你需要创作一段优美且**不**单调的音乐，其中包含乐器产生的每个声音恰好一次。
