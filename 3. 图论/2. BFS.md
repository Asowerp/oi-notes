## BFS 的优化

> bfs、dfs 这种，往往你发现边数很多但是很特殊，点数不多的情况，都可以尝试用数据结构规避掉遍历边的步骤，改为直接找到该访问的点，因为这两个都是找到相邻的点直接加进队列就可以不考虑这些点了。
>
> dij 可能需要什么优化建图或者其他操作。
> ——namelessgugugu

如果我们要访问的区间是一段连续的区间，我们可以采用 `set` 优化的方法。

>[!example]
>给你一个二进制字符串 `s` 和一个整数 `k`。
>
>在一次操作中，你必须选择 **恰好** `k` 个 **不同的** 下标，并将每个 `'0'` **翻转** 为 `'1'`，每个 `'1'` 翻转为 `'0'`。
>
>返回使字符串中所有字符都等于 `'1'` 所需的 **最少** 操作次数。如果不可能，则返回 -1。

手玩几组样例可以发现，肯定和 $k$ 的奇偶性有点关系。假设当前这一步我们有 $val$ 个 1，那我们下一步可以到达：
$$
\{val-k, val-k+2,\cdots,val+k-2,val+k\}
$$
这一段连续的区间。如果直接 bfs 扫复杂度是 $O(nk)$ 级别的，其中 $n$ 为 `s` 的长度，这启示我们要进行优化。

因为这里的到达点很特殊（或者说，边很特殊），我们可以考虑维护尚未到达的点。利用 `set` 来记录哪些点还没有到过，并直接访问之。这样就可以把复杂度降到 $O(n\log n)$ 了。

代码如下：
```cpp
class Solution {
public:
    int minOperations(string s, int k) {
        int cnt = 0, n = s.size();
        for (auto ch : s)
            cnt += (ch - '0');
        if (k % 2 == 0 && (n - cnt) % 2)
            return -1;
        if (k == n && cnt != n && cnt != 0)
            return -1;
        queue<pair<int, int>> q;
        set<int> st[2];
        for(int i=1;i<=n;i++) st[i&1].insert(i);
        q.push({cnt, 0});
        st[cnt&1].erase(cnt);
        while (!q.empty()) {
            auto [val, step] = q.front();
            q.pop();
            if (val == n)
                return step;
            int l=max(k-2*val,-k),r=min(2*n-k-2*val,k);
            auto &tmp=st[(val+l)&1];
            auto it=tmp.lower_bound(val+l);
            while(it!=tmp.end()) {
                if(*it>val+r) break;
                q.push({*it,step+1});
                tmp.erase(it++);
            }
        }
        return -1;
    }
};
```
