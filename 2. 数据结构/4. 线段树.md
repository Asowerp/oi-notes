
**线段树的懒标记本质上是我们先更新这个区间，但是子区间没有更新的值，而之后用于延迟更新子区间！** 

>懒惰标记，简单来说，就是通过延迟对节点信息的更改，从而减少可能不必要的操作次数。每次执行修改时，我们通过打标记的方法表明该节点对应的区间在某一次操作中被更改，但不更新该节点的子节点的信息。实质性的修改则在下一次访问带有标记的节点时才进行。
>当你修改区间 $[L,R]$ 时：
> - 如果完全覆盖当前节点区间 $[l,r]$，就**只更新当前节点**，并打上标记 
> - 如果只覆盖一部分，就必须**下推标记给子节点**，因为子节点需要知道这个信息

正因如此，我们 push_down 写的才应该是更新下面子区间的 lazy 并清空当前的 lazy。

## zkw 线段树（迭代型线段树）

zkw 线段树的存储方式跟普通线段树的存储方式差异很小，都是堆式存储。但是空间上，一般的线段树要求 $O(4n)$ 的空间，zkw 线段树只需要 $O(2n)$ 的空间即可。不仅如此，zkw 线段树的常数也小一点。

### 初始化、单点修改与区间查询

但是，zkw 线段树要求建出来的树一定要是满二叉树（加一堆空节点硬凑）。

![[Pasted image 20251001232227.png]]

如上图，我们发现 $a[i]$ 被存储在 $s[i+n-1]$ 的位子上。剩下的位置只用从下往上，从左右孩子开始更新就好。于是可以很快写出下面的代码。

```cpp
void build(int n) { // 这里 n 得是确定好的 2^k！
    for (int i = 1; i <= n; i++) s[i + n - 1] = a[i];
    for (int i = n - 1; i >= 1; i--) s[i] = s[i << 1] + s[i << 1 | 1];
}
```

所以为了避免 $\log_2 n$ 不是整数的情况，我们可以贴心的写出下面这个代码：

```cpp
void build(int n) {
    for (m = 1; m < n + 2; m <<= 1); // 这里 m 一定是 2 的倍数，所以这上面一定是一个满二叉树，我们可以放心大胆的对下面进行操作。
    for (int i = m + 1; i <= m + n; i ++)
        s[i] = a[i - m];
    for (int i = m - 1; i; i--)
        s[i] = s[i << 1] + s[i << 1 | 1];
}
```

那么，单点修改就变得非常简单了，我们实际上只用修改每个祖先。

```cpp
void update(int pos, int val) {
    pos += m;
    s[pos] = val;
    for (int i = pos >> 1; i >= 1; i >>= 1) {
        s[i] = s[i << 1] + s[i << 1 | 1];
    }
}
```

当然如果是区间加的话还可以更简单：

```cpp
void add(int pos, int val) {
    for (pos += m; pos >= 1; pos >>= 1) {
        s[pos] += val;
    }
}
```

显然，根据前面的分析，不难发现单点查询在这里可以做到惊人的 $O(1)$ 查询，我们只需要返回 `s[pos+m]` 就好了。

因此，目前的关键在于区间查询。我们暂时先假设我们查询的信息合并都是满足交换律的。

![[Pasted image 20260219001857.png]]

假设我们要查询的是 $[l,r]$ 这个区间的信息。我们会这样拆成一堆树上的节点，使得这些节点恰好覆盖 $[l,r]$ 的所有元素。我们做出以下的观察：
1. 当 $l$ 是左孩子时，它（或者它的祖先）会被查询到。反之，它的祖先不会被查询到。
2. 当 $r$ 是右孩子时，它（或者它的祖先）会被查询到。反之，它的祖先不会被查询到。

因此，我们发现，当 $l$ 是左孩子时，可以**跳过当前节点，直接向上查找答案**。反之，我们需要将 $l$ 的答案单独计算，并将 $l\leftarrow l+1$；对 $r$ 同理，当 $r$ 是左孩子时，可以**跳过当前节点，直接向上查找答案**。反之，我们需要将 $r$ 的答案单独计算，并将 $r\leftarrow r-1$。

正因如此，每一步我们要么直接向上跳，要么横向走一步之后向上跳。这样，跳过相同的步数之后，我们一定可以跳到相同的节点。

我们同时还发现，左孩子都是偶数，右孩子都是奇数（在我们 1-indexed 的世界里）。那爽啦，以 P1531 为例，我们可以写出下面的代码。

>[!example] P1531
>给你 $n$，数列 $a$，你要实现两个操作：
> - 操作 1：修改。给定位置 $i$ 和值 $v$，你需要 $a_i\leftarrow \max(a_i,v)$；
> - 操作 2：查询。给定 $l,r$，你需要查询 $\max_{l\leq i\leq r}\{a_i\}$；

```cpp
#include <bits/stdc++.h>

using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

inline int read() {
    int x = 0, f = 1;
    char ch = getchar();
    while (ch < '0' || ch > '9') {
        if (ch == '-') f = -1;
        ch = getchar();
    }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + (ch - '0');
        ch = getchar();
    }
    return x * f;

const int MAXN = 2e5 + 5;

int n, m;
int a[MAXN];

struct Segtree {
    int n, m;
    vector<int> s;

  public:
    Segtree(int n, int *a) {
        this->n = n;
        for (m = 1; m < n; m <<= 1);
        s.assign(m + 2 << 1, 0); // 防止空间炸掉
        for (int i = 1 + m; i <= m + n + 1; i++) {
            s[i] = *(a + i - m);
        }
        for (int i = m - 1; i >= 1; i--) {
            s[i] = max(s[i << 1], s[i << 1 | 1]);
        }
    }
    void modify(int pos, int val) {
        pos += m;
        s[pos] = max(s[pos], val);
        for (pos >>= 1; pos >= 1; pos >>= 1) {
            s[pos] = max(s[pos << 1], s[pos << 1 | 1]);
        }
    }
    int query(int l, int r) {
        int ret = 0;
        for (l += m, r += m; l <= r; l >>= 1, r >>= 1) {
            if (l & 1) ret = max(ret, s[l++]);
            // 如果是奇数，说明是右子树，单独统计，然后再跳上去。
            // 因为肯定是上面一个节点统计了右边的值；反之直接跳上去。
            if (r & 1 ^ 1) ret = max(ret, s[r--]);
            // 类似。
        }
        return ret;
    }
};

    for (int i = 1; i <= m; i++) {
        char op;
        cin >> op;

            int l, r;
            cin >> l >> r;
            cout << st.query(l, r) << '\n';

            int pos, v;

        }
    }
    return 0;
}
```

当然，有些时候可能我们维护的信息并不满足结合律。但是呢，我们可以把我们要用到的状态塞进一个 `Node`，然后把 ret 也设计成 `Node`，再写一个 `combine` 就好了！

这样的一个例子就是下面这道题。

>[!example] LC 3777
>给你一个长度为 `n` 的字符串 `s`，其中仅包含字符 `'A'` 和 `'B'`。
>
>你还获得了一个长度为 `q` 的二维整数数组 `queries`，其中每个 `queries[i]` 是以下形式之一：
>
>- `[1, j]`：**反转** `s` 中下标为 `j` 的字符，即 `'A'` 变为 `'B'`（反之亦然）。此操作会修改 `s` 并影响后续查询。
>- `[2, l, r]`：**计算** 使 **子字符串** `s[l..r]` 变成 **交替字符串** 所需的 **最小** 字符删除数。此操作不会修改 `s`；`s` 的长度保持为 `n`。
>
>如果 **子字符串** 中不存在两个 **相邻** 字符 **相等** 的情况，则该子字符串是 **交替字符串**。长度为 1 的子字符串始终是交替字符串。
>
>返回一个整数数组 `answer`，其中 `answer[i]` 是第 `i` 个类型为 `[2, l, r]` 的查询的结果。
>
>**子字符串** 是字符串中一段连续的 **非空** 字符序列。

实例代码如下：

```cpp
#include <bits/stdc++.h>
using namespace std;

struct Info {
    int cnt; // 相邻相等对数
    int lc;  // left char (0/1)
    int rc;  // right char (0/1)
    bool empty;
};

static inline Info make_empty() { return {0, 0, 0, true}; }

static inline Info mergeInfo(const Info& a, const Info& b) {
    if (a.empty) return b;
    if (b.empty) return a;
    Info c;
    c.empty = false;
    c.lc = a.lc;
    c.rc = b.rc;
    c.cnt = a.cnt + b.cnt + (a.rc == b.lc ? 1 : 0);
    return c;
}

class ZKWSegTree {
public:
    int n;      // 原始长度
    int base;   // 叶子起点(2的幂)
    vector<Info> tr;

    ZKWSegTree(const string& s) {
        n = (int)s.size();
        base = 1;
        while (base < n) base <<= 1;
        tr.assign(base << 1, make_empty());

        // build leaves
        for (int i = 0; i < n; i++) {
            int v = (s[i] == 'A') ? 1 : 0;
            tr[base + i] = {0, v, v, false};
        }
        // padding leaves remain empty; 但为了合并简单，也可以给它们一个合法值并标 empty
        for (int i = base - 1; i >= 1; i--) pull(i);
    }

    void pull(int p) {
        tr[p] = mergeInfo(tr[p << 1], tr[p << 1 | 1]);
    }

    void flip(int pos) { // 0-index
        int p = base + pos;
        // leaf is never empty for valid pos
        tr[p].lc ^= 1;
        tr[p].rc ^= 1;
        tr[p].cnt = 0;
        for (p >>= 1; p >= 1; p >>= 1) pull(p);
    }

    Info query(int l, int r) { // inclusive, 0-index
        l += base;
        r += base;
        Info leftAgg = make_empty();
        Info rightAgg = make_empty();

        while (l <= r) {
            if (l & 1) leftAgg = mergeInfo(leftAgg, tr[l++]);     // append to the right
            if (!(r & 1)) rightAgg = mergeInfo(tr[r--], rightAgg); // prepend to the left (keep order)
            l >>= 1;
            r >>= 1;
        }
        return mergeInfo(leftAgg, rightAgg);
    }
};

class Solution {
public:
    vector<int> minDeletions(string s, vector<vector<int>>& queries) {
        ZKWSegTree st(s);
        vector<int> ans;
        ans.reserve(queries.size());

        for (auto& q : queries) {
            if (q[0] == 1) {
                st.flip(q[1]);
            } else {
                ans.push_back(st.query(q[1], q[2]).cnt);
            }
        }
        return ans;
    }
};
```

当然，你会发现，上面的操作我们都先初始化了一个 `m` 出来——真的有用吗？答案是：我们可以不要这个 $m$。

也就是说，我们的 Segtree 类可以优化成这样：

```cpp
struct Segtree {
    int n;
    vector<int> s;

  public:
    Segtree(int n, int *a) {
        this->n = n;
        s.assign(n + 2 << 1, 0);
        for (int i = 1; i <= n; i++) { s[i + n - 1] = a[i]; }
        for (int i = n - 1; i >= 1; i--) {
            s[i] = max(s[i << 1], s[i << 1 | 1]);
        }
    }
    void modify(int pos, int val) {
        pos += n - 1;
        s[pos] = max(s[pos], val);
        for (pos >>= 1; pos >= 1; pos >>= 1) {
            s[pos] = max(s[pos << 1], s[pos << 1 | 1]);
        }
    }
    int query(int l, int r) {
        int ret = 0;
        for (l += n - 1, r += n - 1; l < r; l >>= 1, r >>= 1) {
            if (l & 1) ret = max(ret, s[l++]);
            if (r & 1) ret = max(ret, s[--r]);
        }
        return ret;
    }
};
```

为什么呢？以 $n=5$ 为例。我们建出来长这样：

![[Pasted image 20260219012438.png]]

简单思考会发现，如果我们依然执行这个 `query(l,r)` 的操作，比如我要查询 $[3,5]$，那么我们相当于先 `query(7,10)`。
1. 初始状态
	- $n = 5$。
	- 查询区间 $[3, 5]$，在 $[l, r)$ 形式下，$l=3, r=6$。
	- **代码第一步：**
	    - $l = 3 + 5 - 1 = \mathbf{7}$
	    - $r = 6 + 5 - 1 = \mathbf{10}$
2. 第一轮循环 ($l=7, r=10$)
	1. **判断 $l < r$**：$7 < 10$。进入循环。
	2. **左侧 `if(l&1)`**：$7$ 是奇数。
	    - `resl = combine(resl, tr[7])`（**拿到了下标 3 的值**）。
	    - $l$ 变成 $8$。
	3. **右侧 `if(r&1)`**：$10$ 是偶数。**不执行**。
	    - 注意！这里没有执行 `--r`，所以 $r$ 依然是 $10$。
	4. **迭代**：$l$ 变成 $4$ ($8 \gg 1$)，$r$ 变成 **$5$** ($10 \gg 1$)。
 3. 第二轮循环 ($l=4, r=5$)
	1. **判断 $l < r$**：$4 < 5$。进入循环。
	2. **左侧 `if(l&1)`**：$4$ 是偶数。**不执行**。
	    - $l$ 保持为 $4$。
	3. **右侧 `if(r&1)`**：$5$ 是奇数。**成立！**
	    - `--r`：$r$ 变成 **$4$**。
	    - `resr = combine(tr[4], resr)`（**重点！** 请看原图，节点 4 刚好是叶子 8 和 9 的父节点，也就是**下标 4 和 5 的合集**）。
	4. **迭代**：$l$ 变成 $2$ ($4 \gg 1$)，$r$ 变成 $2$ ($4 \gg 1$)。
4. 循环结束
	- **判断 $l < r$**：$2 < 2$。**失败，退出。**

当 $r$ 从 $10$ 降到 $5$ 时，看起来它要跨过中线去碰节点 1 了，但此时 $l$ 已经追到了 $4$。下一轮它们都会变成 $2$，循环直接终止。

所以其实无论怎么搞，我们上面的代码都不会出问题。一个更深刻的看法是，虽然树不是满的，但如果我们把数组下标映射到二进制，不难发现，这种写法本质上是在**二进制的位深**上做文章。

> 只要你申请了 $2n$ 的空间，哪怕 $n$ 不是 2 的幂，这棵树在逻辑上依然可以看作是由若干棵**完美的满二元树**拼接而成的森林。当你执行那个 `for` 循环时，你其实是在这些“小森林”之间横跳。
### 区间修改

现在来到了重头戏。我们怎么实现一个区间修改呢？我们没有传统线段树那样的递归结构，懒标记看似派不上用场了。——or is it?

#### 不使用标记永久化

我们依旧模仿区间查询去写一个代码：

```cpp
int modify(int l, int r, int val) { // 修改 [l,r)
    int ret = 0;
    for (l += m, r += m; l < r; l >>= 1, r >>= 1) {
        if (l & 1) change(l++, val);
        if (r & 1) change(--r, val);
        // 如果下一个是左子树，说明这一格是右子树，就需要贡献了

    return ret;
```

问题来了。我们现在只是修改了一个节点的值，怎么处理下推问题呢？画个图看看。

可以看到，类似线段树的 `push_down`，我们会想要从上面把懒标记“拉”下来。

以区间加，区间最小为例。如果我们要模仿的话，我们就这么模仿一个出来：
```cpp
void change(int idx, int val){
    s[idx] += val;
    if(idx < n) tag[idx] += val; // 如果是父节点，那 tag 打上
}

void push_down(int i){
    for(int h = __lg(i); h >= 1; h--){ // __lg(x) 等同于 floor(log2(x))
        int idx = i >> h; // 快速获得 h 级父亲
        // 从更远的父亲往更近的推
        if(!tag[idx]) continue; // 如果没有懒标记直接跳过
        change(idx<<1, tag[idx]); // 如果有实施更改
        change(idx<<1|1, tag[idx]);
        tag[idx] = 0; // 清空懒标记
    }
}
```
那我们递归型最后还有一个 `push_up` 环节，所以我们也可以模仿一个：
```cpp

    for(i>>=1 ; i >= 1; i >>= 1){
        s[i] = combine(s[i<<1],s[i<<1|1])+tag[i]; // 懒标记还没下推，所以要加上
```

问题来了。如果我们要用到区间长度怎么办？简单，利用区间长度都是 $2^k$ 即可。
```cpp
void change(int idx, int val, int h){
    s[idx] += val * (1<<h);
    if(idx < n) tag[idx] += val;
}

void push_up(int i){
    int h;
    for(h = 1, i>>=1 ; i >= 1; i >>= 1){ // 还没下传！
        tr[i] = combine(tr[i<<1],tr[i<<1|1]) + tag[i] * (1<<h);
    }

void modify(int l, int r, int val){
    int tl = l, tr = r, h = 0; 
    push_down(l+n-1), push_down(r-1+n-1); // 拉取（下传环节）
    for(l += n-1, r += n-1; l < r; l >>= 1, r >>= 1, h++){
        if(l&1) change(l++, val, h);
        if(r&1) change(--r, val, h);
```

一种实现 P3372 的方式如下：
```cpp
#include <bits/stdc++.h>

using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

#define int ll

struct SegTree {
    int n;
    vector<int> s, lazy;

  public:
    SegTree(int n, int *a) : lazy(n + 2 << 1, 0) {
        this->n = n;
        s.assign(n + 2 << 1, 0);
        for (int i = 1; i <= n; i++) s[i + n - 1] = a[i];
        for (int i = n - 1; i >= 1; i--)
            s[i] = s[i << 1] + s[i << 1 | 1];
    }

    void apply(int idx, int val, int h) {
        s[idx] += val * (1 << h);
        if (idx < n) lazy[idx] += val;
    }

    void push_down(int i) {
        for (int h = __lg(i); h >= 1; h--) {
            int idx = i >> h;
            if (!lazy[idx]) continue;
            apply(idx << 1, lazy[idx], h - 1);
            apply(idx << 1 | 1, lazy[idx], h - 1);
            lazy[idx] = 0;
        }
    }

    void push_up(int i) {
        int h = 1;
        for (i >>= 1; i >= 1; i >>= 1, h++) { // 从父节点开始更新
            s[i] = s[i << 1] + s[i << 1 | 1] + lazy[i] * (1 << h);
        }
    }

    void add(int l, int r, int val) {
        int h = 0;
        int tl = l, tr = r;
        push_down(l + n - 1), push_down(r - 1 + n - 1);
        for (l += n - 1, r += n - 1; l < r; l >>= 1, r >>= 1, h++) {
            if (l & 1) apply(l++, val, h);
            if (r & 1) apply(--r, val, h);
        }
        push_up(tl + n - 1), push_up(tr - 1 + n - 1);
    }

    int query(int l, int r) {
        int ret = 0;
        push_down(l + n - 1), push_down(r - 1 + n - 1);
        for (l += n - 1, r += n - 1; l < r; l >>= 1, r >>= 1) {
            if (l & 1) ret += s[l++];
            if (r & 1) ret += s[--r];
        }
        return ret;
    }
};

int n, m;

inline int read() {
    int x = 0, f = 1;
    char ch = getchar();
    while (ch < '0' || ch > '9') {
        if (ch == '-') f = -1;
        ch = getchar();
    }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + (ch - '0');
        ch = getchar();
    }
    return x * f;
}

signed main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    n = read(), m = read();
    vector<int> a(n + 1, 0);
    for (int i = 1; i <= n; i++) a[i] = read();
    SegTree st(n, a.data());
    for (int i = 1; i <= m; i++) {
        int op;
        op = read();
        if (op == 1) {
            int x = read(), y = read(), k = read();
            st.add(x, y + 1, k);
        } else {
            int x = read(), y = read();
            cout << st.query(x, y + 1) << '\n';
        }
    }
    return 0;
}
```

#### 标记永久化

当然，我们也可以选择不 push，而是选用标记永久化。而对于每个节点，只需要加上这个节点**实际的修改长度乘以永久化标记数组记录的修改的值**即可，即`ans += xl * mark[x] + yl * mark[y];`

**但是**，由于永久化标记数组并不会下传标记，所以我们在左右端点都是兄弟节点退出循环后，还需要继续向上推到根节点，来获得最终的总效果。

那么一个简单的实现就是下面这样的：

```cpp
int sum[mn], m, n, k, mark[mn];

void build(int n) {
    for (m = 1; m < n + 2; m <<= 1);
    for (int i = m + 1; i <= m + n; i ++)
        cin >> sum[i];
    for (int i = m - 1; i; i --)
        sum[i] = sum[i << 1] + sum[i << 1 | 1];
}

void update_node(int x, int v) {
    for (x += m; x; x >>= 1)
        sum[x] += v;
}

int query(int x, int y) {
    int ans = 0;
    for (x += m - 1, y += m + 1; x ^ y ^ 1; x >>= 1, y >>= 1) {
        if (~ x & 1) ans += sum[x ^ 1];
        if (y & 1) ans += sum[y ^ 1];
    }
    return ans;
}

void update_sum(int x, int y, int v) {
    int len = 1, xl = 0, yl = 0;
    for (x += m - 1, y += m + 1; x ^ y ^ 1; x >>= 1, y >>= 1, len <<= 1) {
        sum[x] += xl * v, sum[y] += yl * v;
        if (~ x & 1) sum[x ^ 1] += v * len, xl += len, mark[x ^ 1] += v;
        if (y & 1) sum[y ^ 1] += v * len, yl += len, mark[y ^ 1] += v;
    }
    while (x) {
        sum[x] += xl * v;
        sum[y] += yl * v;
        x >>= 1, y >>= 1;
    }
}

int query_sum(int x, int y) {
    int ans = 0;
    int len = 1, xl = 0, yl = 0;
    for (x = x + m - 1, y = y + m + 1; x ^ y ^ 1; x >>= 1, y >>= 1, len <<= 1) {
        ans += xl * mark[x] + yl * mark[y];
        if (~ x & 1) ans += sum[x ^ 1],xl += len;
        if (y & 1) ans += sum[y ^ 1], yl += len;
    }
    while (x) ans += xl * mark[x] + yl * mark[y], x >>= 1, y >>= 1;
    return ans;
}
```

- `sum[x]` 维护的是：**其子树中所有元素的值之和 + 该节点及其所有祖先节点的标记影响**。
- `mark[x]` 维护的是：**仅仅作用于当前节点 $x$ 覆盖范围的增量**。

在 `query_sum` 时，你那个 `while(x)` 循环就是在做：**“当我从叶子爬到根时，把一路上碰到的所有标记按覆盖长度加起来。”** 这就是为什么它不需要下推。

## 总结

虽然 zkw 线段树实现简单而且常数很小，但是它不能够很好维护比较复杂的操作，比如 [[9. 线段树维护多种信息、线段树合并、线段树的其他 trick]] 中的线段树合并与分裂。在线段树上二分的问题也是传统线段树更直观。因此绝大多数时候我们都会选择传统线段树。
