## 线段树合并与分裂

本节可以解决一系列 [[5. 启发式合并]] 的问题。甚至，线段树合并可以用来加速 dp 的转移。
### 线段树合并的原理

从下面这道题目开始。

>[!example] P3605
>现给定一个 $n$ 个节点的无向树，每个节点赋值 $p_i$。对每个 $u$，计算 $\sum_{v\in T_u} [p_v>p_u]$。
>
>数据范围：$n\leq 10^5, c_i\leq 10^9$。

一个比较朴素的想法是，我们首先离散化，把 $c_i$ 映射到 $[n]:=[1,n]$，然后我们使用 [[5. 启发式合并]]，对每个子树统计答案。这样子做每个节点合并 $\log n$ 次，偏序结构我们用树状数组维护，这样就是 $\log n$ 的修改查询的代价，总共的时间复杂度就是 $O(n\log^2 n)$，空间复杂度 $O(n)$，可以通过此题。一个实现代码看 [这里](https://www.luogu.com.cn/record/265747045)。

但是这很不牛，我们进行一个更进一步的思考，发现我们考虑 dfs 序以及对应的欧拉序，我们只需要考虑一个节点进栈出栈之间的值大于 $u$ 的个数，这样就是 $O(n\log n)$ 实现了。这个做法的代码参看题解。

那什么是线段树合并呢？我们考虑对每一个点都开一个权值线段树。这样传递到父节点的过程就等价于我们合并两个权值线段树。合并的过程是关键。
 - 对于线段树，我们按照先序遍历的样式去合并节点。先合并自己，再合并左儿子，最后合并右儿子。
 - 接下来的做法和 [[8. 主席树 · 可持久化线段树]] 很像。如果这个节点（在树上）自己和（在树上）儿子都有，那就直接相加加到自己的「老编号」的节点上。如果儿子有自己没有，说明这个下面的自己都没有，可以直接把儿子的抢过来，当前（线段树上的）节点直接指向（树上）儿子的（线段树上的）儿子。如果需要更加直观的讲解可以看下面这个图。

![[Pasted image 20260307144630.png]]

这里 $a$ 是父节点，$b$ 是子节点，我们 1～4 两个都有，所以复用 $a$ 的节点，然后看 1～2，类似。但是 2～2 只有 $b$ 有，所以我们直接抢过来。

这样的复杂度是多少呢？仿照 [[8. 主席树 · 可持久化线段树]] 中的讨论，我们每次相当于是在子树的节点上去加一个父节点的 $\log n$ 个节点，最多最多更改 $\log n$ 个节点，因此合并的复杂度是 $O(\log n)$ 的。最后查询的复杂度也是 $O(\log n)$ 的，因此总共的时间复杂度就是 $O(n\log n)$。掉了一个 $\log$。一个更好的看法是，我们考虑所谓的「往下找」的过程。我们每次向下找都会**废除掉至少 1 个节点**。因此既然空间复杂度是 $O(n\log n)$，我们所有的合并最劣也就是 $O(n\log n)$ 的，也就是摊还 $O(\log n)$ 的，十分优秀。后面的这个看法也就是所谓的「势能分析法」。这个方法在后面多个点合并的时候依然管用。

问题来了，我们要怎么开数组呢？如果我们每个节点都直接开的话空间复杂度是 $O(n^2)$ 肯定会爆掉的。那我们充分发动人类智慧发现每个节点都只用开必要的线段树就好了，这样空间复杂度就是 $O(n\log n)$，可以过。一个实现代码看 [这里](https://www.luogu.com.cn/record/265765638)。

>[!example] P3224
>现有 $n$ 个节点，编号 $[n]$，每个点有一个数字 $p_i$。初时有 $m$ 条边。接下来有 $q$ 条操作。每条操作居以下二者之一：
> - `B x y`：连接 $x,y$；
> - `Q x k`：打印与 $x$ 联通的点中的区间第 $k$ 小。
>
> 数据范围：$m,n\leq 10^5, q\leq 3\times 10^5$。

这个也是线段树合并板子题。难点在于查询，但是很显然可以用下面这个线段树上二分的思路来解决。连通性意味着我们只用在上面那道题的基础上加一个并查集就好了。一个实现代码看 [这里](https://www.luogu.com.cn/record/265786932)。这个时间复杂度是 $O((n+m+q)\log n)$。其中初始化 $O((n+m)\log n)$，查询 $O(q\log n$)。空间复杂度 $O(n\log n)$。

### 用线段树合并优化 dp

>[!example] P4577
>一个公司的组织领导架构可以用一棵领导树来表示。公司的每个成员对应于树中一个结点 $v_i$​，且每个成员都有相应的级别 $w_i$​。越高层的领导，其级别值 $w_i$​ 越小。树中任何两个结点之间有边相连，则表示与结点相应的两个成员属于同一部门。领导集团问题就是根据公司的领导树确定公司的最大部门。换句话说，也就是在领导树中寻找最大的部门结点子集，使得的结点 $v_i$​ 和 $v_j$​，如果 $v_i$​ 是 $v_j$​ 的子孙结点，则 $w_i​\geq w_j$​。

考虑 dp。因为这是类似 LIS 的问题我们肯定定义 $dp_{u,i}$ 表示以 $u$ 为节点，最小点权**恰好为 $i$** 的最大选取的集合。规定 $S_u$ 表示 $u$ 的孩子构成的集合。那么简单思考可以写出下面这个转移方程，如果不选 $u$，当我们进行到 $v$ 时有：
$$dp_{u,i}\gets dp_{u,i}+\max_{j\geq i}\{dp_{v,j}\}.$$
选了 $u$ 那么
$$dp_{u,w_u}=1+\sum_{v\in S_u} \max_{i\geq w_u}\{dp_{v,i}\}.$$
![[Pasted image 20260307194626.png]]

那显然如果直接动态规划那么时空复杂度都是 $O(n^2)$ 的，这显然不可接受。一个比较朴素的想法是利用线段树合并来维护第二维，这是因为我们拼两个线段树的时候，本质上就是在拼两个 dp 表。问题在于，如何合并。现在假设我们有两个权值线段树 $T_1,T_2$，其中 $T_1$ 是父节点 $u$ 的，$T_2$ 是子节点 $v$ 的。

我们引入两个辅助变量 $M_1, M_2$。规定：
 - $M_i$ 表示 $T_i$ 当前范围的右侧范围的最大值，$i=1,2$。换言之，如果 $T_1$ 当前区间是 $[l,r]$，整个区间是 $[n]$，那么 $M_1$ 表示 $[r+1,n]$ 这个区间的最大值；

那更新应该怎么做呢？我们分类讨论。
 - 如果都没有信息，直接倒回去，下面都不存在。
 - 如果来到某个节点 $T_1$ 有信息，$T_2$ 没信息，当前这个节点可以直接打上懒更新的 tag，因为我们这里相当于区间加了一个 $M_2$。（我们要的是大于 $i$ 的所有的和，因此直接合并这个就可以）
 - 如果来到某个节点 $T_1$ 没信息，$T_2$ 有信息，那就反过来，相当于把 $T_2$ 扔给 $T_1$，同时区间加一个 $M_1$。（同理）
 - 如果都有信息，那就合并下一层之后 `push up`。

一个需要注意的点是，最后来到叶子节点的时候，需要和 $T_2$ 中的额外对比一下。

这样子的合并时间复杂度是 $O(n\log n)$ 的，非常优秀。一个实现代码可以看 [这里](https://www.luogu.com.cn/record/266128846)。请注意，`merge` 里面的 `if(!p&&!q) return 0;` 这句是必须的，否则对 `tree[0]` 潜在污染会导致 wa。

以及一个细节是，一开始我们并不需要像之前一样全都 `build`。这是为什么呢？按道理，既然每个节点 $u$ 都是一个潜在的贡献者，那就应该先给每个节点 `build` 一棵只包含自己的小树，然后再合并。那为什么这道题目不行呢？这是因为，节点 $u$ 的选择是**依赖于**其子树**合并后**的状态的。

如果我们一开始就把 $u$ 放在里面，在合并第一棵子树时，$u$ 就会作为「已经选好的方案」参与运算，这打乱了我们 dp「从子节点推导父节点」的顺序。而且，从代码实现方面考量，通过 `dfs` 后半段的 `modify` 动态加入节点，不仅逻辑清晰，还节省了空间。我们只在必要的时候（处理完子树后）才插入 $u$，这使得线段树的节点增长更加可控。

一道进阶的的题目是这道题。

>[!example] P6847
>给定一棵以 $1$ 为根的 $n$ 个节点的树。
>
>有 $m$ 个果实，第 $j$ 个果实于第 $d_j$ 天在节点 $v_j$ 成熟，价值为 $w_j$，只能在成熟当天收获。你可以进行下面的操作任意次。
>
>操作：每天可多次切断树边。断边 $(u,v)$（$v$ 为 $u$ 子节点）可获得 $v$ 子树中当天成熟的所有果实的价值之和，然后该子树脱落，其上的未成熟果实永久消失。
>
>求可获得的最大总价值。

一个比较朴素的思路是 $dp_{u,i}$ 表示第 $i$ 天砍断 $u$ 连父亲的那个节点最多能获得多少果汁。那么转移方程就是
$$dp_{u,i}=\sum_{v\in S_u} \max_{j\leq i}\{dp_{v,j}\}+[i=d_u]\cdot w_u.$$
其中 $[P]=\begin{cases}1,& 如果\ P\ 真,\\ \\ 0, & \text{otherwise}.\end{cases}$ 朴素的转移是 $O(n^2)$ 的。但是通过线段树合并可以做到 $O(n\log n)$，但是实现的过程中有很多坑。

下面的过程中，我们认为 0 是一个「废节点」。一切跟 0 有关的数据都是不可信的。所以一旦处理到 0 我们就扔掉。特别的，这就让我们每一个 dfs 前必须要先 new 一个节点出来。

然后还有一个 trick 就是，我们不考虑使用普通线段树的区间赋值标记（比较难合并），而是记录下区间 $\min,\max$。

如果一个区间的 $\min=\max$，那么这个区间所有值相等，这种情况下，合并时直接被合并，更新时直接删掉左右儿子，修改时添加左右儿子，就相当于有了区间赋值的作用了。

```cpp
#include <algorithm>
#include <iostream>
#include <vector>

using std::max, std::cout, std::min;
using std::vector;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using i64 = long long;
using u64 = unsigned long long;

#define int i64

int read() {
    int x = 0, f = 1;
    char ch = getchar_unlocked();
    while (!isdigit(ch)) {
        if (ch == '-')
            f *= -1;
        ch = getchar_unlocked();
    }
    while (isdigit(ch)) {
        x = x * 10 + ch - '0';
        ch = getchar_unlocked();
    }
    return x * f;
}

const int MAXN = 1e5 + 5;

int head[MAXN], nxt[MAXN << 1], to[MAXN << 1];
int d[MAXN], w[MAXN];
int root[MAXN];

void add_edge(int u, int v) {
    static int cnt = 0;
    nxt[++cnt] = head[u];
    head[u] = cnt;
    to[cnt] = v;
}

struct Node {
    i64 mx, mn;
    int ls, rs;
    i64 add;
    static int cnt;
} tree[MAXN * 80];

int Node::cnt = 0;

int &lson(int p) {
    return tree[p].ls;
}
int &rson(int p) {
    return tree[p].rs;
}

vector<int> pool;

bool same(int p) {
    return tree[p].mx == tree[p].mn;
}

void rmv(int &p) {
    if (!p)
        return;
    pool.push_back(p);
    tree[p].ls = 0;
    tree[p].rs = 0;
    tree[p].mx = 0;
    tree[p].mn = 0;
    tree[p].add = 0;
    p = 0;
}

int assign() {
    int ret;
    if (!pool.empty()) {
        ret = pool.back();
        pool.pop_back();
        return ret;
    }
    ret = ++Node::cnt;
    return ret;
}

void add(int p, int val) {
    if (!p)
        return;
    tree[p].add += val;
    tree[p].mx += val;
    tree[p].mn += val;
}

void push_up(int p) {
    tree[p].mx = max(tree[lson(p)].mx, tree[rson(p)].mx);
    tree[p].mn = min(tree[lson(p)].mn, tree[rson(p)].mn);
    if (same(p)) {
        rmv(lson(p));
        rmv(rson(p));
    }
}

void push_down(int p) {
    if (!tree[p].add)
        return;
    add(lson(p), tree[p].add);
    add(rson(p), tree[p].add);
    tree[p].add = 0;
}

int merge(int p, int q) {
    if (!p && !q)
        return 0;
    if (!p) {
        return q;
    }
    if (!q) {
        return p;
    }
    if (same(p)) {
        add(q, tree[p].mx);
        return q;
    }
    if (same(q)) {
        add(p, tree[q].mx);
        return p;
    }
    push_down(p);
    push_down(q);
    lson(p) = merge(lson(p), lson(q));
    rson(p) = merge(rson(p), rson(q));
    push_up(p);
    return p;
}

i64 query(int pos, int l, int r, int p) {
    if (same(p))
        return tree[p].mx;
    if (l == r)
        return tree[p].mx;
    int mid = (l + r) >> 1;
    push_down(p);
    if (pos <= mid)
        return query(pos, l, mid, lson(p));
    else
        return query(pos, mid + 1, r, rson(p));
}

void modify(int ml, int mr, i64 val, int l, int r, int &p) {
    if (tree[p].mn > val)
        return;
    if (ml <= l && r <= mr && tree[p].mx <= val) {
        tree[p].mx = val;
        tree[p].mn = val;
        tree[p].add = 0;
        rmv(lson(p));
        rmv(rson(p));
        return;
    }
    push_down(p);
    if (same(p)) {
        lson(p) = assign();
        rson(p) = assign();
        tree[lson(p)].mx = tree[lson(p)].mn = tree[rson(p)].mx = tree[rson(p)].mn =
            tree[p].mx;
    }
    int mid = (l + r) >> 1;
    if (ml <= mid)
        modify(ml, mr, val, l, mid, lson(p));
    if (mid < mr)
        modify(ml, mr, val, mid + 1, r, rson(p));
    push_up(p);
}

int n, m, k;

void dfs(int u, int pa) {
    root[u] = assign();
    for (int e = head[u]; e; e = nxt[e]) {
        int v = to[e];
        if (v == pa)
            continue;
        dfs(v, u);
        root[u] = merge(root[u], root[v]);
    }
    if (w[u]) {
        i64 mx = query(d[u], 1, k, root[u]);
        modify(d[u], k, mx + w[u], 1, k, root[u]);
    }
}

signed main() {
    n = read(), m = read(), k = read();
    for (int i = 1; i < n; i++) {
        int p = read();
        add_edge(i + 1, p);
        add_edge(p, i + 1);
    }
    for (int i = 1; i <= m; i++) {
        int v = read();
        d[v] = read();
        w[v] = read();
    }
    dfs(1, 0);
    cout << tree[root[1]].mx << '\n';
    return 0;
}
```

一个类似的题目是下面这题。

>[!example] P7563
>给定 $n$ 个选手，编号 $1$ 到 $n$。 对于每个选手 $i$，已知选手 $i$ 的 Rating 必须 $\ge$ 选手 $a_i$ 的 Rating，并且 $h_i$ 表示当前列表中选手 $i$ 的 Rating（不一定满足前面的约束）。你可以进行任意多次修改选手 $i$ 的 Rating 为任意整数的操作，代价为 $c_i$。
>
>求最小的使得 Rating 满足 $(i,a_i)$ 确定的约束的修改代价。

完全类似的想法，按照偏序关系从 $i$ 往 $a_i$ 连边建立一个基环树。一个 naive 的观察是环上面的元素相等。我们来看剩下的树那一部分要怎么弄，我们规定 $dp_{u,i}$ 表示把 $u$ 变成 $i$ 并且 $T_u$ 都满足约束所需要的代价。于是乎
$$
dp_{u,i}=\sum_{v\in S_u} \min_{j\geq i}\{dp_{v,j}\}+[i\neq h_u]\cdot c_u.
$$
这是和上面那道题完全相似的式子。我们最后枚举一下基环树上面的取值就可以了。

## 线段树上二分

线段树的二叉树结构天然决定了线段树的二分是十分好实现的。有些时候我们还可以解决一些很难的问题。


>[!example] LC 3721

这题用到了 [HH 的项链](https://www.luogu.com.cn/problem/P1972) 的一个 trick。我们要如何快速（离线）查询一个区间的不同个数。当然，如果在线的话我们可以通过莫队实现。

我们需要快速查询一个位置的最长的后缀和为 0 的长度，这时候我们首先需要**维护后缀和**——这个可以通过区间加来实现。并且我们注意到后缀和最多相邻相差 1，因此离散意义下连续，进而可以通过 **二分查找区间最大最小值的方法** 来快速找到第一个为 0 的地方。

## 扫描线

扫描线的本质其实是一种“流程”，我们可以用不同的数据结构来维护扫描线扫描的信息。

利用扫描线思想，我们可以利用线段树解决一些意想不到的问题。一个最典型的应用就是处理图形面积并集、周长并集。

>[!example] ABC499E
>有一个网格，网格中有 $H$ 行和 $W$ 列。从上往下数第 $r$ 行，从左往上数第 $c$ 列的单元格叫做单元格 $(r,c)$ 。每个单元格都被涂成黑色或白色： $(R_k,C_k)$ 单元格为 $k=1,2,\ldots,N$ 的黑色，其余的 $HW-N$ 单元格为白色。
>
>求在这个网格中，有多少个行为 $h$ 列为 $w$ 的矩形区域，使得其中包含的所有单元格都被涂成白色？
>
>数据范围：$H,W,h,w\leq 10^9, N\leq 2\times 10^5$。

求 $N$ 个矩形的并集面积，直接求非常复杂，因为矩形之间会重叠。
- **垂直方向（y轴）**：我们用一把尺子（扫描线）记录当前有多少个矩形覆盖了这把尺子。
- **水平方向（x轴）**：尺子只在矩形的**左边界**和**右边界**处改变覆盖状态。这些边界位置就是“事件点”。
我们将面积切成一个个纵向的条状矩形：
$$\text{总面积} = \sum (\text{相邻两个事件点的距离} \Delta x \times \text{当前扫描线上被覆盖的总长度 } L)$$
本质上，我们把每个矩形拆成了两个事件：
- **入边（左边界）**：扫描线遇到它，覆盖次数 $+1$。
- **出边（右边界）**：扫描线遇到它，覆盖次数 $-1$。

我们需要把所有事件按 $x$ 坐标排序，依次处理。然后，线段树的每个节点代表纵轴上的一个区间。
- `cnt[node]`：记录该区间被完整覆盖了多少次。
- `len[node]`：记录该区间内**至少被覆盖一次**的总长度。

更新的逻辑是，
- 如果 `cnt[node] > 0`，说明该区间被彻底覆盖，`len[node]` = 区间总长度。
- 如果 `cnt[node] == 0`，则 `len[node]` 由左右子节点的 `len` 之和决定（如果是叶子节点则为 0）。

回到本题。首先显然考虑正难则反。那么我们考虑钦定每个矩形覆盖自己和右下角的点，那么不能选用的格子就只有黑色格子周围的那么些点。假设有一个黑格子在 $(R, C)$，它禁用的左上角 $r$ 坐标范围是 $[r_1, r_2]$， $c$ 坐标范围是 $[c_1, c_2]$。
1. **准备事件**：将每个禁区转化成两个事件：
    - 在 $r_1$ 时刻，加入区间 $[c_1, c_2]$（`type = +1`）。
    - 在 $r_2 + 1$ 时刻，移除区间 $[c_1, c_2]$（`type = -1`）。
2. **排序**：按 $r$ 坐标从小到大排序。
3. **扫描**：
    - 走到第一个 $r$ 坐标。
    - 计算当前 $r$ 与上一个 $r$ 的差值 $\Delta r$。
    - `面积 += \Delta r \times 此时线段树中被覆盖的总长度`。
    - 在线段树中更新当前事件的区间 $[c_1, c_2]$。
4. **循环**直到处理完所有事件。

这样就解决了这道题。
