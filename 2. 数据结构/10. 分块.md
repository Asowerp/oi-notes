## 块状数组

分块的基本思想是：**对所有元素分成许多个块，每次操作时分开处理，对在同一个块内或者是块的一部分的元素直接暴力修改，对每个整的块再进行处理。**

设块长为 $k$，那么暴力修改 / 查询至多修改 $k$ 个元素，块的个数至多是 $\dfrac{n}{k}$，因此单次修改的时间复杂度大致是 $O\left(k+\dfrac{n}{k}\right)$，取 $k\sim\sqrt n$ 即可做到 $O(\sqrt n)$ 的单次修改 / 查询的复杂度。还是比较优秀的。

因为每次都带一个根号，分块算法有时也被称作 **根号分治** 算法；也因为分块算法太暴力，我们也把它叫做“优雅的暴力”。

我们从下面这道题目开始理解分块。

>[!example] Give Away
>给定长度为 $n$ 的序列 $a$，你需要实现以下操作：
> - `0 l r v`：输出 $[l,r]$ 中大于等于 $v$ 的数字个数。
> - `1 p v`：把 $a_p$ 修改为 $v$。
> 
> 一共会有 $q$ 次操作。
> 数据范围：$n\leq 5\times 10^5, q\leq 10^5, a_i\leq 10^9$。

简单思考发现线段树似乎不是很能够维护这两个操作。（事实上可以通过树状数组套平衡树实现 $O(m\log^2n)$，但是太复杂了我也不想看）

所以我们考虑利用分块来维护这一整个过程。

最难维护的其实是第一个过程——怎么做到快速查询大于等于 $v$ 的数字个数？

我们思考能否让每个块块内有序，这样每次排序至多是一个 $O(\sqrt n\log \sqrt n)\sim O(\sqrt{ n }\log n)$ 的代价，这是可以接受的。

打个比方，我们现在 $n=100$，$l=13,r=97$。我们把不足一个分块的块叫做散块。根据前面的分析我们块长应该取 $k=10$。
 - 散块 13 - 20：直接数一遍有多少个数大于等于 $v$；
 - 散块 90 - 97：直接数一遍有多少个数大于等于 $v$；
 - 中间：对每一个分块拷贝后 sort 以下二分查找有多少个大于等于 $v$；
因此这样查询的复杂度大致是 $O(\underbracket{3\sqrt n}_{\text{散块、拷贝}}+\underbracket{\sqrt n\log\sqrt n}_{\text{sort}}+\underbracket{\log \sqrt n}_{\text{二分}})\sim O(\sqrt{ n }\log \sqrt{n})$。

对第二个操作呢？当然是直接修改就好啦！复杂度当然是 $O(1)$。

因此总的复杂度是 $O(q\sqrt n\log \sqrt{n})$ 的。在本题数据下大概是 $5\times 10^8$，勉强能过。[提交记录](https://www.spoj.com/status/ns=35535441#)

代码如下。请注意**块的编号从 0 开始**。而且在我们处理 1-indexed 的时候，通常我们分块逻辑是 $(i\cdot k,(i+1)\cdot k]$，这样第 0 块就是 0，1，...，$k$。

```cpp
#include <bits/stdc++.h>
using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

const int MAXN = 5e5 + 5;
const int MAXB = 1005;

int n, q, sz;
int a[MAXN];
vi block[MAXB];

inline int get_id(int x) { return (x - 1) / sz; }

int query(int ql, int qr, int v) {
    int l = get_id(ql), r = get_id(qr);
    int ret = 0;
    if (l == r) {
        for (int i = ql; i <= qr; i++) {
            if (a[i] >= v) ret++;
        }
        return ret;
    }
    for (int i = ql; i <= min(qr, (l + 1) * sz); i++) {
        if (a[i] >= v) ret++;
    }
    for (int i = qr; i > r * sz; i--) {
        if (a[i] >= v) ret++;
    }
    for (int i = l + 1; i <= r - 1; i++) {
        ret += (block[i].size() -
                (lower_bound(all(block[i]), v) - block[i].begin()));
    }
    return ret;
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    cin >> n;
    sz = floor(sqrt(n)) + 1;
    for (int i = 1; i <= n; i++) {
        cin >> a[i], block[get_id(i)].push_back(a[i]);
    }
    for (int i = 0; i <= get_id(n); i++) { sort(all(block[i])); }
    cin >> q;
    for (int i = 1; i <= q; i++) {
        int op;
        cin >> op;
        if (op == 0) {
            int ql, qr, v;
            cin >> ql >> qr >> v;
            cout << query(ql, qr, v) << endl;
        } else {
            int p, v;
            cin >> p >> v;
            int b = get_id(p);
            auto it = lower_bound(all(block[b]), a[p]);
            *it = v;
            sort(all(block[b]));
            a[p] = v;
        }
    }
    return 0;
}
```

本题的一个强化就是下面这道题。把单点修改更改为区间修改。

>[!example] P2801
>[题目链接](https://www.luogu.com.cn/problem/P2801)
>
>教主最近学会了一种神奇的魔法，能够使人长高。于是他准备演示给 XMYZ 信息组每个英雄看。于是 $N$ 个英雄们又一次聚集在了一起，这次他们排成了一列，被编号为 $1, 2, \ldots, N$。
>
>每个人的身高一开始都是不超过 $1000$ 的正整数。教主的魔法每次可以把闭区间 $[L, R]$（$1≤L≤R≤N$）内的英雄的身高全部加上一个整数 $W$。（虽然 $L=R$ 时并不符合区间的书写规范，但我们可以认为是单独增加第 $L(R)$ 个英雄的身高）
>
>CYZ、光哥和 ZJQ 等人不信教主的邪，于是他们有时候会问 WD 闭区间 $[L, R]$ 内有多少英雄身高大于等于 $C$，以验证教主的魔法是否真的有效。
>
>WD 巨懒，于是他把这个回答的任务交给了你。

很显然，我们仿照线段树区间加的想法，只需要额外维护一个 `lazy` 的标记就好了。在这里我们可以手动管理一下内存，这样 block 每次直接能够做到 $O(n)$ 的复制，而不用像之前一样写个二分（而且在这里因为是区间查找，复制的话会更省时间复杂度）。

```cpp
#include <bits/stdc++.h>
using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

const int MAXN = 1e6 + 5;
const int MAXB = 1e3 + 5;

int n, q, sz;
int a[MAXN];
int block[MAXN];
int lazy[MAXB];

inline int get_id(int x) { return (x - 1) / sz; }

void build(int id) {
    memcpy(block + id * sz + 1, a + id * sz + 1, sz * sizeof(int));
    sort(block + id * sz + 1, block + (id + 1) * sz + 1);
    // 从 id*sz+1 开始，长度为 sz 的块的 sort 不要把+1 漏了！！
    // 就像你 sort 一个 n 的数组是 sort(a+1,a+n+1); 一样
}

void add(int al, int ar, int val) {
    int l = get_id(al), r = get_id(ar);
    if (l == r) {
        for (int i = al; i <= ar; i++) { a[i] += val; }
        build(l);
        return;
    }
    for (int i = al; i <= (l + 1) * sz; i++) { a[i] += val; }
    build(l);
    for (int i = ar; i > r * sz; i--) { a[i] += val; }
    build(r);
    for (int i = l + 1; i < r; i++) { lazy[i] += val; }
}

int query(int ql, int qr, int val) {
    int l = get_id(ql), r = get_id(qr), ret = 0;
    if (l == r) {
        for (int i = ql; i <= qr; i++) {
            if (a[i] + lazy[l] >= val) ret++;
        }
        return ret;
    }
    for (int i = ql; i <= (l + 1) * sz; i++) {
        if (a[i] + lazy[l] >= val) ret++;
    }
    for (int i = qr; i > r * sz; i--) {
        if (a[i] + lazy[r] >= val) ret++;
    }
    for (int i = l + 1; i < r; i++) {
        int cur = val - lazy[i];
        int pos = lower_bound(block + i * sz + 1,
                              block + (i + 1) * sz, cur) -
                  block - i * sz;
        ret += sz - pos + 1;
    }
    return ret;
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    cin >> n >> q;
    sz = ceil(sqrt(n));
    for (int i = 1; i <= n; i++) { cin >> a[i]; }
    for (int i = 0; i <= get_id(n); i++) { build(i); } // 从 0 开始！
    for (int i = 1; i <= q; i++) {
        char op;
        cin >> op;
        if (op == 'A') {
            int l, r, c;
            cin >> l >> r >> c;
            cout << query(l, r, c) << '\n';
        } else {
            int l, r, w;
            cin >> l >> r >> w;
            add(l, r, w);
        }
    }
    return 0;
}
```

当然，以上都是手动管理了块的开始与结束。我们也可以像这样先用一个 `st, ed` 数组来维护起点终点，以及一个 `pos` 数组取代 `get_id(x)` 这个函数。

```cpp
void init(){
	sz = sqrt(n);
	m = n / sz;
	if (n % sz) m++;
	for (int i = 1; i <= m; i++) {
		st[i] = ed[i - 1] + 1;
		ed[i] = i * sz;
	}
	ed[m] = n;
	for (int i = 1; i <= n; i++) {
		pos[i] = (i - 1) / sz + 1;
	}
	for (int i = 1; i <= m; i++) {
		build(i);
	}
}
```

当然这都是常数优化了。我们现在可以总结一下分块的适用场景：
1. 区间不可合并的信息，线段树等结构难维护，分块往往容易，因为块内信息是暴力维护的
2. 数据量允许的情况下，分块优于暴力，又比高级结构的实现更容易分析
3. 分块维护信息很容易，会衍生出很多强大变体
4. 正式比赛中很少出现以分块作为最优解的题目，分块优势是 **好想、好写、常数小**
5. 题目思维难度较大时，如果分块的做法已经很难，那么更优结构的做法只会更难
6. 分块的地位，非必需但收益高

接下来我们看一下怎么维护区间众数。

>[!example] P4168
>给定 $n$ 个数的数组 $a$，$m$ 次询问，每次询问一个区间 $[l,r]$，你需要回答 $a$ 在区间 $[l,r]$ 里的众数。若有多个，则输出最小的那个。
>
>**强制在线。**
>
>数据范围：$n\leq 4\times 10^4, m\leq 5\times 10^4, a_i\leq 10^9$。

这一题，首先离散化把 $a_i$ 映射到 $[1,n]$。

秉承着直接暴力的思想，我们定义 $f$ 与 $m$ 两个数组。$f[i][j]$ 表示前 $i$ 块中 $j$ 出现的次数（freq），$m[l][r]$ 表示 $l$ 到 $r$ 的众数（mode）。这样子的空间复杂度是 $O\left(n\cdot\dfrac{n}{L}+\dfrac{n^2}{L^2}\right)=O\left(\dfrac{n^2}{L}\right)$，能开下。

很显然，我们期待整块直接获得众数 $m[l][r]$，然后我们对散块里面用 $f[i][j]$ 快速（$f[r][j]-f[l][j]$）获得整块个数，并且统计散块里面的个数能不能超过 $m[l][r]$。这样对每个数都判断一下是 $O(L)$ 的，因此查询时间复杂度是 $O(mL)$ 的。

现在我们来看预处理的时间复杂度。我们遍历一遍整个数组，每个数组贡献一次，这是 $O(n)$ 的 $f[i][j]$ 计算，然后众数是每一块都需要对 $n$ 个数取统计的，因此暴力是 $O\left(\left(\dfrac{n}{L}\right)^2\cdot n\right)=O\left(\dfrac{n^3}{L^2}\right)$ 的。

因此，总的复杂度是 $O\left(mL+\dfrac{n^3}{L^2}\right)\gtrsim  O\left(mL+mL+\dfrac{n^3}{L^2}\right)$，不难看出下界是 $O(\sqrt[3]{m^2n^3})\sim O(nm^{2/3})$。取等的时候 $mL=\dfrac{n^3}{L^2}\implies L=\dfrac{n}{m^{1/3}}$。加上一个离散化的过程，时间复杂度就是 $O(n\log n+nm^{2/3})$，空间复杂度就是 $O(nm^{1/3})$。

但是简单思考发现，如果我们从 $m[l][r-1]$ 取更新 $m[l][r]$，那我们思考更新的众数来源只有可能是：
 - $m[l][r-1]$ 本身；
 - 块 $r$ 中出现的数。

因此我们可以枚举 $r$ 中所有数的 $f[r][j]-f[l-1][j]$，这样时间复杂度是 $O(L)$ 的，加上一个 $O\left(\dfrac{n^2}{L^2}\right)$ 的时间复杂度，这样就得到了一个 $O\left(\dfrac{n^2}{L}\right)$ 的 $m[i][j]$ 更新。这样就优化为了 $O\left(mL+\dfrac{n^2}{L}\right)\gtrsim O\left(n\sqrt m\right)$，取下界当 $L=\dfrac{n}{m^{1/2}}$。总的时间复杂度就是 $O(n\log n+n\sqrt m)$，空间复杂度就是 $O(nm^{1/2})$。

```cpp
#include <bits/stdc++.h>

using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

inline int read() {
    int x = 0, f = 1;
    char ch = getchar();
    while (ch < '0' || ch > '9') {
        if (ch == '-') f = -1;
        ch = getchar();
    }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + (ch - '0');
        ch = getchar();
    }
    return x * f;
}

const int MAXN = 4e4 + 5;
const int MAXB = 255;

int n, m;
int sz, up;
int freq[MAXB][MAXN], mode[MAXB][MAXB], mp[MAXN];

inline int get_id(int x) { return (x + sz - 1) / sz; }

inline int l(int idx) { return (idx - 1) * sz + 1; }
inline int r(int idx) { return min(idx * sz, n); }

void build(const vi &a) {
    for (int i = 1; i <= n; i++) { freq[get_id(i)][a[i]]++; }
    for (int i = 1; i <= get_id(n); i++) {
        for (int j = 1; j <= up; j++) {
            freq[i][j] += freq[i - 1][j];
        }
    }
    for (int i = 1; i <= get_id(n); i++) {
        for (int j = i; j <= get_id(n); j++) {
            mode[i][j] = mode[i][j - 1];
            for (int k = l(j); k <= r(j); k++) {
                int cur = freq[j][a[k]] - freq[i - 1][a[k]],
                    lst =
                        freq[j][mode[i][j]] - freq[i - 1][mode[i][j]];
                mode[i][j] = cur > lst    ? a[k]
                             : cur == lst ? min(a[k], mode[i][j])
                                          : mode[i][j];
            }
        }
    }
}

int query(int ql, int qr, const vi &a) {
    int bl = get_id(ql), br = get_id(qr);
    static int cnt[MAXN]{};
    int ret = 0;
    if (bl == br) {
        for (int i = ql; i <= qr; i++) {
            cnt[a[i]]++;
            if (cnt[ret] < cnt[a[i]] ||
                (cnt[ret] == cnt[a[i]] && ret > a[i]))
                ret = a[i];
        }
        for (int i = ql; i <= qr; i++) cnt[a[i]] = 0;
        return ret;
    }
    bl++, br--;
    for (int i = ql; i <= r(bl - 1); i++) {
        if (!cnt[a[i]])
            cnt[a[i]] += freq[br][a[i]] - freq[bl - 1][a[i]];
        cnt[a[i]]++;
        if (cnt[ret] < cnt[a[i]] ||
            (cnt[ret] == cnt[a[i]] && ret > a[i]))
            ret = a[i];
    }
    for (int i = qr; i >= l(br + 1); i--) {
        if (!cnt[a[i]])
            cnt[a[i]] += freq[br][a[i]] - freq[bl - 1][a[i]];
        cnt[a[i]]++;
        if (cnt[ret] < cnt[a[i]] ||
            (cnt[ret] == cnt[a[i]] && ret > a[i]))
            ret = a[i];
    }
    if (!cnt[mode[bl][br]]) {
        int _mode = mode[bl][br];
        cnt[_mode] = freq[br][_mode] - freq[bl - 1][_mode];
        if (cnt[ret] < cnt[_mode] ||
            (cnt[ret] == cnt[_mode] && ret > _mode))
            ret = _mode;
    }
    for (int i = ql; i <= r(bl - 1); i++) cnt[a[i]] = 0;
    for (int i = l(br + 1); i <= qr; i++) cnt[a[i]] = 0;
    cnt[mode[bl][br]] = 0;
    return ret;
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    n = read(), m = read();
    sz = n / ceil(sqrt(m));
    vi a(n + 1, 0);
    for (int i = 1; i <= n; i++) a[i] = read();
    auto b = a;
    sort(b.begin() + 1, b.end());
    b.erase(unique(b.begin() + 1, b.end()), b.end());
    up = b.size();
    for (auto &x : a) {
        int v = lower_bound(b.begin() + 1, b.end(), x) - b.begin();
        mp[v] = x;
        x = v;
    }
    build(a);
    int lst = 0;
    for (int i = 1; i <= m; i++) {
        int l, r;
        l = read(), r = read();
        tie(l, r) =
            minmax(((l + lst - 1) % n) + 1, ((r + lst - 1) % n) + 1);
        if (l > r) swap(l, r);
        cout << (lst = mp[query(l, r, a)]) << '\n';
    }
    return 0;
}
```

但是，如果空间卡的死一点呢？

>[!example] P5048
>给定 $n$ 个数的数组 $a$，$m$ 次询问，每次询问一个区间 $[l,r]$，你需要回答 $a$ 在区间 $[l,r]$ 里的众数的出现次数。
>
>**强制在线。**
>
>数据范围：$n,m,a_i\leq 5\times 10^5$，空间限制 64MB。

这题神了。

如果沿用上题的思路，那么我们的空间复杂度是 $O(n\sqrt m)$，大致是 1.5GB，非常不牛。所以我们要思考怎么优化。

我们不能够计算 $freq$ 了，一个很好的转换思路是考虑去维护每一个数出现的位置。然后我们暴力讨论一下散块中的元素能不能成为众数。这样空间复杂度是 $O\left(\left(\dfrac{n}{L}\right)^2+n\right)$ 的，完全能过。现在我们来思考散块中的怎么维护。

显然一个非常暴力的思路是去二分查找出现的位置。这样子散块中每个元素查询需要 $O(\log n)$ 的时间复杂度，至多要查询 $L$ 个，因此查询的时间复杂度是 $O(mL\log n)$ 的。预处理如果能够按照前面的做法的话，那就是 $O\left(\dfrac{n^2}{L}\right)$ 的（但是这里注意对每个大循环内部我们都需要维护一个 cnt 数组），因此对每一块我们按次序处理，因此总时间复杂度是 $O\left(mL\log n+\dfrac{n^2}{L}\right)\gtrsim O(n\sqrt{m\log n})$，此时 $L=\dfrac{n}{\sqrt{m\log n}}$。空间复杂度就是特别美好的 $O(m\log n+n)$，大致是 20MB 上下。虽然理论上足够通过此题了，但是 [交上去](https://www.luogu.com.cn/record/263360587) 会发现过不了。这很不牛。一个非常好的追问是，我们能不能做到 $O(n\sqrt m)$ 的时间复杂度？答案是肯定的。

于是现在我们来进行进一步的思考。我们发现我们前面的瓶颈是二分查找，我们期待维护一个新结构来实现 $O(1)$ 的查找，这样我们就可以扔掉一个 $O(\sqrt{\log n})$ 了。——这就是所谓的“秩查找法”。它的核心思想是：我们用 $O(n)$ 的预处理获得每个元素在所有相同数值中的“排名”（即它是该值第几次出现），通过 $O(1)$ 的地址偏移判断某个值在区间内的频率是否能达到 $ans+1$。

为了实现 $O(1)$ 查询，我们需要建立元素与其在同类数值中位置的映射关系：
- **`pos[]` (位置表)**：将所有相同数值出现的下标收集在一起。例如，如果数值 `5` 在原数组下标 `2, 10, 15` 出现，那么在 `pos_data` 某段连续区域内就按顺序存着 `[2, 10, 15]`。
- **`head[v]`**：记录数值 `v` 在 `pos` 中的起始偏移量。
- **`rank[i]` (秩)**：记录原数组下标 `i` 处的元素，在它所属的那组 `pos` 中是第几个。
    - 例如：`a[10] = 5`，它是第 2 次出现，则 `rank[10]` 指向 `pos` 中存储 `10` 的那个位置。
那么接下来，假设我们当前已知一个**可行**的众数频率 `freq`（最初来自预处理的整块众数频率）。那么接下来，为了判断散块元素是否能够得到一个更高的 `freq`，我们进行如下查询：
- 找到 $x$ 在原数组中的下标 $i$，通过 `rank[i]` 知道它是该数值的第 $k$ 个。
- 如果我们想要频率达到 `ans + 1`，那么该数值的第 $k + ans$ 个（或者第 $k - ans$ 个，其实只用判断一边就够了）出现的下标，必须还在当前的查询区间 $[ql, qr]$ 之内。如果成立，说明 $x$ 至少出现了 `ans + 1` 次。我们将 `ans` 自增，并**原地继续循环**，尝试能不能推到 `ans + 2`。

这样，查询的复杂度就变成了美丽的 $O(mL)$。加上预处理的 $O\left(\dfrac{n^2}{L}\right)$，总复杂度就是非常牛的 $O\left(mL+\dfrac{n^2}{L}\right)\gtrsim O(n\sqrt{m})$，此时块长为 $L=\dfrac{n}{m^{1/2}}$。空间复杂度就是 $O(n+m)$，完全足够通过此题。特别地，本题中 `mode` 维护的是出现次数了。

```cpp
#include <bits/stdc++.h>

using namespace std;

inline int read() {
    int x = 0;
    char ch = getchar();
    while (ch < '0' || ch > '9') { ch = getchar(); }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + (ch - '0');
        ch = getchar();
    }
    return x;
}

const int MAXN = 500005;
const int MAXB = 750;

int n, m, sz, num;
int a[MAXN], rank_idx[MAXN];
int pos[MAXN], head[MAXN], v_size[MAXN];
int mode[MAXB][MAXB], cnt[MAXN];

// 预处理位置信息与秩
void build_rank() {
    for (int i = 1; i <= n; i++) v_size[a[i]]++;
    for (int i = 1; i < MAXN; i++)
        head[i] = head[i - 1] + v_size[i - 1];

    // 借用 v_size 统计当前填充进度
    static int cur_ptr[MAXN];
    for (int i = 1; i <= n; i++) {
        int v = a[i];
        int p = head[v] + cur_ptr[v]++;
        pos[p] = i;
        rank_idx[i] = p;
    }
}

inline int get_l(int id) { return (id - 1) * sz + 1; }
inline int get_r(int id) { return min(id * sz, n); }
inline int get_id(int x) { return (x - 1) / sz + 1; }

void build() {
    build_rank();
    num = get_id(n);
    for (int i = 1; i <= num; i++) {
        int cur_max = 0;
        for (int j = i; j <= num; j++) {
            int br = get_r(j);
            for (int k = get_l(j); k <= br; k++) {
                cur_max = max(cur_max, ++cnt[a[k]]);
            }
            mode[i][j] = cur_max;
        }
        for (int k = get_l(i); k <= n; k++) cnt[a[k]] = 0;
    }
}

int query(int ql, int qr) {
    if (ql > qr) return 0;
    int bl = get_id(ql), br = get_id(qr);
    int ans = 0;

    if (bl == br) {
        for (int i = ql; i <= qr; i++) {
            // 在同一块内，简单的 ++cnt 逻辑
            ans = max(ans, ++cnt[a[i]]);
        }
        for (int i = ql; i <= qr; i++) cnt[a[i]] = 0;
        return ans;
    }

    // 1. 初始答案为中间整块的最大频率
    if (bl + 1 <= br - 1) { ans = mode[bl + 1][br - 1]; }

    // 2. 扫描左散块：尝试利用 Rank 增加 ans
    int Lr = get_r(bl);
    for (int i = ql; i <= Lr; i++) {
        int v = a[i];
        // 关键：检查 v 在 [ql, qr] 内部是否能达到 ans + 1 的频率
        // 偏移 ans 位后，如果下标仍在 qr 之内，说明频率足够
        while (rank_idx[i] + ans < head[v + 1] &&
               pos[rank_idx[i] + ans] <= qr) {
            ans++;
        }
    }

    // 3. 扫描右散块：同理逻辑
    int Rl = get_l(br);
    for (int i = Rl; i <= qr; i++) {
        int v = a[i];
        // 检查 v 向左偏移 ans 位是否仍在 ql 之内
        while (rank_idx[i] - ans >= head[v] &&
               pos[rank_idx[i] - ans] >= ql) {
            ans++;
        }
    }

    return ans;
}

int main() {
    n = read();
    m = read();
    sz = n / sqrt(m + 1);
    if (sz < 1) sz = 1;

    for (int i = 1; i <= n; i++) a[i] = read();

    build();

    int lst = 0;
    while (m--) {
        int l = read() ^ lst, r = read() ^ lst;
        if (l > r) swap(l, r);
        printf("%d\n", lst = query(l, r));
    }
    return 0;
}
```

## 块状链表

块状链表可以看作 FHQ Treap 的平替。

>[!hint]- 为什么？
>FHQ Treap 最有用功能是支持**区间操作**（翻转、删除、插入、旋转）。它能够在 $O(\log n)$ 的复杂度完成前面的这些操作，但是块状链表却需要 $O(\sqrt n)$ 的时间复杂度。

>[!example] P4008
>一开始文本为空，光标在文本开头，也就是 1 位置，请实现如下 6 种操作
>-  `Move k`：将光标移动到第 $k$ 个字符之后，操作保证光标不会到非法位置
>- `Insert n s`：在光标处插入长度为 $n$ 的字符串 $s$，光标位置不变
>- `Delete n`：删除光标后的 $n$ 个字符，光标位置不变，操作保证有足够字符
>- `Get n`：输出光标后的 $n$ 个字符，光标位置不变，操作保证有足够字符
>- `Prev`：光标前移一个字符，操作保证光标不会到非法位置
>- `Next`：光标后移一个字符，操作保证光标不会到非法位置
>
>`Insert`操作时，字符串 $s$ 中 ASCII 码在 $[32,126]$ 范围上的字符一定有 $n$ 个，其他字符请过滤掉

诚然，`__gnu_cxx::rope` 秒了，这个 stl 我们在 [[6. __gnu_cxx 与 pb_ds 中的若干容器]] 中介绍，我们现在先学习一下块状链表怎么解决这道题。

我们规定链表的容量固定，但是内容大小不定，可能未满。也就是用链表连接的一堆数组。

这样，如果块内遍历大致是 $O(L)$，块间就是 $O\left(\dfrac{n}{L}\right)$，一样在 $L\sim\sqrt n$ 取到下界。

先从 `Insert` 操作开始。比如我们在第 $X$ 块的 $pos$ 插入一个 $s$，那么我们现在在 $pos$ 处把 $X$ 拆成两个节点，$X\to Y$。$X$ 储存 $pos$ 之前的，$Y$ 储存 $pos$ 之后的。我们在 $X$ 后面不断插入节点，直到 $s$ 能够完全插入，这样就实现了 `Insert`。类似实现 `Delete` 以及其他的所有操作。

每一次找到 $pos$ 我们也暴力跳链表，这就是至多 $O(n/L)$ 的，单次操作也是 $O(n/L+L)$ 的。因此总复杂度下界就是 $O(q\sqrt n)$。

但是，这个复杂度可能不对。我们仔细思考，我们每次都进行了一个分裂的操作，会不会越分越小以至于最后块间遍历复杂度逼近 $O(n)$ 呢？这是容易解决的。我们在操作结束之后扫一遍所有的块。一旦可以合并我们就合并。这样相邻两项至少得是 $O(L)$ 的，单个至少是 $O\left(\dfrac L2\right)$ 的，因此最后总个数总是能够保证是 $O\left(\dfrac{2n}{L}\right)$。总复杂度就是 $O\left(\dfrac{2n}{L}+\dfrac L2\right)\gtrsim O(\sqrt n)$。我们依旧取 $L\sim 2\sqrt {n}$ 即可达成一个 $O(q\sqrt n)$ 的复杂度。

这样区间增删、区间移动的问题，往往利用块状链表可以获得一个比较快速且便捷的 $O(\sqrt n)$ 实现。

```cpp
#include <cstdio>
#include <cstring>
#include <iostream>
#include <vector>

using namespace std;

const int BLOCK_SIZE = 1500; // 块大小取 sqrt(N) 左右

struct Node {
    char data[BLOCK_SIZE << 1];
    int size;
    Node *next;

    Node() {
        size = 0;
        next = nullptr;
        memset(data, 0, sizeof(data));
    }
};

Node *head = new Node();
int cursor = 0; // 全局光标位置

// 辅助函数：分裂块，使得第 k 个字符是所在块的最后一个字符
void split(Node *p, int k) {
    if (p == nullptr || k >= p->size) return;
    Node *newNode = new Node();
    newNode->next = p->next;
    p->next = newNode;

    // 将 p 中 k 之后的字符移到 newNode
    memcpy(newNode->data, p->data + k, p->size - k); // 直接管理
    newNode->size = p->size - k;
    p->size = k;
}

// 定位光标所在的块以及在块内的偏移
void find(Node *&p, int &pos) {
    p = head;
    while (p != nullptr && pos > p->size) {
        pos -= p->size;
        p = p->next;
    }
}

// 维持块的大小平衡，防止退化
void maintain() {
    for (Node *p = head; p != nullptr; p = p->next) {
        while (p->next != nullptr &&
               p->size + p->next->size <= BLOCK_SIZE) {
            Node *del = p->next;
            memcpy(p->data + p->size, del->data, del->size);
            p->size += del->size;
            p->next = del->next;
            delete del;
        }
    }
}

void insert(int n) {
    Node *p;
    int pos = cursor;
    find(p, pos);
    split(p, pos);

    Node *cur = p;
    for (int i = 0; i < n;) {
        int len = min(n - i, BLOCK_SIZE);
        Node *newNode = new Node();
        newNode->next = cur->next;
        cur->next = newNode;

        for (int j = 0; j < len; ++j) {
            char ch = getchar();
            while (ch < 32 || ch > 126) ch = getchar();
            newNode->data[j] = ch;
        }
        newNode->size = len;
        i += len;
        cur = newNode;
    }
    maintain();
}

void remove(int n) {
    Node *p;
    int pos = cursor;
    find(p, pos);
    split(p, pos);

    Node *delHead = p->next;
    while (delHead != nullptr && n > 0) {
        if (n >= delHead->size) {
            n -= delHead->size;
            p->next = delHead->next;
            delete delHead;
            delHead = p->next;
        } else {
            split(delHead, n);
            Node *tmp = delHead;
            p->next = delHead->next;
            delete tmp;
            n = 0;
        }
    }
    maintain();
}

void get(int n) {
    Node *p;
    int pos = cursor;
    find(p, pos);

    int firstBlock = min(n, p->size - pos);
    for (int i = 0; i < firstBlock; ++i) putchar(p->data[pos + i]);
    n -= firstBlock;

    p = p->next;
    while (p != nullptr && n > 0) {
        int len = min(n, p->size);
        for (int i = 0; i < len; ++i) putchar(p->data[i]);
        n -= len;
        p = p->next;
    }
    putchar('\n');
}

int main() {
    int t;
    if (scanf("%d", &t) == EOF) return 0;
    char op[20];
    while (t--) {
        scanf("%s", op);
        if (op[0] == 'M') {
            scanf("%d", &cursor);
        } else if (op[0] == 'I') {
            int n;
            scanf("%d", &n);
            insert(n);
        } else if (op[0] == 'D') {
            int n;
            scanf("%d", &n);
            remove(n);
        } else if (op[0] == 'G') {
            int n;
            scanf("%d", &n);
            get(n);
        } else if (op[0] == 'P') {
            cursor--;
        } else if (op[0] == 'N') {
            cursor++;
        }
    }
    return 0;
}
```

>有一个纵向 $H$ 行、横向 $W$ 列的网格。初始时，从上往下第 $i$ 行、从左往右第 $j$ 列的单元格上写有一个英文字母 $A_{i,j}$。
>
>对这个网格进行 $Q$ 次操作。在第 $i$ 次操作中，将给出满足 $1 \leq a_i \leq H-1$, $1 \leq b_i \leq W-1$ 的整数 $a_i$ 和 $b_i$，然后执行以下操作：
>
>- 在网格内定义四个矩形区域 $R_1$, $R_2$, $R_3$, $R_4$，定义如下：
  >	- 从上往下 $a_i$ 行、从左往右 $b_i$ 列的部分为 $R_1$。
  >	- 从上往下 $a_i$ 行、从右往左 $W-b_i$ 列的部分为 $R_2$。
  >	- 从下往上 $H-a_i$ 行、从左往右 $b_i$ 列的部分为 $R_3$。
  >	- 从下往上 $H-a_i$ 行、从右往左 $W-b_i$ 列的部分为 $R_4$。
>- 将 $R_1$, $R_2$, $R_3$, $R_4$ 每个区域分别旋转 180 度。
>
>在完成全部 $Q$ 次操作后，请输出操作后网格的状态。
>数据范围：$Q\leq 2\times 10^5, HW\leq 5\times 10^5$。

一个比较简单的思考是我们分开行列，分别维护最后第 $i$ 行是一开始的第几行，第 $j$ 列是一开始的第几列。这样单次操作是 $O(\sqrt h+\sqrt w)$ 的，最后的统计是 $O(hw(\sqrt h+\sqrt w))$ 的。这样总时间复杂度大致就是 $O((q+hw)\sqrt{hw})$。差不多 $5\times 10^8$ 次运算，应该也许能过。

### 二分答案的应用

>[!example] P5356
>给你一个长为 $n$ 的序列 $a$，需要支持 $m$ 次操作，操作有两种：
>
>1. 查询区间 $[l,r]$ 的第 $k$ 小值。
>2. 区间 $[l,r]$ 加上 $k$。

我们首先思考**二分查找**，但是这只能找到一个数。那二分查找不行，我们就想**二分答案**。

**二分答案是非常好用的！！** 然后就是试块长。我试出来 500 可以过。时间复杂度是 $O\left(q\left(L\log L+\dfrac{2n}{L}+L+\log V\cdot \dfrac nL \log L\right)\right)\gtrsim O(q\sqrt{n}\log n)$，取等当 $L\sim \sqrt{n}$，理论上 $L\approx 892$ 是最优解，但实测会 tle，有点玄学。

```cpp
#include <bits/stdc++.h>

using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

inline int read() {
    int x = 0, f = 1;
    char ch = getchar();
    while (ch < '0' || ch > '9') {
        if (ch == '-') f = -1;
        ch = getchar();
    }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + (ch - '0');
        ch = getchar();
    }
    return x * f;
}

const int MAXN = 1e5 + 5;
const int MAXB = 350;

int n, m;
int siz;
int a[MAXN], block[MAXN], lazy[MAXB];

inline int get_id(int x) { return (x - 1) / siz; }
inline int l(int id) { return id * siz + 1; }
inline int r(int id) { return min(n, (id + 1) * siz); }

void build(int id) {
    memcpy(block + l(id), a + l(id),
           (r(id) - l(id) + 1) * sizeof(int));
    sort(block + l(id), block + r(id) + 1);
}

void modify(int ml, int mr, int val) {
    int bl = get_id(ml), br = get_id(mr);
    if (bl == br) {
        for (int i = ml; i <= mr; i++) a[i] += val;
        build(bl);
        return;
    }
    for (int i = ml; i <= r(bl); i++) { a[i] += val; }
    build(bl);
    bl++;
    for (int i = mr; i >= l(br); i--) { a[i] += val; }
    build(br);
    br--;
    for (int i = bl; i <= br; i++) { lazy[i] += val; }
}

int check(int cl, int cr, int val, int k) {
    int bl = get_id(cl), br = get_id(cr), cnt = 0;
    if (bl == br) {
        for (int i = cl; i <= cr; i++) {
            if (a[i] + lazy[bl] <= val) cnt++;
        }
        return cnt;
    }
    for (int i = cl; i <= r(bl); i++) {
        if (a[i] + lazy[bl] <= val) cnt++;
    }
    bl++;
    for (int i = cr; i >= l(br); i--) {
        if (a[i] + lazy[br] <= val) cnt++;
    }
    br--;
    for (int i = bl; i <= br; i++) {
        int v = val - lazy[i];
        cnt += (upper_bound(block + l(i), block + r(i) + 1, v) -
                block - l(i));
    }
    return cnt;
}

int query(int ql, int qr, int k) {
    int bl = get_id(ql), br = get_id(qr);

    ll L_bound = 2e18, R_bound = -2e18;
    auto update = [&](ll val) {
        if (val < L_bound) L_bound = val;
        if (val > R_bound) R_bound = val;
    };

    if (bl == br) {
        for (int i = ql; i <= qr; i++) update((ll) a[i] + lazy[bl]);
    } else {
        for (int i = ql; i <= r(bl); i++)
            update((ll) a[i] + lazy[bl]);
        for (int i = l(br); i <= qr; i++)
            update((ll) a[i] + lazy[br]);
        for (int i = bl + 1; i <= br - 1; i++) {
            update((ll) block[l(i)] + lazy[i]);
            update((ll) block[r(i)] + lazy[i]);
        }
    }

    ll ans = L_bound;
    ll left = L_bound, right = R_bound;
    while (left <= right) {
        ll mid = left + right >> 1;
        if (int q = check(ql, qr, mid, k); q >= k) {
            right = mid - 1;
            ans = mid;
        } else
            left = mid + 1;
    }
    return ans;
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    n = read(), m = read();
    for (int i = 1; i <= n; i++) a[i] = read();
    siz = 500;
    for (int i = 0; i <= get_id(n); i++) build(i);
    for (int i = 1; i <= m; i++) {
        int op = read(), l = read(), r = read(), k = read();
        if (op == 2) {
            modify(l, r, k);
        } else {
            cout << query(l, r, k) << '\n';
        }
    }
    return 0;
}
```

### 时间分块·操作分块

>[!example] P3863
>给定一个长度为 $n$ 的序列，给出 $q$ 个操作，形如：
>- `1 l r x` 表示将序列下标介于 $[l,r]$ 的元素加上 $x$ （请注意，$x$ 可能为负）
>- `2 p y` 表示查询 $a_p​$ 在过去的多少秒时间内不小于 $y$ （不包括这一秒）
>
>开始时为第 0 秒，第 $i$ 个操作发生在第 $i$ 秒。
>
>数据范围：$n,q\leq 10^5,|x|,|y|,|a_i|\leq 10^9$。

这是典型的**时间分块**问题。很显然加上时间这一维之后我们显然划归成了一个二维数点问题。如果只是实现维护朴实的二维信息（加、减、求和...）那很简单，最简单的扫描线即可，选择一个维度扫过去，另外一个维护用数据结构维护。这个典型的操作就是通过扫描线来解决。

问题是，我们扫哪一维？很显然我们不能扫时间这一维，不然问题没有发生任何转化。因此我们只能够扫描位置这一维。

看这张图。发现如果只维护一个数字的话可以时间分块解决时间区间加、单点查询历史不小于某个值的时间的问题。

我们尝试把上面这个东西从维护一个数字变成维护 $n$ 个数字。不妨直接在分块的基础上增加一维，变成一个二维的问题，时间维仍然是分块，空间维可以从前往后一个一个地处理，这也就是扫描线的思想。这样单次修改，查询都是 $O\left(\underbracket{L}_{\text{扫散块}}+\max\left\{\underbracket{\dfrac{q}{L}\log L}_{\text{扫整块}},\underbracket{L\log L}_{\text{修改操作 build 排序}}\right\}\right)$，纵沟查询 $q$ 次；往右扫一格是 $O\left(L+\dfrac{q}{L}\right)$（全体加这个相差前面的差值），总共扫 $n$ 次，因此取 $L\sim\sqrt q$ 的话，总的时间复杂度大致就是 $O(q\log q+n\sqrt q+q\sqrt q\log q)\sim O(n\sqrt q+q\sqrt q\log q)$ 的，然后记得用一下前面提到的 lazy 优化，这就足以通过此题了。

```cpp
#include <bits/stdc++.h>

using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using ll = long long;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

inline int read() {
    int x = 0, f = 1;
    char ch = getchar();
    while (ch < '0' || ch > '9') {
        if (ch == '-') f = -1;
        ch = getchar();
    }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + (ch - '0');
        ch = getchar();
    }
    return x * f;
}

#define int ll

const int MAXN = 1e5 + 5;
const int MAXB = 350;

int n, q;
int siz;
int a[MAXN], block[MAXN], lazy[MAXB];
vector<tuple<int, int, int>> queries[MAXN];
vi ans;

inline int get_id(int x) { return (x - 1) / siz; }
inline int get_l(int id) { return id * siz + 1; }
inline int get_r(int id) { return min(q, (id + 1) * siz); }

void build(int id) {
    memcpy(block + get_l(id), a + get_l(id),
           (get_r(id) - get_l(id) + 1) * sizeof(int));
    sort(block + get_l(id), block + get_r(id) + 1);
}

void add(int al, int ar, int val) {
    int bl = get_id(al), br = get_id(ar);
    if (bl == br) {
        for (int i = al; i <= ar; i++) a[i] += val;
        build(bl);
        return;
    }
    for (int i = al; i <= get_r(bl); i++) a[i] += val;
    build(bl), bl++;
    for (int i = ar; i >= get_l(br); i--) a[i] += val;
    build(br), br--;
    for (int i = bl; i <= br; i++) lazy[i] += val;
}

int query(int ql, int qr, int val) {
    int bl = get_id(ql), br = get_id(qr), ret = 0;
    if (bl == br) {
        for (int i = ql; i <= qr; i++)
            if (a[i] + lazy[bl] >= val) ret++;
        return ret;
    }
    for (int i = ql; i <= get_r(bl); i++)
        if (a[i] + lazy[bl] >= val) ret++;
    bl++;
    for (int i = qr; i >= get_l(br); i--)
        if (a[i] + lazy[br] >= val) ret++;
    br--;
    for (int i = bl; i <= br; i++) {
        int v = val - lazy[i];
        ret +=
            (get_r(i) + 1) -
            (lower_bound(block + get_l(i), block + get_r(i) + 1, v) -
             (block));
    }
    return ret;
}

signed main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    n = read(), q = read();
    for (int i = 1; i <= n; i++) {
        int t = read();
        queries[i].emplace_back(1, 1, t);
        queries[i + 1].emplace_back(1, 1, -t);
    }
    q++;
    siz = ceil(sqrt(q));
    for (int i = 2; i <= q; i++) {
        int op = read();
        if (op == 1) {
            int l = read(), r = read(), v = read();
            queries[l].emplace_back(i, 1, v);
            queries[r + 1].emplace_back(i, 1, -v);
        } else {
            int p = read(), y = read();
            queries[p].emplace_back(i, 2, y);
        }
    }
    ans.assign(q + 1, -1);
    for (int i = 1; i <= n; i++) {
        for (auto [t, op, val] : queries[i]) {
            if (op == 1)
                add(t, q, val);
            else
                ans[t] = query(1, t - 1, val);
        }
    }
    for (int i = 2; i <= q; i++) {
        if (ans[i] == -1) continue;
        cout << ans[i] << '\n';
    }
    return 0;
}
```

>在一片广袤无垠的原野上，散落着 $N$ 块磁石。
>
>每个磁石的性质可以用一个五元组 $(x,y,m,p,r)$ 描述，其中 $x,y$ 表示其坐标，$m$ 是磁石的质量，$p$ 是磁力，$r$ 是吸引半径。
>
>若磁石 $A$ 与磁石 $B$ 的距离不大于磁石 $A$ 的吸引半径，并且磁石 $B$ 的质量不大于磁石 $A$ 的磁力，那么 $A$ 可以吸引 $B$。
>
>小取酒带着一块自己的磁石 $L$ 来到了这片原野的 $(x_0,y_0)$ 处，我们可以视磁石 $L$ 的坐标为 $(x_0,y_0)$。
>
>小取酒手持磁石 $L$ 并保持原地不动，所有可以被 $L$ 吸引的磁石将会被吸引过来。
>
>在每个时刻，他可以选择更换任意一块自己已经获得的磁石（当然也可以是自己最初携带的 $L$ 磁石）在 $(x_0,y_0)$ 处吸引更多的磁石。
>
>小取酒想知道，他最多能获得多少块磁石呢？
>
>数据范围：$1 \le N \le 250000$，$-10^9 \le x,y \le 10^9$，$1 \le m,p,r \le 10^9$。
