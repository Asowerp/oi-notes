#树 #区间查询 

本文将对 Sqrt Tree 进行讲解。在 $N = Q$ 左右的情况下，它比线段树或 DST 能达到更好的时间复杂度，而且思路简单，非常有趣。

此外，将整个数列分割成若干块，根据查询区间与块边界的位置关系进行分类处理（特别是完全包含在块内的查询成为问题）的方法，也出现在线性时间 LCA、线性时间 RMQ 中，并且可以推广到用 $O((N+Q)\alpha(N))$ 时间处理区间积查询的问题（静态数列的区间积查询）。
## 概要

Sqrt Tree 是通过对数列递归地进行平方分割而得到的树形结构。特别地，对于静态的幺半群数列，它可以高速处理区间积（对应幺半群中的运算）查询。具体来说，设数列长度为 $N$，在 $O(N \log \log N)$ 时间的预处理下，每个查询可以在 $O(1)$ 时间内处理区间积。

作为区间积查询的解法，使用 [[4. 线段树]] 或 [[2. ST 表]]（以下简称 DST）的方法也很著名。但在查询个数 $Q$ 接近 $N$ 的情况下，Sqrt Tree 比线段树或 DST 能达到更好的时间复杂度。
## 算法介绍

我们通过下面这个题目来引入 Sqrt Tree。

>[!example]
>设 $(M, *)$ 是一个幺半群。给定一个由 $M$ 中元素组成、长度为 $N$ 的数列 $A = (A_0, A_1, \dots, A_{N-1})$。请回答 $Q$ 个查询。每个查询给定满足 $0 \le L < R \le N$ 的整数 $L, R$，要求输出
>$$A_L * A_{L+1} * \cdots * A_{R-1}$$
>另外，在分析时间复杂度时，假设对 $M$ 中元素的操作（数组读写或幺半群运算）花费 $O(1)$ 时间。

### 算法原理

Sqrt Tree 是通过对数列递归地进行平方分割而得到的树形结构。首先，我们暂时忽略「递归地进行」这一点，只考虑单纯使用平方分割时能做的事情。

取块大小 $S = \lfloor \sqrt{N} \rfloor$，将整个数列分割成 $K = \lfloor N/S \rfloor$ 个块。给块编号 $0, 1, \dots, K-1$，预先计算以下三个表：
- **Prefix**：对于属于块 $j$ 的每个 $A_i$，计算从块 $j$ 的开头到 $A_i$ 的总积。
- **Suffix**：对于属于块 $j$ 的每个 $A_i$，计算从 $A_i$ 到块 $j$ 的末尾的总积。
- **Between**：对于每个块对 $(i, j)$（其中 $i \le j$），计算从块 $i$ 的开头到块 $j$ 的末尾的总积。

当 $N=12, S=3$ 时，这些表中存储了哪些区间的总积，如下图所示。

![[Pasted image 20260413155834.png]]

这样的预处理可以在 $O(N)$ 时间内完成。基于以上预处理，我们来考虑回答查询。设查询区间为 $[L, R)$，如果 $A_L$ 和 $A_{R-1}$ 所属的块不同，则可以在 $O(1)$ 时间内回答查询。具体地，将 $[L, R)$ 分割成以下三个子区间：
- 从 $A_L$ 到 $A_L$ 所属块的末尾；
- 夹在 $A_L$ 所属块与 $A_{R-1}$ 所属块之间的完整块；
- 从 $A_{R-1}$ 所属块的开头到 $A_{R-1}$。
其中，第二个「夹在中间的完整块」可能为空。

![[Pasted image 20260413160104.png]]

这三个区间的总积（如果非空）都包含在预计算的值中。因此，通过从表中读取这些值，然后进行 2 次幺半群运算，就可以回答区间查询。

问题来了，如果查询区间完全落在一个块内，那怎么办呢？这时候我们通过对上面的分割递归进行来解决。这也就是所谓的 Sqrt Tree。
- 将区间 $[0, N)$ 称为「第 0 层」。
- 对于非负整数 $d$，将第 $d$ 层的区间按照上述方式进行平方分割得到的区间称为「第 $d+1$ 层」，从而定义第 $1, 2, 3, \dots$ 层的区间。
- 当区间长度达到某个适当阈值（例如 2）时停止分割。
例如，对于长度为 16 的数列，Sqrt Tree 的结构如下图所示。

![[Pasted image 20260413160131.png]]

对于 Sqrt Tree 中除叶子以外的所有节点，执行前文所述的预处理。即，在第 $d$ 层的区间中：
 - 对每个元素 $A$，考虑 $A$ 所属的第 $d+1$ 层区间，计算以下两项：
	  - 从该区间开头到 $A$ 的总积，
	  - 从 $A$ 到该区间末尾的总积。
 - 对于由第 $d+1$ 层区间的总积排列而成的数列，计算其所有子区间的总积。

查询按照如下的方式查询。按 $d = 0, 1, 2, \dots$ 的顺序：
- 如果查询区间不完全包含在任何一个第 $d+1$ 层区间内，那么如 4.1 节所述，可以在 $O(1)$ 时间内回答查询。
- 否则，查询区间完全包含在某个第 $d+1$ 层区间内，则进入该节点。
如此沿着树结构向下深入。如果到达叶子节点，则直接暴力计算总积。通过将叶子区间的长度设为 $O(1)$，无论在哪一层结束查询，都可以用 $O(1)$ 次幺半群运算回答查询。

设 Sqrt Tree 的高度为 $H$。那么：
- 对于每个 $d = 0, 1, \dots, H-1$，设第 $d$ 层各区间长度为 $n_1, n_2, n_3, \dots$，则对每个区间进行 $O(n_i)$ 时间的预计算。固定 $d$ 时，这些区间的总长度为 $O(N)$，因此全部预计算的总时间为 $O(NH)$，空间同理。
- 查询处理中，幺半群运算的次数为 $O(1)$。此外还有树结构搜索的 $O(H)$ 时间，因此每个查询花费 $O(H)$ 时间。

现在问题在于，Sqrt Tree 的高度 $H$ 是多少？它是从 $N$ 开始，反复取平方根直到达到某个适当阈值所需的次数。由此可知 $H = O(\log \log N)$。

>[!hint] 为什么 H=O(log log N)
>设 $N = 2^x$，则 $\sqrt{N} = 2^{x/2}$。对 $N$ 反复取平方根 $k$ 次后，区间长度变为 $2^{x / 2^k}$。当这个长度小于等于某个常数阈值（比如 $2$）时停止，即要求 $x / 2^k \le C$，也就是 $2^k \ge x / C$，从而 $k \ge \log_2 x - \log_2 C$。因此所需次数 $k = O(\log x)$。
>
>而 $x = \log_2 N$，所以 $k = O(\log \log N)$。这就是 Sqrt Tree 的高度 $H = O(\log \log N)$ 的原因。

综上所述，使用 Sqrt Tree 可以在：
- 空间 $O(N \log \log N)$，时间 $O(N \log \log N)$ 的预计算下，
- 每个查询 $O(\log \log N)$ 时间
从而可以在总时间复杂度为 $O((N + Q) \log \log N)$ 的情况下解决前面的问题。

问题来了，我们前面不是说是 $O(N\log \log N)-O(1)$ 吗？为什么这里只能 $O(\log \log N)$ 查询？通过一些技巧，可以将每个查询的时间复杂度降至 $O(1)$。

为了做到 $O(1)$，只需要知道查询区间包含在哪个层次的区间中即可。对于每个 $d$，将第 $d$ 层区间的长度（除最右侧的区间外）调整为适当的 2 的幂次。此时，若使用 0-based 索引，则对于查询区间 $[L, R)$，通过 $L \oplus (R-1)$（$\oplus$ 为按位异或）的值可以决定查询区间包含在哪个层次的区间中。因此，可以预先计算每个 $L \oplus (R-1)$ 值对应的层次，或者适当地使用位运算，从而每个查询达到 $O(1)$ 时间。

这个和我们之前 [[4. 线段树]] 中提到的 zkw 线段树的想法一样，我们先把 $N$ 怼到 $2^k$，这样我们查询的时候更加方便。

## 4.4. 实现示例

```cpp
#include <bits/stdc++.h>

using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using i64 = long long;
using u64 = unsigned long long;
using i128 = __int128_t;
using u128 = __uint128_t;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

int read() {
    int x = 0, f = 1;
    char ch = getchar_unlocked();
    while (!isdigit(ch)) {
        if (ch == '-')
            f *= -1;
        ch = getchar_unlocked();
    }
    while (isdigit(ch)) {
        x = x * 10 + ch - '0';
        ch = getchar_unlocked();
    }
    return x * f;
}

template <typename Op> class SqrtTree {
    Op op;
    int n;
    vector<int> a;
    vector<vi> pref, suf, btwn;
    vi siz;
    vi lvl;
    int topbit(int x) { return x == 0 ? -1 : 31 - __builtin_clz(x); }
    void build(int lv) {
        vi p = a, s = a;
        int k1 = siz[lv], k2 = k1 >> 1;
        int mask = (1 << k2) - 1;
        for (int i = 0; i < n; i++) {
            if (mask & i) {
                p[i] = op(p[i - 1], a[i]);
            }
        }
        for (int i = n - 1; i >= 0; i--) {
            if (mask & i) {
                s[i - 1] = op(s[i], a[i - 1]);
            }
        }
        pref.emplace_back(p);
        suf.emplace_back(s);
        int _n = n >> k2;
        vi blk(_n);
        for (int i = 0; i < _n; i++) {
            blk[i] = s[i << k2];
        }
        vi b;
        for (int j = 0; j < _n; j++) {
            int s = j >> (k1 - k2) << (k1 - k2);
            b.emplace_back(blk[j]);
            for (int i = j - 1; i >= s; i--) {
                b.emplace_back(op(blk[i], b.back()));
            }
        }
        btwn.emplace_back(b);
    }

  public:
    SqrtTree(const vi &a) : a(a) {
        n = sz(a);
        int k = 1;
        for (; (1 << k) <= n; k++)
            ;
        lvl.assign(k + 1, -1);
        while (k > 1) {
            for (int i = k >> 1; i < k; i++)
                lvl[i] = sz(siz);
            siz.emplace_back(k);
            k >>= 1;
        }
        for (int lv = 0; lv < sz(siz); lv++)
            build(lv);
    }
    int query(int l, int r) {
        if (l == r)
            return a[l];
        int lv = lvl[topbit(l ^ r)];
        if (lv == -1) {
            return op(a[l], a[r]);
        }
        int k1 = siz[lv], k2 = k1 >> 1;
        int ans = suf[lv][l];
        int a = 1 + (l >> k2), b = (r >> k2) - 1;
        if (a <= b) {
            int m = 1 << (k1 - k2);
            int k = a >> (k1 - k2);
            a -= k * m, b -= k * m;
            int idx = k * (m * (m + 1) / 2); // 大块的起始索引
            idx += (b + 1) * b / 2;          // 加上右端点 b 之前的所有区间数
            idx += b - a;                    // 加上从 a 到 b 的偏移
            ans = op(ans, btwn[lv][idx]);
        }
        ans = op(ans, pref[lv][r]);
        return ans;
    }
};

struct Max {
  public:
    int operator()(int a, int b) { return max(a, b); }
};

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    int n = read(), q = read();
    vi a(n + 1);
    for (int i = 1; i <= n; i++)
        a[i] = read();
    SqrtTree<Max> st(a);
    for (int i = 1; i <= q; i++) {
        int l = read(), r = read();
        cout << st.query(l, r) << '\n';
    }
    return 0;
}
```
## 与其他数据结构的比较

线段树、DST、Sqrt Tree 的复杂度对比如下表所示。

| 数据结构      | 空间                 | 时间                     |
| --------- | ------------------ | ---------------------- |
| 线段树       | $O(N)$             | $O(N + Q \log N)$      |
| DST       | $O(N \log N)$      | $O(N \log N + Q)$      |
| Sqrt Tree | $O(N \log \log N)$ | $O(N \log \log N + Q)$ |
当 $N = Q$ 左右时，Sqrt Tree 在时间复杂度上优于线段树和 DST。特别是与 DST 相比，在渐进意义上 Sqrt Tree 更优。

然而，DST 对每个区间查询只需 1 次幺半群运算，而 Sqrt Tree 需要 2 次幺半群运算，因此在某些问题中 Sqrt Tree 并不能作为 DST 的完全上位替代。
