## 后缀树与后缀数组

> 后缀树和后缀数组可以解决大部分字符串问题，前面提到的字符串匹配问题，如查找子串、最长重复子串、最长公共子串等，都可以用后缀数组解决，这类题目是编程竞赛的常见题型。

### 后缀树

所谓后缀树，就是把所有后缀子串（按照原先在字符串中的顺序）通过 [[7. 字典树·前缀树（Trie）]] 建立一颗树。如下图所示，就是 `vamamadn` 的后缀树。

![[Pasted image 20260314165334.png]]

自然，因为 Trie 的空间表现比较差，后缀树的空间表现相对而言也比较差，大致是 $O(n^2)$ 的级别。所以一般而言不太用后缀树来储存后缀的信息。当然如果你用压缩后缀树，那根据后面 SAM 那里母树相关的讨论，我们可以知道这个复杂度也是 $O(n)$ 的，但是你都用 SAM 了你还用什么后缀树呢。
### 后缀数组

因此，后缀数组（SA，Suffix Array）应运而生。在下图中，后缀数组就是按字典序对应的后缀下标。下面的例子中就有 `sa[]={5,3,1,6,4,2,7,0}`。

换句话说，后缀数组的数字顺序就是后缀子串的字典顺序，记录了子串的有序排列。例如，`sa[0]=5` 意思是排名 0，即字典序最小，的子串，是原字符串中从第 5 个位置开始的后缀子串，即 "adn"。（下图第一个打错了）

![[Pasted image 20260314165519.png]]

利用这个和二分的思想，我们可以很快捷地实现一个 $O(|T|\log |S|)$ 的在 $S$ 中查找 $T$ 的方法。

```cpp
// 假设已经求好了 sa 数组

int find(string S, string T, int* sa) {
    int left = 0, right = S.length() - 1;
    
    // 二分查找
    while (left <= right) {
        int mid = (left + right) / 2;
        
        // 比较：从 sa[mid] 位置开始，取 T.length() 长度的子串，与 T 比较
        int cmp = S.compare(sa[mid], T.length(), T);
        
        if (cmp == 0) {
            // 找到了，返回起始位置
            return sa[mid];
        } 
        else if (cmp < 0) {
            // 当前后缀小于 T，向右半区查找
            left = mid + 1;
        } 
        else {
            // 当前后缀大于 T，向左半区查找
            right = mid - 1;
        }
    }
    
    return -1; // 没找到
}
```

现在问题转化为，如何快速求出 `sa` 数组呢？一个标准方法是倍增法，可以实现 $O(|S|\log^2 |S|)$ 的时间复杂度初始化。可以发现，上面的代码实际上做到了 $O(|S|\log^2 |S|+|T|\log |S|)$ 的字符串查找，相比 KMP 的 $O(|S|+|T|)$，其实差距不大。更甚，当 $|S|\gg|T|$ 的时候，反而会有更好的表现。
### 后缀数组的实现

为了方便叙述，下面令 $n:=|S|$。

如果我们直接快速排序，那么注意到字符串之间的比较是 $O(n)$ 的，那么总的时间复杂度将会是 $O(n^2\log n)$，这是不可接受的。于是我们考虑用别的方法来实现排序。

一个典型的方法是「倍增」。它的思路是，每次从第一个字符开始比较，然后每次倍增比较的长度，直到比较完成。它的最基本的想法是：后缀的前缀是决定字典序的关键，所以我每次倍增前缀的长度，一旦全都不一样了，就可以快速地比较了。

所以我每一步都去合并。如果没有合并的了就补 0。

![[Pasted image 20260314171400.png]]

这个过程中，倍增至多进行 $O(\log n)$ 次，最后再排序数字，这是 $O(n\log n)$ 的，因此时间复杂度就是 $O(n\log n)$。

然而，如果是 $n$ 很大的情况，数字根本存不下来。根本不可能做到 $O(n\log n)$。于是我们可以考虑在中间先排序一下压到 $[n]$ 之后再拼起来，这样子不会改变最终答案的正确性（这是因为如果相同，那么排序之后依然相同，反之亦然，整个排序的过程是保证顺序的）。

那么这个过程中，我们引入 `rk[]` 数组。最后得到的 `rk[]` 也是 $0 \sim n-1$ 的全排列，保存 suffix(i) 在所有后缀中按字典序排序的名次。也就是说，`rk[sa[i]]=i`，这是因为 `sa[i]` 存的是在后缀中按照字典序排序的名次为 $i$ 的字符串编号，而 `rk[i]` 表示这个字符串在后缀中按照字典序排序的名次。由此可见，$\texttt{sa}\circ \texttt{rk}=\rm id$，也就是说我们可以处理其中一个，自然就可以通过 `rk[sa[i]]=i` 或 `sa[rk[i]]=i` 的赋值得到另外一个。下文中，$\text{rank}$ 指代 `rk` 数组。

现在假设已经求得了长度为 $k$ 的子串的顺序，要求长度为 $2k$ 的子串的顺序。记 $\text{rank}_k(i)$ 为 $S[i, i+k-1]$ （不足补空）在所有排好序的长度为 $k$ 的子串中是第几小的。要计算长度为 $2k$ 的子串的顺序，就只要对两个 `rk` 组成的数对进行排序就好了。

我们通过对 $\text{rank}_k(i)$ 与 $\text{rank}_{k+1}(i+k)$ 的数对和 $\text{rank}_k(j)$ 与 $\text{rank}_{k+1}(j+k)$ 的数对的比较来替代对 $S[i, i+2k-1]$ 和 $S[j, j+2k-1]$ 的直接比较。因为比较 $\text{rank}_k(i)$ 和 $\text{rank}_k(j)$ 就相当于比较 $S[i, i+k-1]$ 和 $S[j, j+k-1]$，比较 $\text{rank}_{k+1}(i+k)$ 和 $\text{rank}_{k+1}(j+k)$ 就相当于比较 $S[i+k, i+2k-1]$ 和 $S[j+k, j+2k-1]$。所以，我们可以这样高效地比较长度为 $2k$ 的子串，并将它们排序。

但是我们没必要开一个二维数组来记录 `rk`，我们只需要每一轮结束的时候进行排序就好了。我们注意到 `sa` 作为 $0\sim n-1$ 的排列是不能动的，因此为了实现上面的思路，代码的逻辑如下：
- 首先，在每步用当前的 `rk[]` 通过 `sort` 经过一次排序得到当前的 `sa[]`。
- 然后用 `sa[]` 更新下一步用到的 `rk[]`。
	- 注意到每步的 `sa[]`，其中任意两个 `sa[i]` 和 `sa[j]` 都不同，但是下一步的 `rk[]` 中有一些是相同的，即 `sa[]` 和 `rk[]` 还不是一一对应的；
	- 所以需要先用 `sa[]` 根据原来的 `rk[]` 中的记录推导新的 `rk[]`，这需要一个临时 `tmp[]` 数组存放新值，然后再赋值给 `rk[]`。只有到了最后，`sa[]` 和 `rk[]` 才是一一对应的。

```cpp
int n, k;
int rank[MAX_N + 1];
int tmp[MAX_N + 1];

// 比较(rank[i], rank[i + k])和(rank[j], rank[j + k])
bool compare_sa(int i, int j) {
    if (rank[i] != rank[j]) return rank[i] < rank[j];
    else {
        int ri = i + k <= n ? rank[i + k] : -1;
        int rj = j + k <= n ? rank[j + k] : -1;
        return ri < rj;
    }
}

// 计算字符串S的后缀数组
void construct_sa(string S, int *sa) {
    n = S.length();

    // 初始长度为1，rank直接取字符的编码
    for (int i = 0; i <= n; i++) {
        sa[i] = i;
        rank[i] = i < n ? S[i] : -1;
    }

    // 利用对长度为k的排序的结果对长度为2k的排序
    for (k = 1; k <= n; k *= 2) {
	    // 先一步 sort 得到长度为 2k 的时候的 sa 是什么样的
        sort(sa, sa + n + 1, compare_sa);

        // 先在tmp中临时存储新计算的rank，再转存回rank中
        // 利用上面的到的 sa 反过来计算当前的 rank
        // 这里不能直接赋值 rank 是因为排名相邻两个还需要用 rank
        tmp[sa[0]] = 0;
        for (int i = 1; i <= n; i++) {
            tmp[sa[i]] = tmp[sa[i - 1]] + (compare_sa(sa[i - 1], sa[i]) ? 1 : 0);
        }
        for (int i = 0; i <= n; i++) {
            rank[i] = tmp[i];
        }
    }
}
```

这样，至多倍增 $O(\log n)$ 次，每次排序 $O(n\log n)$，时间复杂度就是 $O(n\log^2n)$。在大多数场景下，这已经够用了。

但是，有些极端情况下，我们需要掉一个 $\log$，这时候我们就需要所谓的「基数排序」了。我们每次先遍历一遍，并开一个第二关键字的桶，按照第二关键字塞进去，然后从小到大遍历这些桶，按照第一关键字塞到新的桶里面去。这样两次遍历得到的就是一个排好序的数组了，这样排序的时间复杂度是 $O(n)$ 的。总的时间复杂度就变成了 $O(n\log n)$，十分优秀。

>[!example] 基数排序的例子
>打个比方，我们要排序一大堆两位数。例如，排序 $(47,23,19,17,31)$，步骤如下。
>
>第 1 步，先按个位大小排序，得到 $(31,23,47,17,19)$。
>
>第 2 步，再按十位大小排序，得到 $(17,19,23,31,47)$，结束，得到有序排列。
>
>上面的「排序」，指的就是直接把数字放到对应的桶中，第 1 步按个位放，第 2 步按十位放。

一个实现代码如下：

```cpp
void build() {
    int m = 127;
    int i, *rk = t1, *y = t2;
    // 下面这里实现了第一轮的排序（k=1）
    for(i = 0; i < m; i++) cnt[i] = 0;
    for(i = 0; i < n; i++) cnt[rk[i] = s[i]]++; 
			    // 统计每个字符 rk[i]=s[i] 的同时记录 cnt[s[i]]
    for(i = 1; i < m; i++) cnt[i] += cnt[i - 1];
			    // cnt 现在处理前缀和，表示当前的末尾指向的排名
    for(i = n - 1; i >= 0; i--) sa[--cnt[rk[i]]] = i;
			    // 先更新排名（因为是 0~n-1 的），当前这个后缀就是这个更新后的排名
			    // 所以按照 sa 的定义，这个排名对应的 sa 就是原来的编号 i
    // ----------------------------------------------
    for(int k = 1; k <= n; k *= 2) { //利用对长度为 k 的数组的排序结果对长度为 2k 的
    //数组排序
        int p = 0;
        // 先按第二关键字排序，我们接下来都是在收集起点！
        for(i = n - k; i < n; i++) y[p++] = i;
			        // 首先处理第二关键字为空的
			        // 这是因为 s[i+k,i+2k-1] 为空，所以直接扔到最前面
        for(i = 0; i < n; i++) if(sa[i] >= k) y[p++] = sa[i] - k;
				    // 这里相当于在从小到大遍历 rk 数组
			        // sa[i]>=k 意思就是
				    // 上一轮排序第 i 大的，前面有 k 个字符了
				    // 换言之这个字符串可以作为某个字符串的第二变量。
				    // 因此按照第二变量遍历顺序，这个就是当前最小的第二变量
				    // 它对应的起点就是 sa[i]-k
				    // 从而，我们应该排到 p 这个位置然后 p++
        // 再按第一关键字排序，这里是类似第一轮的方法的
        for(i = 0; i < m; i++) cnt[i] = 0;
        for(i = 0; i < n; i++) cnt[rk[i]]++;
					// 统计第一关键字的个数
        for(i = 1; i < m; i++) cnt[i] += cnt[i - 1];
			        // 做前缀和
        for(i = n - 1; i >= 0; i--) sa[--cnt[rk[y[i]]]] = y[i];
			        // 先更新排名，然后 y[i] 指的是按照第二关键字排序的字符串的起点
			        // 我们去看对应的第一关键字（rk），按照前面基数排序的原理
			        // 这个排名对应的起点位置就是 y[i] 所指向的起点
        swap(rk, y); // 这里 y 没有用，因为每次循环都要重新更新
				    // 这里相当于弃置了，因为我们接下来要更新下一轮的 rk
				    // 同时也要用到这一轮的 rk（rk），所以我们换过去
        p = 1; rk[sa[0]] = 0; // 不同字符串至少有 1 个，然后开头的排位是 0
        // 前面 sa 已经排好序了，所以我们接下来直接用
        for(i = 1; i < n; i++) // 按照排名遍历
            rk[sa[i]] =
            y[sa[i - 1]] == y[sa[i]] && y[sa[i - 1] + k] == y[sa[i] + k] ? p - 1 : p++; // 如果前面这一个跟后面这一个在这一轮的排名完全一样
             // 并且加了 k 之后两个对应位置的排名也一样
             // 说明这两个完全一样，因此当前的就是不同个数-1（因为 0-based）；
             // 反之他是作为一个新的不同加进来的，所以赋 p 并且 p++。
        if(p >= n) break;
	        // 如果 rk_i 互不相同，说明已经做完了所有的排序了，也就是说 sa 搞好了
        m = p; // 不同字母的数量，供给下一轮
    }
}
```

利用这个可以 AC [P3809](https://www.luogu.com.cn/problem/P3809)。

在 `sa` 和 `rk` 之外，后缀数组的一个关键技术是 `height[]` 数组。它和最长公共前缀相关。使用 SA 的很多题目都与 `height` 脱不了干系。

首先，我们规定所谓最长公共前缀（LCP）表示两个字符串的最长的公共前缀长度。

记 $\newcommand{LCP}{\operatorname{LCP}}\LCP(i, j)$ 是 `suffix(sa[i])` 与 `suffix(sa[j])` 的最长公共前缀长度，即排序后第 $i$ 个后缀和第 $j$ 个后缀的最长公共前缀长度。仔细思考可以发现，
$$
\LCP(i, j) = \min\{\LCP(k-1, k)\}, \quad i < k \leq j
$$
那么我们规定 
$$\mathrm{height}_i=\LCP(i-1,i).$$
例如，在 “vamamadn” 中，`sa[1]` 对应 “amadn”，`sa[2]` 表示 “amamadn”，那么 `height[2]=3`，表示 `sa[1]` 和 `sa[2]` 这两个后缀的前 3 个字符相同。

暴力去做肯定是 $O(n^2)$ 的，但是利用 `rk` 和 `sa` 我们可以做到 $O(n)$ 解决。我们只需要注意到下面这个性质：

>[!hint] 性质
> 如果 `height[rk[i]] = h`，且 $h > 0$，那么对于下一个后缀 `S[i+1:]`，其对应的 `height[rk[i+1]]` 至少是 $h-1$。

我们下面证明之。假设起始位置为 $i-1$ 的后缀，在**后缀数组**中排它前面的那个后缀起始位置是 $j-1$。它们的最长公共前缀长度为 $k$。既然 $\newcommand{\Suffix}{\operatorname{Suffix}}\Suffix(i-1)$ 和 $\Suffix(j-1)$ 有长度为 $k$ 的公共前缀。那么去掉第一个字符，剩下的 $\Suffix(i)$ 和 $\Suffix(j)$ 至少有长度为 $k-1$ 的公共前缀。而 $k-1=\LCP(j,i)=\min_{j<k\leq i}\{\LCP(k-1,k)\}\leq \LCP(i-1,i)$，这就完成了证明。

所以可以写出下面的代码：

```cpp
for(i = 0; i < n; i++) {
    if(k) k--;                 // 我们刚刚处理完i-1的
								// 现在的 k 就是height[rk[i-1]]
								// 也就是起始位置为 i-1 的后缀
    int j = sa[rk[i] - 1];     // j 是排 S[i:] 前面的那个后缀的起始位置
    while(s[i + k] == s[j + k])// 所以直接从第 k 位开始继续比较，看还能匹配多少
        k++;
    height[rk[i]] = k;         // 将计算得到的长度赋值给对应的 height 数组
}
```

### 后缀数组的应用

有以下这些经典问题可以通过 SA 解决。
 - 查找子串。
 - 查找最长重复子串。我们先求 height 数组，然后不难发现最大值就是最长重复子串的长度。
 - 找到 $s_1,s_2$ 的最长公共子串。不难发现令 $s\gets s_1+s_2$ 就转化为了最长重复子串的问题。这样时间复杂度就从 $O(nm)$ 变成 $O((n+m)\log(n+m))$ 了，非常实用。
 - 找到最长回文子串。不难发现令 $s'\gets s+\text{reverse}(s)$ 也转化为了最长重复子串问题。但是相对 Manacher 还是相对较劣。

## 后缀自动机

这一部分的撰写部分参考了 https://www.luogu.com.cn/article/a5cq0p9r 与 https://cp-algorithms.com/string/suffix-automaton.html 。

在前面的后缀数组中，我们发现后缀数组虽然可以有优秀的 $O(n\log n)$ 的时间复杂度和 $O(n)$ 的空间复杂度，但是逻辑、编码都过于复杂了。因此我们又想要启用后缀树这样的东西。但是 $O(n^2)$ 的空间复杂度太贵了，有没有 $O(n)$ 的数据结构呢？这就是所谓的后缀自动机（Suffix AutoMaton）。后缀自动机具有 $O(n)$ 的时空复杂度，不可谓不牛。

所谓后缀自动机，自然而然，就是能够存储识别 $S$ 的后缀的自动机。因此类似 [[2. AC 自动机]]，我们可以通过 DAG 实现。根据自动机的定义，把节点看作状态，节点之间的有向边是状态的转移，每条边上有一个字符表示转移值。有且仅有一个初始态 $t_0$​，从 $t_0$​ 出发能单向到达其他所有节点。有一些节点是终结状态，任意一条从 $t_0$​ 出发到达某个终结状态所经过的路径上的字符组合是 $S$ 的一个子串。不同的路径表示不同的子串，这些路径与 $S$ 的子串一一对应，不多也不少。后缀树也可以看作一种简单的后缀自动机，下面研究一种复杂度为 $O(n)$ 的后缀自动机。

以 `abcbc` 为例，下图左侧是「字典树」，而右侧就是利用 DAG 存储的。优势显然。

![[Pasted image 20260314215411.png]]

那么，怎么得到一个类似的 DAG 呢？节点个数一定是 $O(n)$ 吗？我们按照递增法依次加入一个字符来建立这颗 DAG。
1. 加节点。在上次的末尾后面添加一个新的节点。比如，如果 $S$ 是 `ab`，节点 1 表示 `a`，下一个是 b，现在 $S$ 的子串有 `a,b,ab`，因此需要在节点 1 后面加一个节点 2 和边 $a\to b$，得到 ab。
2. 加边。由于 `b` 是还没有出现过的新字符，所以需要在 0 加一个边到 2；反之则不用加边。
3. 重复以上操作，直到遍历完成。

一个自然的问题是，这个 DAG 有多少节点？显然，每新加入一个字符，最长的子串就是 $S$ 本身，$S$ 的长度为 $n$，需要用一条长度为 $n$ 的路径表示。这条路径上共有 $n$ 个节点，加上根节点 $0$，至少有 $n+1$ 个节点。后面举例构造后缀自动机时将指出，节点数量最多为 $2n$，远小于后缀树的 $O(n^2)$。

上面这些操作优化的关键在于利用所有子串的包含关系建 DAG，这个图省去了重合的部分，从而减少存储空间，并附带提高了操作效率。仍然可以用字典树构建这个 DAG。

我们接下来说明，节点数量至多是 $O(n)$，为此，我们引入 `endpos` 的概念。下文中，记之为 $\newcommand{\endpos}{\operatorname{endpos}}\endpos$。我们定义，$\endpos(T)$ 表示子串 $T$ 在 $S$ 中所有出现位置的右端点集合。

>[!hint]+ endpos 是什么？
>简单来说，**endpos 决定了哪些子串可以“共用”同一个状态**。
> - **朴素想法（字典树）**：把每个子串都单独列出来，节点数会是 $O(n^2)$（就像你图中提到的后缀树是 $O(n^2)$ 的情况，但那通常是指后缀树占用的空间是 $O(n)$，这里的 $O(n^2)$ 可能指未优化的 Trie 树）。这导致空间爆炸。  
> - **优化思路（后缀自动机）**：把所有 **endpos 相同** 的子串合并到同一个节点里。

下面以 $S = \texttt{"abcbc"}$ 为例，说明如何计算 $\endpos$。先给出 $S$ 中每个字符的位置，从 $1$ 开始计数。

| 位置  |  1  |  2  |  3  |  4  |  5  |
| :-: | :-: | :-: | :-: | :-: | :-: |
| $S$ |  a  |  b  |  c  |  b  |  c  |
$S$ 共有 $12$ 个子串，如 $\texttt{"bc"}$ 的末尾在 $S$ 的第 $3$ 位置和第 $5$ 位置，$\texttt{"bc"}$ 的 $\endpos$ 为 $\{3, 5\}$。把 $\endpos$ 相等的称为等价类，如 $\texttt{'c'}$ 和 $\texttt{'bc'}$ 的 $\endpos$ 都等于 $\{3, 5\}$，是等价类。除了 $\varnothing$，一共有 $7$ 个等价类。如下图所示。

![[Pasted image 20260315150117.png]]

$\endpos$ 体现了后缀之间的包含关系，具有以下性质。
1. 同一个等价类中，较短的子串是较长子串的后缀。
2. 同一个等价类中，子串长度互不相同，并且依次递增 1。
3. 如果 $u$ 是 $v$ 的后缀，那么 $\endpos(u)\subset \endpos(v)$。
4. 对于 $u\neq v$，如果 $|u|<|v|$，那么要么 $\endpos(u)\subset \endpos(v)$，要么 $\endpos(u)\cup \endpos(v)=\varnothing$。
5. 一个长度为 $n$ 的字符串 $S$ 的等价类数量不超过 $2n$。

>[!hint]- 性质 5 的证明
>前面 4 个性质是不言自明的。我们下面证明性质 5。首先，性质 4 保证了要么包含，要么相斥，这启发我们建立一颗节点是集合树来描述这个包含关系。
>
>首先它的根节点就是 $\endpos(\varnothing)=[n]$。如果 $\endpos(u)\subsetneqq\endpos(v)$ 且不存在 $w$ 使得 $\endpos(u)\subsetneqq\endpos(w)\subsetneqq\endpos(v)$，我们就说 $\endpos(v)$ 是 $\endpos(u)$ 的父亲。一个比较显然的事实是如果一棵树有 $k$ 个叶子节点，且每个内部节点都至少有两个子节点，那么总节点数 $V \leq 2k - 1$。那么，我们将说明我们这样构造的树满足这两个条件。这是因为：
 > - 叶子节点是那些不再包含其他 $\endpos$ 集合的集合。 由于 $S$ 的每个位置 $i \in [n]$ 最多作为一个单元素集合 $\{i\}$ 出现在叶子中，所以叶子节点的数量上限是 $n$。
 > - 假设有一个内部节点 $P$，它只有一个子节点 $C$。 这就意味着 $C$ 中出现的位置首先 $P$ 中都出现了，所以 $C\subset P$。另一方面，$P$ 中所有的元素必然存在一个子节点出现，反之这意味着某个后缀没有出现却在 $P$ 中的位置出现了，这是不可能的！这说明 $P\subset C$，因此 $P=C$，矛盾！因此这必然意味着它的 $\endpos$ 集合被拆分成了至少两个互不相交的子集，也就是每个非叶子节点都有两个子节点。
>
>综上所述，这棵树的节点数量就是至多 $2n-1$ 的，从而性质 5 得证。

由于一个 $\text{endpos}$ 等价类中的子串具有包含关系，那么后缀自动机 SAM 的状态就用等价类来表示，一个状态就是一个等价类。这样就实现了对普通后缀树的优化。

性质 1 和性质 2 说明了同一个等价类内部子串的包含关系，且它们是连续的，是同一种状态。

性质 3 说明了状态如何转移。若 $\text{endpos}(u) \subset \text{endpos}(v)$，那么定义转移方向为从 $v$ 转移到 $u$，即 $v$ 是 $u$ 的父节点。

性质 5 说明了这个 SAM 的规模为 $O(n)$，远小于后缀树的 $O(n^2)$。

下面以 $S = \texttt{"abcb"}$ 为例按照上面的说法建一个树。初始状态 $t_0$ 是空集，每个节点是一个状态，即一个等价类。这棵树上只有 8 个点，少于后缀树的 13 个点。称这棵树为母树（Parent Tree），如图所示，它的每个节点是一个等价类。

![[Pasted image 20260315152053.png]]

这棵母树完整表达了所有子串。它有 5 个叶子节点，5 个叶子节点的 $\text{endpos}$ 正好是 $1$~$5$ 的完整位置。从根到一个叶子节点的路径上，包含了以这个位置为终点的所有后缀，如图左图中最右边的路径，包括 abcbc、bcbc、cbc、bc、c，是位置 $5$ 的全部后缀。

然而，「母树」这个概念并不实用。我们用节点表示子串集合，那么我们每个节点存储多个子串空间爆炸同时，统计的时候时间也爆炸了。SAM 的精妙之处就在于，我们可以通过「路径」来存储子串。SAM 的核心就在于所谓的「后缀链」。

记母树的节点 $v$ 上最长的一个子串为 $\text{longest}(v)$，长度为 $\text{len}(v)$。类似地，记 $\text{shortest}(v)$ 为最短子串，长度为 $\text{minlen}(v)$。这个节点上所有的子串都是 $\text{longest}(v)$ 的后缀，它们的长度覆盖了 $[ \text{minlen}(v), \text{len}(v) ]$ 的每个整数。例如，上面那颗母树的 $\{4\}$ 节点，是 "abcb" "bcb" "cb" 这 3 个子串，最长子串是 "abcb"，长度为 4。

我们定义，一个节点 $v$ 的 **后缀链接** 指向上层的一个节点 $u$（记为 $u = \text{link}(v)$），其中 $u$ 满足 $\text{len}(u) = \text{minlen}(v) - 1$，换言之，$u$ 就是 $v$ 中最短的子串去掉最前面的一个字符串所在的等价类。例如，图中节点 $v = \{5\}$ 的最短子串 "cbc"，$\text{minlen}(v) = 3$，它指向节点 $u = \{3, 5\}$，$u$ 的最长子串为 "bc"，$\text{len}(u) = 2$，"cbc" 和 "bc" 是两个连续的子串。

>[!hint] 为什么 len 和 minlen 有这样的关系？
>这是因为，状态 $v$ 覆盖了长度区间 $[\text{minlen}(v), \text{len}(v)]$，状态 $u$ 覆盖了长度区间 $[\text{minlen}(u), \text{len}(u)]$。因为 $u$ 是 $v$ 的紧邻后缀节点，所以 $u$ 中**最长**的子串，长度正好比 $v$ 中**最短**的子串小 1。如果不然，考虑 $v$ 中最短子串 $t$ 踢掉最前面字符得到的 $t'$ 所在的等价类，那么这个一定是满足 $\endpos(t')\supsetneqq\endpos(t)$，然后一定又满足 $\endpos(t')\subset u$，这表明必然有 $\endpos(t')=u$，也就是 $t'$ 就在 $u$ 涵盖的子串之中。

后缀链的作用是把两个不同节点（等价类、状态）的连续子串连接起来。如图所示，$v$ 的最短子串的长度等于 $u$ 的最长子串长度加 1，它们是相邻的两个后缀。也就是说，后缀链相当于在找，**当前后缀中，最长的出现次数更多的真后缀。**

![[Pasted image 20260315153915.png]]

每个节点只有一个 link，沿着后缀链向上走，子串长度一直减小，最后一定到达根。反过来，一条从根出发到某个节点的后缀链，表达了一个完整的后缀组合，这就是母树的本质。换言之，从根到节点 $v$ 的整条路径，恰好表达了 $v$ 中最长子串的所有后缀。

> 例如：如果 $v$ 中最长子串是 `abcde`，那么这条链上的所有节点加起来，包含了 `{abcde, bcde, cde, de, e, ""}` 这所有的后缀，一个不落。

>[!hint] 为什么我们需要后缀链？
>在构造过程中，每次加入一个新字符 `c`，我们遇到的核心问题是：  
>
>**新来的这个字符 `c`，到底应该和哪些旧的后缀组合起来？**
>
>为了找到答案，我们需要遍历当前字符串 `s` 的所有后缀。如果不用后缀链接，我们就得从很长的状态开始，一个一个字符地往回退，复杂度会很高。
>
>后缀链接正好解决了这个问题：**它让我们能在不同长度的后缀之间瞬间跳跃，而不用沿着字符路径一步步回溯。**
>
>换句话说，后缀链接通过连接不同的等价类，让我们能够快速找到上一个字符串后缀的所有不同的等价类（特别是最长的和上一个等价类不同的等价类），然后在回退等价类的基础上判断：往里面塞一个还是说我们得把这个等价类拆开（因为这个等价类可能拥有其他的不满足我现在前面这个子串的后缀）。
>
>其实，这和 ACAM 中的 fail 指针很像。可以类似理解一下。

>[!remark]- 注记
>加以简单的观察，不难发现，字符串 $S$ 的 SAM 的 Parent Tree，本质上就是 $S$ 的反向字符串 $S^{\rm rev}$ 的压缩后缀树。所谓「压缩后缀树」，就是指，在后缀树的基础上把只有单分支的路径压缩起来。

那么接下来我们就可以开始建 SAM 了。这个算法是 **在线** 算法，我们可以逐个加入字符串中的每个字符，并且在每一步中对应地维护 SAM。也就是说，我们完全可以实现在线维护添加字符的操作。当然，离线删除也能做到，因为时光倒流就是添加了。

采用递增法建 SAM，从 $S$ 的第 $1$ 个字符开始建只有一个节点的图，然后逐步添加字符，增加 SAM 的节点，直到所有字符都加入 SAM。建 SAM 的关键如下。
1. 起点和终点之间的边代表在当前字符串后增加一个字符。
2. 从根到达图中任意点的路径形成的子串是 $S$ 中的一个子串。
3. **保证每个点的所有子串属于同一个 $\text{endpos}$ 等价类。** 因此下面理解的过程，可以直接把每个节点等同于等价类。
4. 点之间需要符合母树的父子关系。到达点 $i$ 的所有字符串的长度都必须大于到达 $\text{link}(i)$ 的所有字符串的长度，且到达 $\text{link}(i)$ 的任意子串必为到达 $i$ 的任意子串的后缀。

更加完整的流程叙述是：
- 令 $\text{last}$ 为添加字符 $c$ 之前，整个字符串对应的状态（一开始我们设 $\text{last} = 0$，算法的最后一步更新 $\text{last}$）。
- 创建一个新的状态 $\text{cur}$，并将 $\text{len}(\text{cur})$ 赋值为 $\text{len}(\text{last}) + 1$，在这时 $\text{link}(\text{cur})$ 的值还未知。
- 现在我们按以下流程进行（从状态 $\text{last}$ 开始）。
	- 对于每一个节点，如果还没有到字符 $c$ 的转移，我们就添加一个到状态 $\text{cur}$ 的转移，继续沿后缀链接往上跳。
	- 如果在某个点已经存在到字符 $c$ 的转移，我们就停下来，并将这个状态标记为 $p$。
	- 如果没有找到这样的状态 $p$，我们就到达了虚拟状态 $-1$，我们将 $\text{link}(\text{cur})$ 赋值为 $0$ 并退出。
	- 假设现在我们找到了一个状态 $p$，其可以通过字符 $c$ 转移。我们将转移到的状态标记为 $q$。
	- 现在我们分类讨论两种状态，要么 $\text{len}(p) + 1 = \text{len}(q)$，要么不是。
		- 如果 $\text{len}(p) + 1 = \text{len}(q)$，我们只要将 $\text{link}(\text{cur})$ 赋值为 $q$ 并退出。
		- 否则就会有些复杂。需要 **复制** 状态 $q$：我们创建一个新的状态 $\text{clone}$，复制 $q$ 的除了 $\text{len}$ 的值以外的所有信息（后缀链接和转移）。我们将 $\text{len}(\text{clone})$ 赋值为 $\text{len}(p) + 1$。复制之后，我们将后缀链接从 $\text{cur}$ 指向 $\text{clone}$，也从 $q$ 指向 $\text{clone}$。最终我们需要使用后缀链接从状态 $p$ 往回走，只要存在一条通过 $p$ 到状态 $q$ 的转移，就将该转移重定向到状态 $\text{clone}$。
- 以上三种情况，在完成这个过程之后，我们将 $\text{last}$ 的值更新为状态 $\text{cur}$。
- 如果我们还想知道哪些状态是 **终止状态** 而哪些不是，我们可以在为字符串 $s$ 构造完完整的 SAM 后找到所有的终止状态。
	- 为此，我们从对应整个字符串的状态（存储在变量 last 中），遍历它的后缀链接，直到到达初始状态。我们将所有遍历到的节点都标记为终止节点。容易理解这样做我们会准确地标记字符串 s 的所有后缀，这些状态都是终止状态。

我们现在说明这个流程的正确性。

我们先给出一个定义。若一个转移 $(p, q)$ 满足 $\text{len}(p) + 1 = \text{len}(q)$，则我们称这个转移是 **连续的**。否则，即当 $\text{len}(p) + 1 < \text{len}(q)$ 时，这个转移被称为 **不连续的**。

从算法描述中可以看出，连续的、不连续的转移是算法的不同情况。连续的转移是固定的，我们不会再改变了。与此相反，当向字符串中插入一个新的字符时，不连续的转移可能会改变（转移边的端点可能会改变）。

为了避免引起歧义，我们记向 SAM 中插入当前字符 $c$ 之前的字符串为 $s$。

算法从创建一个新状态 $\text{cur}$ 开始，对应于整个字符串 $s + c$。我们创建一个新的节点的原因很清楚。与此同时我们也创建了一个新的字符和一个新的等价类。

在创建一个新的状态之后，我们会从对应整个字符串 $s$ 的状态通过后缀链接进行遍历。假定现在已经建立的 $s$ 的 SAM 是正确的。现在，对于每一个状态，我们尝试添加一个通过字符 $c$ 到新状态 $\text{cur}$ 的转移。然而我们只能添加与原有转移不冲突的转移。因此我们只要找到已存在的 $c$ 的转移，我们就必须停止。
- 最简单的情况是我们到达了虚拟状态 $-1$，这意味着我们为所有 $s$ 的后缀添加了 $c$ 的转移。这也意味着，字符 $c$ 从未在字符串 $s$ 中出现过。因此 $\text{cur}$ 的后缀链接为状态 $0$。
- 第二种情况下，我们找到了现有的转移 $(p, q)$。这意味着我们尝试向自动机内添加一个 **已经存在的** 字符串 $x + c$（其中 $x$ 为 $s$ 的一个后缀，且字符串 $x + c$ 已经作为 $s$ 的一个子串出现了）。
	- 因为我们假设字符串 $s$ 的自动机的构造是正确的，我们不应该在这里添加一个新的转移。
	- 现在问题转化为从状态 $\text{cur}$ 出发的后缀链接应该连接到哪个状态呢？我们要把后缀链接连到一个状态上，且其中 **最长** 的一个字符串恰好是 $x + c$，即这个状态的 $\text{len}$ 应该是 $\text{len}(p) + 1$。然而如果还不存在这样的状态，即 $\text{len}(q) > \text{len}(p) + 1$。这种情况下，我们必须通过拆开状态 $q$ 来创建一个这样的状态。
	- 如果转移 $(p, q)$ 是连续的，那么 $\text{len}(q) = \text{len}(p) + 1$。在这种情况下一切都很简单。我们只需要将 $\text{cur}$ 的后缀链接指向状态 $q$。
		- 为什么？回顾后缀链接的定义。我们期望寻找一个最长长度恰好是最短长度 - 1的上层节点。换句话说，我们现在在找 $s+c$ 的最长真后缀，并且满足 $\endpos$ 的元素比 $s+c$ 多（反之则合并成一个状态了）。
		- 现在我们有一个 $p\xrightarrow{c}q$ 的边了。再次回顾后缀链接的定义，这时候 $p$ 是最长的真后缀，并且 $p$ 是满足 $\endpos$ 比 $s$ 多的，如果 $s+c$ 的后缀链接是另外一个，那么我们删掉 $c$，这时候 $p'$ 是一个不同于 $p$ 的 $s$ 的子串，并且 $p'$ 的 endpos 也是最长的。既然前面的构造都是正确的，那么这只能说明 $p'=p$。
		- 从另一个角度想，我们跳到的第一个是 $p\xrightarrow{c}q$ 的边，也就是第一个存在的 $c$ 转移，换句话说，$p$ 就是 $s$ 的最长真后缀，使得 $p+c$ 是 $s+c$ 的最长真后缀，并且满足 $p+c$ 比 $s+c$ 有更多的出现位置（在 $s$ 中出现过）。因此，$q$ 就是 $s+c$ 的后缀链接（因为 $\text{len}$ 只变化 1，说明 $q$ 中最长的字符串就是 $p+c$ 了）
	- 否则转移是不连续的（我们下面会解释这种状态出现的原因），即 $\text{len}(q) > \text{len}(p) + 1$，这意味着状态 $q$ 不只对应于长度为 $\text{len}(p) + 1$ 的后缀 $s + c$，还对应于 $s$ 的更长的子串。这时候，除了将状态 $q$ 拆成两个子状态以外我们别无他法，所以第一个子状态的长度就是 $\text{len}(p) + 1$ 了。也就是说，我们下面所有的操作，目的都是拎出来所有 $p+c$ 的子串，形成一个新状态 clone。
		- 我们如何拆开一个状态呢？我们 **复制** 状态 $q$，产生一个状态 $\text{clone}$，我们将 $\text{len}(\text{clone})$ 赋值为 $\text{len}(p) + 1$。
		- 我们现在明晰一下状态。现在 clone 代表了所有 $p+c$ 的子串。$q$ 则代表比 $p+c$ 更长的子串。
		- 由于我们不想改变遍历到 $q$ 的路径，我们将 $q$ 的所有转移复制到 $\text{clone}$。
			- 这是很自然的，因为从 $q$ 能走到的子串，从 $\text{clone}$ 一样能走到。
		- 我们也将从 $\text{clone}$ 出发的后缀链接设置为 $q$ 的后缀链接的目标，并设置 $q$ 的后缀链接为 $\text{clone}$。
			- 这也是很自然的，因为我们只有一次拆分，这说明 $\text{clone}$ 里面的最长子串一定是第一个满足比 $q$ 出现位置多的，那么出现次数比 $\text{clone}$ 还多的自然就是 $q$ 的后缀链接了。
		- 在拆开状态后，我们将从 $\text{cur}$ 出发的后缀链接设置为 $\text{clone}$。
			- 这也是很自然的。和前一个情况的原因完全类似。
		- 最后一步我们将一些到 $q$ 转移重定向到 $\text{clone}$。我们需要修改哪些转移呢？只重定向相当于所有字符串 $w + c$（其中 $w$ 是 $p$ 的最长字符串）的后缀就够了。
			- 换句话说，我们要把从所有「比 $p$ 更短但仍能使 $x+c$ 出现在 $q$ 里的 $p$ 的祖先」出发的、字符为 $c$ 的、目标为 $q$ 的转移，都改为目标为 clone。
			- 更形象地来看，我们还是想要拎出来所有 $p+c$ 的子串，然后指向 clone。
			- 为了实现这个，我们需要继续沿着后缀链接遍历，从结点 $p$ 直到虚拟状态 $-1$ 或者是转移到不是状态 $q$ 的一个转移。
				- 这是因为如果某个状态 $p'$ 的 $c$ 转移指向的不是 $q$，那么比 $p'$ 更短的那些后缀，它们的最长字符串长度更短，此时必然是通过其他路径到别的状态。
				- 我们这里关心的是 **所有能使 $\boldsymbol{x+c}$ 以及更短的那些后缀通过 $\boldsymbol c$ 转移到 $\boldsymbol q$ 的状态**——这些状态的 $c$ 转移目标必须是 $q$。一旦某个状态的 $c$ 转移指向别处，说明 $x+c$ 已经不经过 $q$ 了，它的更短的后缀自然也不会经过 $q$，所以后面的不用再改。

所以，SAM 这样的构造是正确的。我们下面用更加具体的例子来说明。

 - 然后发现 0 已经是虚拟节点了，我们就退出循环，设置 link(3)=0，len(3)=3。

简单总结一下，我们把上面的三步重新用通俗的语言来描述一遍。
1. **第一步：铺设新轨道**
    - 我们创建新状态 `cur`，对应整个新串 `s + c`。
    - 我们从 `last`（对应 `s` 的状态）出发，沿着**后缀链接**一路向上走。每走到一个状态 `v`，就在它和 `cur` 之间铺设一条标有 `c` 的转移边。
    - **作用**：这相当于告诉系统：所有以 `s` 的某个后缀为前缀，再加上 `c` 形成的字符串，我都承认你是 `cur` 这个新站点的乘客。他们都被迫只能够在 `c` 这里出现一次（因为他们本身并不存在一个通过 `c` 连接的别的节点，所以他们往后面塞一个 `c` 只能在 `s + c` 这里出现），因此都属于 `cur` 所代表的等价类。
2. **第二步：遇到老乘客（连续情况）**
    - 走着走着，我们发现某个状态 `p` 已经有一条 `c` 转移边指向 `q` 了。
    - 这意味着 `longest(p) + c` 这个字符串**已经存在**，它不是新面孔。
    - 如果 `len(q) == len(p) + 1`（连续转移），说明 `q` 这个站里装的全是 `longest(p) + c` 及其后缀，恰好就是我们要找的 `cur` 的最长的**等价类不同的**后缀。（如果存在一个更长的，那么我们之前应该已经遇到了）
    - **作用**：我们直接把 `cur` 的**后缀链接**指向 `q`。这条链告诉系统："我的最长等价类不同的后缀，要去 `q` 站找。"
3. **第三步：拆站点的复杂情况（不连续转移）**
    - 这是最精妙的地方。如果 `len(q) > len(p) + 1`，说明 `q` 这个站里混进了两种乘客：一种是比较短的（`longest(p) + c`），另一种是比较长的（来自其他路径）。
    - 这两种乘客其实**不该在同一个站**，因为他们的 endpos 不同（短的出现在更多地方，比如这里的 `c`）。
    - **后缀链接的作用**：
        - **克隆**：我们新建一个站 `clone`，把 `q` 站的设施（转移边）复制一份，专门接待 `longest(p) + c` 这种短乘客。
        - **重新接线**：把 `q` 站的**后缀链接**指向 `clone`（因为 clone 在这里是 q 的最长的等价类不同的真后缀），把 `clone` 站的**后缀链接**指向原来 `q` 的后缀链接（同理）。
        - **重定向轨道**：再次沿着**后缀链接**从 `p` 往回走，把所有本来要通向 `q` 的 `c` 转移边，都改到 `clone`。
    - 这一通操作下来，**后缀链接**始终保证了：任何一个状态，顺着它的后缀链接走一步，就能找到包含它最长真后缀的等价类。

最后我们来看时间空间复杂度。我们注意到每个节点至多复制一次，而 $n$ 个字符对应 $n$ 个节点，因此空间复杂度最多 $2n$。我们只需要把空间定义为 `MAXN<<1` 就好了。时间复杂度是线性的，但是证明较为复杂，留在下面选看了。

>[!hint]- 关于 SAM 操作次数必然是线性的证明
>首先我们假设字符集大小为 **常数** 。如果字符集大小不是常数，SAM 的时间复杂度就不是线性的。从一个结点出发的转移存储在支持快速查询和插入的平衡树中。因此如果我们记 $\Sigma$ 为字符集， $|\Sigma|$ 为字符集大小，则算法的渐进时间复杂度为 $O(n\log|\Sigma|)$ ，空间复杂度为 $O(n)$ 。然而如果字符集足够小，可以不写平衡树，以空间换时间将每个结点的转移存储为长度为 $|\Sigma|$ 的数组（用于快速查询）和链表（用于快速遍历所有可用关键字）。这样算法的时间复杂度为 $O(n)$ ，空间复杂度为 $O(n|\Sigma|)$ 。
>
>如果我们考虑算法的各个部分，算法中有三处时间复杂度不明显是线性的：
>
>- 第一处是遍历所有状态 last 的后缀链接，添加字符 c 的转移。
>- 第二处是当状态 q 被复制到一个新的状态 clone 时复制转移的过程。
>- 第三处是修改指向 q 的转移，将它们重定向到 clone 的过程。
>
>我们使用 SAM 的大小（状态数和转移数）为线性的的事实，这是因为每一个转移等价于一个母树上的边，因此不超过 $2n$ 次。
>
>因此上述 **第一处和第二处** 的总复杂度显然为线性的，因为单次操作均摊只为自动机添加了一个新转移。
>
>还需为 **第三处** 估计总复杂度，我们将最初指向 $q$ 的转移重定向到 $\text{clone}$。我们记 $v = \text{longest}(p)$，这是一个字符串 $s$ 的后缀，每次迭代长度都递减——因为字符串 $s$ 的位置每次迭代都单调上升。这种情况下，如果在循环的第一次迭代之前，相对应的字符串 $v$ 在距离 $\text{last}$ 的深度为 $k$ （$k \geq 2$） 的位置上（深度记为后缀链接的数量），那么在最后一次迭代后，字符串 $v + c$ 将会成为路径上第二个从 $\text{cur}$ 出发的后缀链接（它将会成为新的 $\text{last}$ 的值）。
>
>因此，循环中的每次迭代都会使作为当前字符串的后缀的字符串 $\text{longest}(\text{link}(\text{link}(\text{last})))$ 的位置单调递增。因此这个循环最多不会执行超过 $n$ 次迭代，这正是我们需要证明的。

我们通过这道题目给出 SAM 的代码。

>[!example] HDU 4622
>给定一个仅由小写英文字母组成的字符串 $s$，定义 $f(s)$ 为 $s$ 中不同子串的数量。  
>
>你需要处理 $q$ 次查询，每次查询需要计算 $f(s[l...r])$，其中 $s[l...r]$ 表示从第 $l$ 个字符开始到第 $r$ 个字符结束的子串。
>
>数据范围：$|s|\leq 2000, q\leq 10000$。

我们期待能够预处理所有区间的答案。那么怎么做呢？肯定要思考怎么 dp，从 $[L..R-1]$ 推到 $[L..R]$。思考：在一个字符串后面加一个字符，那么子串数量会增加多少？

在 SAM 中，每次加入一个新字符，只会新增一个状态（节点）。那么这个会多的子串个数当然就是 $\text{len}(\text{cur})$ 了。那有一些之前有的，那就是 $\text{len}(\text{link}(\text{cur}))$ 了。所以做个差就是新的子串个数。

于是，我们每一次都暴力求一遍 $[l,r]$ 的 SAM，就可以 $\displaystyle O\left(\sum_{l=1}^n\sum_{r=l}^n |r-l|\right)=O(n^2)$ 预处理所有的答案了。最后时间复杂度是 $O(n^2+q)$ 的。

一个实现代码如下。

```cpp
#include <bits/stdc++.h>

using namespace std;

#define all(x) begin(x), end(x)
#define sz(x) (int) (x).size()

using i64 = long long;
using u64 = unsigned long long;
using i128 = __int128_t;
using u128 = __uint128_t;
using pii = pair<int, int>;
using vi = vector<int>;
using usi = unordered_set<int>;

class SAM {
    struct Node {
        int son[26]{};
        int link = -1;
        int len = 0;
        Node() = default;
        Node(int len) : len(len) {}
    };
    vector<Node> t;
    int cnt = 0;
    int n, lst = 0;

  public:
    SAM() {}
    SAM(int n) : t(n << 1), n(n) { t[0] = Node(0); }
    void insert(int ch) {
        t[++cnt] = Node(t[lst].len + 1);
        int p = lst, cur = cnt;
        while (p != -1 && !t[p].son[ch]) {
            t[p].son[ch] = cur;
            p = t[p].link;
        }
        if (p == -1) {
            t[cur].link = 0;
        } else {
            int q = t[p].son[ch];
            if (t[q].len == t[p].len + 1) {
                t[cur].link = q;
            } else {
                t[++cnt] = Node(t[p].len + 1);
                memcpy(t[cnt].son, t[q].son, sizeof(t[q].son));
                t[cnt].link = t[q].link;
                t[cur].link = t[q].link = cnt;
                while (p != -1 && t[p].son[ch] == q) {
                    t[p].son[ch] = cnt;
                    p = t[p].link;
                }
            }
        }
        lst = cur;
    }
    int get() { return t[lst].len - t[t[lst].link].len; }
};

int ans[2005][2005]{};

void solve() {
    string s;
    cin >> s;
    int n = s.size();
    for (int i = 0; i < n; i++) {
        SAM sam(n - i);
        for (int j = i; j < n; j++) {
            sam.insert(s[j] - 'a');
            if (j)
                ans[i][j] = ans[i][j - 1] + sam.get();
            else
                ans[i][j] = sam.get();
        }
    }
    int q;
    cin >> q;
    while (q--) {
        int l, r;
        cin >> l >> r;
        cout << ans[l - 1][r - 1] << '\n';
    }
}

int main() {
    cin.tie(nullptr)->sync_with_stdio(false);
    int t;
    cin >> t;
    while (t--) { solve(); }
    return 0;
}
```
### 应用场景

后缀自动机几乎是「全能」的处理字符串算法，模式匹配、回文串等问题都可以用后缀自动机实现，时间复杂度也一样好。

我们来看一些常见的应用场景。

简单起见，假设字符集的大小 $|\Sigma|$ 为常数。这允许我们认为增加一个字符和遍历的复杂度为常数。
#### 检查字符串是否出现

> 给一个文本串 $T$ 和多个模式串 $P$ ，我们要检查字符串 $P$ 是否作为 $T$ 的一个子串出现。

我们在 $O(|T|)$ 的时间内对文本串 $T$ 构造后缀自动机。为了检查模式串 $P$ 是否在 $T$ 中出现，我们沿转移（边）从 $t_0$​ 开始根据 $P$ 的字符进行转移。如果在某个点无法转移下去，则模式串 $P$ 不是 $T$ 的一个子串。如果我们能够这样处理完整个字符串 $P$，那么模式串在 $T$ 中出现过。

对于每个字符串 $P$，算法的时间复杂度为 $O(|P|)$ 。此外，这个算法还找到了模式串 $P$ 在文本串中出现的最大前缀长度。因此总时间复杂度为 $O(|T|+\sum |P|)$。
#### 不同子串个数

> 给一个字符串 $S$ ，计算不同子串的个数。

对字符串 $S$ 构造后缀自动机。

每个 $S$ 的子串都相当于自动机中的一些路径。因此不同子串的个数等于自动机中以 $t_0$​ 为起点的不同路径的条数。考虑到 SAM 为有向无环图，不同路径的条数可以通过动态规划计算。记 $dp_u$ 表示以 $u$ 开始的子串个数，转移是显然的。这样不会重复计算，因为最后我们要统计的事 $dp_{t_0}$，相当于从 $t_0$ 出发的。时间复杂度是 $O(|S|)$。
#### 所有不同子串的总长度

> 给定一个字符串 $S$ ，计算所有不同子串的总长度。

本题做法与上一题类似，只是现在我们需要考虑分两部分进行动态规划：不同子串的数量 $d_v$​ 和它们的总长度 $\text{ans}_v$​ 。

我们已经在上一题中介绍了如何计算 $d_v$​ 。 $\text{ans}_v​$ 的值自然也是显然的了。时间复杂度依旧是 $O(|S|)$。
#### 字典序第 k 大子串

> 给定一个字符串 $S$ 。多组询问，每组询问给定一个数 $K_i$​ ，查询 $S$ 的所有子串中字典序第 $K_i$​ 大的子串。

解决这个问题的思路可以从解决前两个问题的思路发展而来。字典序第 $k$ 大的子串对应于 SAM 中字典序第 $k$ 大的路径，因此在计算每个状态的路径数后，我们可以很容易地从 SAM 的根开始找到第 $k$ 大的路径。这是因为，我们可以通过一次 DAG 上 dp 处理出经过每个结点的路径数，然后通过试填法解决。

预处理的时间复杂度为 $O(|S|)$ ，单次查询的复杂度为 $O(|ans|\cdot |\Sigma|)$，这里 $|ans|$ 表示答案的大小，换言之查询最劣复杂度是常数很小的 $O(|S||\Sigma|)$。

具体的实现代码可以看 [这里](https://www.spoj.com/status/ns=35608199)。
#### 最小循环移位

>给定一个字符串 $S$ 。找出字典序最小的循环移位。

容易发现字符串 $S+S$ 包含字符串 $S$ 的所有循环移位作为子串。所以问题简化为在 $S+S$ 对应的后缀自动机上寻找最小的长度为 $|S|$ 的路径，这可以通过平凡的方法做到：我们从初始状态开始，贪心地访问最小的字符即可。

总的时间复杂度为 $O(|S|)$ 。
#### 出现次数

> 对于一个给定的文本串 $T$ ，有多组询问，每组询问给一个模式串 $P$ ，回答模式串 $P$ 在字符串 $T$ 中作为子串出现了多少次。

对文本串 $T$ 构造后缀自动机。

接下来做预处理：对于自动机中的每个状态 $v$ ，预处理 $\text{cnt}_v$​ ，使之等于 $\endpos(v)$ 集合的大小。事实上，对应同一状态 $v$ 的所有子串在文本串 $T$ 中的出现次数相同，这相当于集合 $\endpos$ 中的位置数。然而我们不能明确的构造集合 $\endpos$ ，因此我们只考虑它们的大小 $\newcommand{\cnt}{\operatorname{cnt}}\cnt$。

为了计算这些值，我们进行以下操作。对于每个状态，如果它不是通过复制创建的（且它不是初始状态 $t_0$​ ），我们将它的 $\cnt$ 初始化为 1。然后我们按它们的长度 $\text{len}$ 降序遍历所有状态，并将当前的 $\newcommand{\link}{\operatorname{link}} \cnt_v$​ 的值加到后缀链接指向的状态上，即：
$$
\cnt(\link(v))\gets \cnt(\link(v))+\cnt(v).
$$
为什么这是正确的？不是通过复制获得的状态，恰好有 $|T|$ 个，并且它们中的前 $i$ 个在我们插入前 $i$ 个字符时产生。因此对于每个这样的状态，我们在它被处理时计算它们所对应的位置的数量。因此我们初始将这些状态的 $\cnt$ 的值赋为 1 ，其它状态的 $\cnt$ 值赋为 0 。同时这个位置也包含了所有 $\endpos$ 对应的 $\cnt$，因此完全没问题。

>[!hint] 怎么做到倒序遍历？
>因为 len 最大为 $|T|$，可以用**桶排序**的思想。

```cpp
vector<int> order; // 存储按 len 升序的节点编号
vector<int> cnt_len(maxlen+1, 0);
// 1. 统计每个长度有多少节点
for (int i = 1; i <= sz; i++)  // 假设状态编号从 1 到 sz，t0=1
    cnt_len[len[i]]++;
// 2. 前缀和，确定每个长度在 order 中的起始位置
for (int i = 1; i <= maxlen; i++)
    cnt_len[i] += cnt_len[i-1];
// 3. 倒序填充 order（稳定排序不重要，我们只要按 len 排序）
vector<int> order(sz+1);
for (int i = sz; i >= 1; i--)  // 倒序遍历原顺序
    order[cnt_len[len[i]]--] = i;
// 4. 现在 order[1..sz] 是按 len 升序的节点编号
// 要降序遍历：for (int i = sz; i >= 1; i--) 使用 order[i]
for (int i = sz; i >= 1; i--) {
    int v = order[i];
    cnt[link[v]] += cnt[v];
}
```

接下来我们对每一个 $v$ 执行操作 $\cnt(\link(v))\gets \cnt(\link(v))+\cnt(v)$。其背后的含义是，如果有一个字符串 $v$ 出现了 $\cnt_v$​ 次，那么它的所有后缀也在完全相同的地方结束，即也出现了 $\cnt_v$​ 次。

>[!hint]- 为什么我们在这个过程中不会重复计数呢？
>因为我们只将一个状态的位置添加到 **一个** 其它的状态上，所以一个状态不可能以两种不同的方式将其位置重复地指向另一个状态。

因此，我们可以在 $O(|T|)$ 的时间内计算出所有状态的 $\cnt$ 的值。

最后回答询问只需要查找值 $\cnt(t)$​，其中 $t$ 为模式串对应的状态，如果该模式串不存在答案就为 0。单次查询的时间复杂度为 $O(|P|)$。

于是可以 AC [P3804](https://www.luogu.com.cn/problem/P3804)，一个代码可以看 [这里](https://www.luogu.com.cn/record/267557616)。
#### 第一次出现的位置

>给定一个文本串 $T$ ，多组查询。每次查询字符串 $P$ 在字符串 $T$ 中第一次出现的位置（ $P$ 的开头位置）。

我们构造一个后缀自动机。我们对 SAM 中的所有状态预处理位置 $\text{firstpos}$。即，对每个状态 $v$ 我们想要找到第一次出现这个状态的**末端**的位置 $\text{firstpos}[v]$。换句话说，我们希望先找到每个集合 $\text{endpos}$ 中的最小的元素（显然我们不能显式地维护所有 $\text{endpos}$ 集合）。

为了维护 $\text{firstpos}$ 这些位置，我们将原函数扩展为 `sam_extend()`。当我们创建新状态 $\text{cur}$ 时，我们令：
$$\text{firstpos}(\text{cur}) = \text{len}(\text{cur}) - 1$$
当我们将结点 $q$ 复制到 $\text{clone}$ 时，我们令：
$$\text{firstpos}(\text{clone}) = \text{firstpos}(q)$$
（因为值的唯一的其它选项 $\text{firstpos}(\text{cur})$ 显然太大了）。

那么查询的答案就是 $\text{firstpos}(t) - |P| + 1$，其中 $t$ 为对应字符串 $P$ 的状态。单次查询只需要 $O(|P|)$ 的时间。
#### 所有出现的位置

> 问题同上，这一次需要查询文本串 $T$ 中模式串出现的所有位置。

同样，我们为文本 $T$ 构建一个后缀自动机。与上一个任务类似，我们计算所有状态的 `firstpos`。

显然，如果 $t$ 是查询串 $P$ 对应的状态，那么 `firstpos(t)` 就是答案的一部分。这样我们就考虑了自动机中包含 $P$ 的那个状态。还需要考虑哪些其他状态呢？所有那些以 $P$ 为后缀的字符串所对应的状态。换句话说，我们需要找到所有能通过后缀链接到达状态 $t$ 的状态。这是因为，反过来看，如果 $P$ 是某个更长字符串 $S'$ 的后缀，那么包含 $S'$ 的状态 $v$ 沿着后缀链接走若干步，一定能到达包含 $P$ 的状态 $t$。因此，**所有以 $P$ 为后缀的字符串** 对应的状态，就是那些能通过后缀链接 **反向** 到达 $t$ 的状态。

因此，为了解决这个问题，我们需要为每个状态保存一个指向它的后缀引用列表。那么查询的答案将包含从状态 $t$ 出发，仅使用后缀引用进行 DFS/BFS 所能找到的每个状态的 `firstpos`。

总的来说，这需要 $O(|T|)$ 的预处理时间，以及每个请求 $O(|P| + \text{answer}(P))$ 的时间，其中 $\text{answer}(P)$ 是答案的大小。

首先，我们沿着自动机根据模式串的每个字符向下走，找到起始节点，这需要 $O(|P|)$ 的时间。然后，我们使用上述方法，它将在 $O(\text{answer}(P))$ 时间内完成，因为我们不会两次访问同一个状态（因为每个状态只引出一条后缀链接，所以不可能有两条不同的路径通向同一个状态）。

我们只需要注意一点：两个不同的状态可能具有相同的 `firstpos` 值。如果某个状态是通过克隆另一个状态得到的，就会发生这种情况。然而，这不会破坏复杂度，因为每个状态最多只能有一个克隆。

此外，如果我们不输出克隆状态的 `firstpos`，还可以**去除重复的位置**。事实上，克隆状态能到达的任何状态，原始状态也能到达。因此，如果我们为每个状态记住一个 `is_cloned` 标志，就可以简单地忽略克隆状态，只输出所有其他状态的 `firstpos`。

下面就是一个用 dfs 实现的伪代码。

```cpp
struct state {
    ...
    bool is_clone;
    int first_pos;
    vector<int> inv_link;
};

// after constructing the automaton
for (int v = 1; v < sz; v++) {
    st[st[v].link].inv_link.push_back(v);
}

// output all positions of occurrences
void output_all_occurrences(int v, int P_length) {
    if (!st[v].is_clone)
        cout << st[v].first_pos - P_length + 1 << endl;
    for (int u : st[v].inv_link)
        output_all_occurrences(u, P_length);
}
```
#### 最短未出现字符串

> 给定一个字符串 $S$ 和一个特定的字母表。我们需要找到一个长度最小的、没有在 $S$ 中出现的字符串。
##### 法一
我们将在为字符串 $S$ 构建的后缀自动机上应用 DAG 动态规划。

令 $d[v]$ 表示节点 $v$ 的答案，即我们已经处理了子串的一部分，当前处于状态 $v$，并且想要找到需要添加的最少字符数，以便找到一个不存在的转移。计算 $d[v]$ 非常简单。如果不存在使用字母表中至少一个字符的转移，那么 $d[v] = 1$。否则，一个字符是不够的，因此我们需要取所有转移的答案的最小值：
$$d[v] = 1 + \min_{w: (v,w,c) \in SA} d[w].$$
问题的答案将是 $d[t_0]$，并且可以使用计算出的 $d[]$ 数组来恢复实际的字符串，我们每一步走向差为 1 的节点，然后最后输出未出现的，这样时间复杂度是 $O(|\text{Answer}|\cdot |\Sigma|)$。
##### 法二
我们考虑在 SAM 上直接 bfs。第一个扫到的没出现的就是我们要的答案。时间复杂度也是 $O(|\text{Ans}||\Sigma|)$。
#### 两个字符串的最长公共子串

> 给定两个字符串 $S$ 和 $T$。我们需要找到最长的公共子串，即一个字符串 $X$，它同时作为 $S$ 的子串和 $T$ 的子串出现。

我们为字符串 $S$ 构建一个后缀自动机。

现在我们将处理字符串 $T$，并针对每个前缀，寻找这个前缀在 $S$ 中的最长后缀。换句话说，对于字符串 $T$ 中的每个位置，我们希望找到 $S$ 和 $T$ 的、以该位置结尾的最长公共子串。换言之，我们遍历另一个字符串 $T$ 的每个字符，模拟在 SAM 上运行 $T$ 的过程，并 **动态维护当前匹配的长度**。这个长度就是以当前 $T$ 的字符结尾的、同时也是 $S$ 的子串的最长长度。阅读下面的过程并和 ACAM / KMP 类比可以发现两者的相似之处。

为此，我们将使用两个变量：**当前状态** $v$ 和 **当前答案** $l$。这两个变量将描述当前匹配的部分：它的长度以及与之对应的状态。初始时 $v = t_0$ 且 $l = 0$，即匹配为空。现在让我们描述如何添加一个字符 $T[i]$ 并为其重新计算答案。
- 如果从状态 $v$ 存在一条以字符 $T[i]$ 为标记的转移，那么我们只需沿着该转移，并将 $l$ 增加 1。
- 如果不存在这样的转移，我们必须缩短当前匹配的部分，这意味着我们需要沿着后缀链接走：$v = \text{link}(v)$。同时，当前长度也必须缩短。显然，我们需要设 $l = \text{len}(v)$，因为经过后缀链接后，我们最终到达的状态所对应的最长字符串是一个子串。
- 如果仍然没有使用所需字符的转移，我们重复此过程，再次走后缀链接并减小 $l$，直到我们找到一个转移，或者到达虚拟状态 $-1$（这意味着字符 $T[i]$ 根本没有在 $S$ 中出现，因此我们设 $v = l = 0$）。
不难发现，我们上面的每个转移中，$v$ 永远代表着最长能够和当前 $T$ 匹配的子串。

任务的答案将是所有 $l$ 值中的最大值。这一部分的复杂度是 $O(|T|)$，因为每一步中，我们要么将 $l$ 增加 1，要么多次经过后缀链接，而每一次经过后缀链接最终都会导致 $l$ 值的减少。

一个实现代码如下：
```cpp
string lcs (string S, string T) {
    sa_init();
    for (int i = 0; i < S.size(); i++)
        sa_extend(S[i]);

    int v = 0, l = 0, best = 0, bestpos = 0;
    for (int i = 0; i < T.size(); i++) {
        while (v && !st[v].next.count(T[i])) {
            v = st[v].link ;
            l = st[v].len;
        }
        if (st[v].next.count(T[i])) {
            v = st [v].next[T[i]];
            l++;
        }
        if (l > best) {
            best = l;
            bestpos = i;
        }
    }
    return T.substr(bestpos - best + 1, best);
}
```
#### 多个字符串的最长公共子串

> 给定 $k$ 个字符串 $S_i$。我们需要找到最长的公共子串，即一个字符串 $X$，它作为子串出现在每一个字符串 $S_i$ 中。

我们将所有字符串连接成一个大的字符串 $T$，用特殊字符 $D_i$（每个字符串对应一个）分隔开：
$$T = S_1 + D_1 + S_2 + D_2 + \cdots + S_k + D_k.$$
然后我们为字符串 $T$ 构建后缀自动机。

现在我们需要在自动机中找到一个包含在所有字符串 $S_i$ 中的字符串，这可以通过使用添加的特殊字符来实现。注意，如果一个子串包含在某个字符串 $S_j$ 中，那么在后缀自动机中存在一条从该子串出发的路径，该路径包含字符 $D_j$ 并且不包含其他字符 $D_1, \ldots, D_{j-1}, D_{j+1}, \ldots, D_k$。

因此，我们需要计算可达性，即对于自动机的每个状态和每个符号 $D_i$，判断是否存在这样一条路径。这可以通过 DFS 或 BFS 以及动态规划轻松计算。（本质上这也是一个 DAG 上的 dp，如果一个状态 $v$ 本身有通过字符 $D_j$ 的转移，那么 `reachable[v][j] = True`。如果状态 $u$ 有某个字符 $c$（不是分隔符）的转移指向状态 $v$，并且 `reachable[v][j] = True`，那么 `reachable[u][j] = True`。）当我们为每个状态 $v$ 计算出它对于每个分隔符 $D_i$ 的可达性后，如果一个状态 $v$ 对于 **所有** $i$ 都有 `reachable[v][i] = True`，那就说明：从 $v$ 代表的等价类出发，可以找到路径 **同时经过所有分隔符**，或者说，$v$ 中的子串出现在每一个 $S_i$ 中。之后，问题的答案将是状态 $v$ 的字符串 $\text{longest}(v)$，其中从状态 $v$ 出发存在通向所有特殊字符的路径。

>[!example] AWC047E
> https://atcoder.jp/contests/awc0047/tasks/awc0047_e
> 
> 给定无限重复字符串 $S^\infty$，对每个询问 $[L_i, R_i]$，判断子串 $T_i = S^\infty[L_i..R_i]$ 是否等于 $S$ 的某个长度相同的连续子串。若子串长度大于 $N$ 则直接回答 No。
> 
> $|S|\leq 5\times 10^5,q\leq 2\times 10^5$。

这题有很多做法。一种是 [[4. Manacher 算法与 Z 算法（扩展 KMP）]]，另外一种就是 SAM。SAM 的思维门槛相对较低，而且复杂度更低，所以在这里讲一下。

注意到 $S^\infty$ 退化成 $SS$ 即可完成所有的匹配，所以我们思考预处理一下每个回答的查询。注意到我们可以预处理出来 **所有位置结束的最长匹配**，一个注意点是中途的 dp 必须得额外维护当前匹配的长度，不然会大于实际值。

> 当你顺着转移边 `son[ch]` 走了一步时，你的**实际匹配长度只是在原来的基础上增加了 1**。状态 `cur` 所代表的最长字符串可能比你当前实际匹配的字符串要长得多，这就导致 `dp[i]` 被错误地赋了一个偏大的值

这就完成了这道题。时间复杂度是 $O(n+q)$。

这题给出的一个启发是，通常来讲维护字符串信息在 SAM 上跑 dp 是一个非常常见，而且非常有效的操作。而且字符串 dp，通常考虑到某位置结束的 dp 是更加优秀的。

当然，这道题通过 SA 也可以解决。

使用后缀数组解决这道题，是非常经典的 **“SA + LCP + ST 表 + 二分”** 套路。整体思路是把问题转化为“在字典序相近的后缀中寻找原串位置”。

1. **构造字符串：** 同样令文本串为 $S \cdot S$（长度为 $2N$），并对其构造 Suffix Array (`sa` 数组)、Rank 数组 (`rk` 数组) 和 Height 数组 (`ht` 数组)。
2. **预处理 ST 表：**
    - 建立一个 ST 表用于维护 `ht` 数组的**区间最小值**（用于快速查询任意两个后缀的 LCP）。
    - 再建立一个 ST 表用于维护 `sa` 数组的**区间最小值**（用于快速查询某段排名区间内，后缀的最小起始位置）。
3. **在线处理查询：**
    - 对于每次查询，给定起始位置 $l$ 和长度 $len$。
    - 通过 `rk[l]` 找到该后缀在 SA 中的排名。
    - **二分边界：** 以 `rk[l]` 为中心，在 LCP ST 表的辅助下，分别向左和向右二分，找到一个最大的排名区间 $[L, R]$，使得这个区间内所有的后缀与后缀 $l$ 的 LCP 长度都 $\ge len$。
    - **判断存在性：** 这个区间 $[L, R]$ 包含了所有“以查询串为前缀”的后缀。我们只需要在这个区间里，用 SA 的 ST 表查询一下最小的 `sa` 值（即最早出现的位置）。如果这个最小位置 $\le N - len$，说明这个子串在原串 $S$ 中完整出现过，输出 `Yes`；否则输出 `No`。
