#KMP

KMP 全称 Knuth-Morris-Pratt 算法。
# 应用场景

快速查找一个字符串。

# 算法内容

**核心：** $next$ 数组。

现有 $s_1$ 和 $s_2$ 两个字符串（0-indexed）。 假设我们要在 $s_1$ 中寻找 $s_2$，我们定义 $next$ 数组的第 $i$ 项为 **$s_2[0..i-1]$ 的前缀后缀（不含全字符串）的最大匹配长度**。

打个比方。设 $s_2$ 为 `aabaabsaabaaa`，规定 $next_0=-1$（因为其前面的字符串为空）。

- $next_1 = 0$，因为 $s[0..0]$ 为 `"a"`，所以 $next_1=0$，因为有且仅有空前后缀可以匹配。
- $next_2 = 1$，因为 $s[0..1]$ 为 `"aa"`，所以 $next_2=1$，因为 `"a"` 和 `"a"` 可以匹配。
- $next_3=0$，类似推理。

我们 $next$ 数组的项数是 $s_2$ 的长度加一。

在讲述如何快速求出 $next$ 数组之前，我们需要理解，为什么 $next$ 数组可以加速字符串匹配呢？

打个比方，$s_1$ 为 `aabaabcaabaaba...`，$s_2$ 为 `aabaabcaabaabt`。

首先计算出 $next$ 数组为 `[-1,0,1,0,1,2,3,0,1,2,3,4,5,6]`。

回顾暴力算法，我们对每个位置都进行匹配。但是拥有 $next$ 数组之后，我们可以加速。

注意到我们第一次匹配出现在 `a!=t`，这时候我们去查询 `t` 位置的 $next$ 数组的值。得到 $next_{13}=6$，所以，我们用 $s_2[6]$ 去和 $s_1[13]$ 配对。换言之，$s_2[6]$ 跳到了 $s_1[13]$ 的位子。

 一方面，$s_2[0..5]$ 不用配对，是因为我们 $next$ 数组的定义说明了前缀和后缀相等，而且在 $13$ 位才开始初次不匹配，这说明 $s_2[0..5]$ 和 $s_1[7..12]$ 一定是相等的，我们直接跳到 $s_2[6]$ 即可。

抽象来讲，上面的过程，就是下面这张图。

从 $s_1[i..j]$ 匹配 $s_2[0..k]$，而（设）$next[k]=7$，我们直接平移到这里。凭什么能够断言 $s_1[i..p]$ 不能匹配出一个 $s_2$？

考虑反证法。若存在某个开头可以配出 $s_2$，那么从这个位置出发到 $j$ 一定可以和 $s_2$ 对应等长的字符串长度相等！
![[Pasted image 20251116211902.png]]

这也就说明，$next$ 数组的值肯定大于 7（我们设的，可以为其他的数）。这与我们 $next$ 数组的定义矛盾！

于是假设我们已经有了 $next$ 数组，我们立刻可以写出下面的代码。

```cpp
int kmp(string s1, string s2) {
	int n=s1.size(), m=s2.size(), x=0,y=0;
	while(x<n&&y<m) {
		if(s1[x]==s2[y]) {
			x++,y++; // 匹配了，继续
		} else if(y==0) {
			x++; // 开头就没匹配，动 s1
		} else {
			y=next[y]; // 把 s2 进行移动然后继续比对。
		}
	}
	return y==m?x-y:-1;
}
```

下面的问题就是，如何快速求出 $next$ 数组？

很显然，$next_0=-1,next_1=0$。$next_2$ 也是很简单可以看出来的。这种由前面推后面，立刻想到 dp。

第一种情况：
![[Pasted image 20251116213411.png]]

我们已有 $next_{i-1}=7$。我们要计算 $next_i$。我们看 $s[7]$ 和 $s.back()$ 一不一样——一样，那就说明是 8。

>[!question]- 为什么不可能更大？
> 与前面为什么可以跳过原理类似。如果可能有更大的，说明前面的 $next$ 计算有误了，这是显然不可能的。

其他情况：
![[Pasted image 20251116213721.png]]

**可以看作反复跳回去，然后重新重复第一种情况的操作！** 以本图为例，这是因为我们 $s_{15}$ 的 $next$ 为 7，因此我们相当于看 $s[0..6]$ 中的能够和 **$s_{15}$ 及其附带的后缀**匹配的前缀——我们要找尽量长的后缀，这实质上就是重复第一种情况——直到我们找到相同的｢最后一个字符｣！

为什么长度不可能大于 $next$ 见上，已经解释了，所以我们的操作的正确性是有保证的。

因此我们只要反复利用已有的 next 数组，就可以推理出后面所有的 next 数组！

```cpp
int next[100000];
void init(string s) {
	int m=s.size();
	next[0]=-1,next[1]=0;
	int i=2, cur=0; // cur: 当前字符要比对的字符位置
	while(i<=m) {
		if(s[i-1]==s[cur]) {
			next[i++]=++cur; 
			// ++cur：一方面，我们现在要储存的值就是 ++cur，而且
			// 下一个字符比对的 next 对应的就是 ++cur
			// 也就直接和 s[cur] 比就可以了！
		} else if(cur==0) {
			next[i++]=cur;
		} else {
			cur=next[cur];
		}
	}
}
```

总体来讲，KMP 复杂度是 $O(|S_1|+|S_2|)$。

例题：[P3375 【模板】KMP](https://www.luogu.com.cn/problem/P3375)。

关于 $next$ 数组的妙用：[P9606 \[CERC2019\] ABB](https://www.luogu.com.cn/problem/P9606)、[P4391 \[BalticOI 2009\] Radio Transmission 无线传输](https://www.luogu.com.cn/problem/P4391)、超级好题 [P2375](https://www.luogu.com.cn/problem/P2375)。

更现代的写法是把 $next$ 数组变身称为前缀函数 $\pi(i)$，满足 $\pi(i)=next_{i+1}$。
