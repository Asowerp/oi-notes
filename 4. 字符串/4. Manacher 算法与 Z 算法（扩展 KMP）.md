回文问题。

# Manacher 算法讲解

下面用 $|\cdot|$ 表示长度。并且字符串都是 0 作为起始下标。

首先先给出两个定义。
 - 奇回文：长度为奇数的回文串，如 `a`，`aaa`，`abcba`；
 - 偶回文：长度为偶数的回文串，如 `aa`，`abba`。

Manacher 算法最经典的应用就是求解最长字串数组。然而 Manacher 最大的用处是理解所谓**回文半径数组**。

首先我们先看一下暴力是怎么找回文串的。很容易想到每个位置向两边扩展看最长的回文长度。——这显然会让我们错过所有的偶数长度的回文串。因此，我们需要一个转换，能够把所有的回文都变成奇回文。我们的做法是插入一个特殊字符。比如 `ababa`，我们插入若干特殊字符（也叫做虚字符[^1]），变成 `#a#b#a#b#a#`（这样的串叫做 Manacher 扩展串），这样所有的偶回文都变成了奇回文。因此，以下所有的回文，如不加特殊说明，均指代奇回文。同时，如果我们从真实存在的字符开始扩展，我们就说这个真实存在的字符是所谓实轴，反之则称虚轴。

接下来我们定义回文直径、半径。

```
# a # b # a  # b # a #
1 3 1 7 1 11 1 7 1 3 1
1 2 1 4 1 6  1 4 1 4 1
```

上面第一行给出了回文直径，也就是以某个字符为中心，能够扩展形成回文串的最大长度。类似可以定义回文半径，也就是以某个字符出发，往某个方向走，走到最长回文串的最大步数。打个比方，`#` 只能走到自己，因此长度是 1。第一个 a 可以走到 `a#`（`#a#` 中），因此长度是 2。

因此，不难发现，对应回的原始长度就是**回文半径减 1**。我们还可以发现，真实串中的终止位置乘 2 就是扩展串中的位置。这是在说什么呢？`#a#b#a#` 的终止位置是下标 6，因此在原串中的位置就是 3（`abab`）

通过这样的方法，我们不会放过任何一个回文数组，这是非常好的。然而，上述暴力扩展的方法复杂度显然是 $O(n^2)$，这又非常不好了。

$[l,r)$ 的中心对应的就是 $p[l+r]$。 

## 优化——回文半径数组 $p$

我们接下来引入回文覆盖右边界 $r$ 和回文中心 $c$。

所谓回文覆盖右边界，就是指在前面向右侧扩展的过程中，**恰好**扩展**不到**的位置。

打个比方，对扩展串 $s=$`#a#b#c#`，$s[0]=\#$，它的 $r$ 就是 1，因为只有 $\#$ 是回文串。同理，$s[1]=a$，然后往右侧跑最远可以跑到 $s[2]$，所以 $r=3$。以此类推。只要突破了右边界，那就更新 $r$，与回文串长度无关。

那么回文中心 $c$ 呢？它就是最早取得最右边界的回文串中心。
![[Pasted image 20251219205011.png]]
打个比方，这里 $r=7$，对应的最早是 $s[3]$ 作为中心，而后还有 $s[5],s[6]$ 也满足最右边界是 7，但是 3 是最早的，所以 $c=3$。来到 $s[7]$ 的时候突破了，所以 $r$ 更新成为 9。相应的，$c$ 也更新为 7。

为什么要有这两个概念？——因为这样可以使得我们求回文串的过程更加方便。

来看下面这个例子。

![[Pasted image 20251219205437.png]]

初始 $r=0,c=0$，我们来模拟整个过程，记当前访问的下标是 $i$。
 - $i=0$，此时 $s[i]=\#$。$r=i$，此时没有被包住，因为 $r$ 的定义是第一个没有被包住的位置。这个时候暴力扩展。直接看左右两个。此时完成之后 $r=1,c=0$。回文半径为 1。
 - $i=1$，此时 $s[i]=a$。$r=i$，仍然没有被包住，继续暴力扩展。完成之后 $r=3,c=1$。回文半径为 2。
 - $i=2$，此时 $r>i$，属于被包住的情况。我们下面讲一讲怎么加速计算。

>[!remark] 用对称点加速回文半径的计算
>现在假想，我们在 $c$ 的位置扩充了一个非常远的 $r$，现在在范围内，我们有一个点 $i$（我们形象的说 $i$ 被 $r$ 包住，我们怎么确定回文半径呢？
>
>显然，在关于 $c$ 对称的位置，也就是 $i':=2c-i$，满足 $s[i]=s[i']$。对应的，$2c-i$ 也有一个回文半径。
> - 如果 $i'$ 为中心的回文串**完完全全**（不含恰好撞线）在 $c$ 确定的回文串内部，那么 $i$ 的回文半径显然等于 $i'$ 的回文半径。这是顺序确定好了的，亦即 $p[i]=p[i']$。
> - 如果 $i'$ 的回文串跑到了回文串以外（不含恰好撞线），那么，必然有 $i$ 的回文半径就是 $r-i$。这个正确性是显然的。$i'$ 既然扩充到了外面，而 $r$ 是第一个不满足回文的，那 $i$ 扩充撞线的时候已经不满足了（因为前面都得回文，而这里不一样。如果这里一样，那么 $r$ 应该更远）。因此有 $p[i]=r-i$。
> - 如果 $i'$ 的回文串边界恰好是回文串边界，这时候，我们需要扩展了。因为在 $c$ 确定的回文串边界内的部分，一定是满足会问的。但是在外面可能存在使得 $i$ 为中心扩展更远的回文半径。因此需要暴力扩展，同时更新 $r,c$。
> ![[Pasted image 20251219211159.png]]
> 在上面这个图里面，就有可能是 $?=e$ 的情况，因此需要暴力扩展，同时更新 $r,c$。

通过上面的更新，我们发现，在整个过程中，每个字符至多被访问 2 次。最劣的情况是被暴力扩展一次同时计算 $p$ 一次。因此，时间复杂度是 $O(|S|)$ 的。这是相当巧妙的方法。

代码如下。

```cpp
#include <bits/stdc++.h>

using namespace std;

const int MAXN = 1.1e7 + 5;

int p[MAXN << 1];

int manacher(const string &s) {
    string ss = "#";
    for (int i = 0; i < s.size(); i++) {
        ss.push_back(s[i]), ss.push_back('#');
    }
    int mx = 0, n = ss.size();
    for (int i = 0, c = 0, r = 0, len; i < n; i++) {
        len = r > i ? min(p[2 * c - i], r - i) : 1;
        while (i + len < n && ss[i + len] == ss[i - len]) len++;
        if (i + len > r) { r = i + len, c = i; }
        mx = max(len, mx);
        p[i] = len;
    }
    return mx - 1;
}

int main() {
    string s;
    cin >> s;
    cout << manacher(s) << endl;
    return 0;
}
```

回文半径数组的理解就是 Manacher 里面最重要的东西。

# Z 算法（扩展 KMP）

所谓的 Z 算法，就是要计算一个叫做 Z 数组的东西。

$z[i]$ 的定义是从 $i$ 开始的后缀串和原字符串的最长公共字符串的长度。打个比方，对 `aaabaac`，显然有 $z[0]=7,z[1]=2,\cdots, z[4]=2,\cdots$。

我们接下来看看怎么求 $z$ 数组。更特别地，怎么有优化地求 $z$ 数组。

$z[0]$ 是毫无用处的，我们从 $i=1$ 开始看看。
 - $i=1$，只能够暴力扩展。这样扩展的右边界（如 Manacher 定义）记作 $r$，同时我们记录 $c=i$。
 - 接下来，类似 Manacher 讨论。
	 - 如果 $i<r$，我们要找一个类似对称点的关键点。打个比方，如果像下图这样。
		![[Pasted image 20251219213911.png]]
		那么，我们思考，怎么去找这个关键点——我们实际上希望是看 s 从 3 开始的字符串和 s 本身有多大重叠，因此，我们直接考虑关键点 $i':=i-c$。
		- 那么，类似讨论发现 $z[i]=\min\{z[i'],r-i\}$。这是完全包裹 / 超出整个串两个决定的。而如果恰好碰到边界，我们也类似地进行暴力扩展。
	- 如果 $i=r$，直接暴力扩展即可。

因此，我们也实现了 $O(|S|)$ 计算 $z$ 数组！

类似，还有 $e$ 数组。给定两个字符串 $a,b$，定义 $e$ 数组满足
$$
e[i]:=\operatorname{LCS}(a[i..n], b[0..m]).
$$
完全类似之前的讨论，就可以轻松写出来代码了。

```cpp
#include <bits/stdc++.h>

using namespace std;

const int MAXN = 2e7 + 5;

#define int long long

int z[MAXN], e[MAXN];

void zArray(const string &s, int n) {
    z[0] = n;
    for (int i = 1, r = 1, c = 1, len; i < n; i++) {
        len = i < r ? min(z[i - c], r - i) : 0;
        while (i + len < n && s[i + len] == s[len]) len++;
        if (i + len > r) r = i + len, c = i;
        z[i] = len;
    }
}

void eArray(const string &s, const string &t, int n, int m) {
    for (int i = 0, r = 0, c = 0, len; i < n; i++) {
        len = i < r ? min(z[i - c], r - i) : 0;
        while (i + len < n && len < m && s[i + len] == t[len]) len++;
        if (i + len > r) r = i + len, c = i;
        e[i] = len;
    }
}

signed main() {
    string a, b;
    cin >> a >> b;
    zArray(b, b.size());
    eArray(a, b, a.size(), b.size());
    int ans = 0;
    for (int i = 0; i < b.size(); i++) ans ^= (i + 1) * (z[i] + 1);
    cout << ans << endl;
    ans = 0;
    for (int i = 0; i < a.size(); i++) ans ^= (i + 1) * (e[i] + 1);
    cout << ans << endl;
    return 0;
}
```


但是！一个更好的实现是考虑 $S+\texttt{"\#"}+T$，这样前面的位置就是 $s$ 的 z 数组，后面的每一个位置按照之前的定义就是 $s$ 的开头和 $t$ 的后缀的 e 数组！特别牛。

[^1]: 事实上，虚字符不必要是字符串中没有出现的字符。这是因为我们在扩展过程中，无论以实轴扩展还是以虚轴扩展，总是有实字符碰实字符，虚字符碰虚字符，不会出现实字符碰虚字符的情况。
